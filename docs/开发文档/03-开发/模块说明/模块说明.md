## 2. 各模块说明

### 2.1 api/v1/routes/ — API 路由

**职责**: 请求校验、调用 service、格式化响应。不含业务逻辑。

| 文件 | 接口 | 核心逻辑 |
|------|------|----------|
| import_routes.py | POST /import/start | 校验路径 → 调用 ingestion pipeline → 返回 task_id |
| | GET /import/status/{id} | 查询 import_logs |
| query_routes.py | POST /query (SSE) | 解析输入源 → 调用 retrieval pipeline → 流式返回 |
| | POST /query/preview | 只跑 Layer 1+2，不调 LLM |
| paper_routes.py | GET/DELETE /papers | paper_service CRUD |
| conversation_routes.py | GET/POST/DELETE /conversations | conversation_service CRUD |
| poll_routes.py | POST /poll/sync | 写入 Redis 缓存 |

### 2.2 clients/ — 外部服务客户端

**职责**: 封装外部 API 调用，处理认证、重试、错误转换。

| 文件 | 封装服务 | 关键点 |
|------|----------|--------|
| llm_client.py | DeepSeek 等（OpenAI 兼容） | LangChain ChatOpenAI，用户自配 URL/Key |
| vlm_client.py | Kimi Coding API | 三个必需 Header（UA 伪造），仅图片理解 |
| embedding_client.py | 硅基流动 | Qwen3-Embedding-8B，1536 维，批量处理 |
| mineru_client.py | MinerU API | PDF → MD + JSON，超时降级 |

**设计原则**: 每个客户端只负责一个外部服务，接口简单（调用→返回结果）。不关心业务上下文。

### 2.3 pipelines/ingestion/ — 数据清洗管道

**职责**: PDF 导入全流程。

```
service.py（编排）
  ├─ mineru_client.py → PDF 解析
  ├─ cleaner.py       → 噪音过滤
  ├─ image_handler.py → 图片理解（调 vlm_client）
  ├─ chunker.py       → 语义切块
  ├─ embedding_client → 向量化
  └─ zvec_store       → 入库
```

| 文件 | 职责 | 可复用旧代码 |
|------|------|-------------|
| cleaner.py | 正则过滤噪音，保留 MD+JSON 双格式 | `processing/cleaner.py` 基础规则 |
| image_handler.py | 提取图片 → 调 VLM → 标签附加 | 新写 |
| chunker.py | 语义切块（章节级），>32k 均分+重叠 | `processing/chunker.py` 核心逻辑 |
| service.py | 编排整个流程，记录 import_logs | 新写 |

### 2.4 pipelines/retrieval/ — 检索问答管道

**职责**: 三层检索架构的实现。

```
service.py（编排）
  ├─ source_understanding.py  → Layer 1: 意图 + 分源
  ├─ query_generation.py      → Layer 2: Dense/BM25 查询
  ├─ bm25_store + zvec_store  → 双路召回
  ├─ fusion.py                → Layer 3: 融合去重
  ├─ conversation_service     → 拼对话历史
  └─ llm_client               → 流式生成
```

| 文件 | 职责 | 说明 |
|------|------|------|
| source_understanding.py | 识别 intent，为每个 source 生成 dense_query + sparse_query | prompt→意图提取，selection→概念聚焦，draft→缺口检测 |
| query_generation.py | 为 Dense 和 BM25 生成各自的查询 | Dense 用自然语言，BM25 用关键词+实体 |
| fusion.py | source 内融合 → 跨 source 融合 → 去重 → Top-K | per_source_cap, per_doc_cap, weighted_rrf |
| service.py | 编排三层 + 拼 context + 调 LLM | 全流程 |

### 2.5 services/ — 业务服务

| 文件 | 职责 |
|------|------|
| conversation_service.py | Redis 对话 CRUD、滑动窗口管理、token 估算截断 |
| paper_service.py | 论文列表/详情/删除（SQLite 查 + Zvec delete_by_filter 联动） |

### 2.6 stores/ — 存储层

| 文件 | 封装 | 关键操作 |
|------|------|----------|
| zvec_store.py | Zvec 单 Collection | insert, query(filter), delete_by_filter, fetch |
| bm25_store.py | jieba + BM25 索引 | build_index, search(top_k), add_document, remove_document |
| sqlite_repo.py | SQLite | papers/chunks/import_logs/config 四表 CRUD |
| redis_cache.py | Redis | conversation 读写（LIST）、poll 缓存（STRING）、TTL 管理 |

### 2.7 models/ — 数据模型

Pydantic v2 模型，用于请求校验和响应序列化。

| 文件 | 模型 |
|------|------|
| paper.py | Paper, ImportTask, ImportStatus |
| query.py | QueryRequest, QuerySchema, SourceItem, FusionConfig |
| conversation.py | Message, Conversation |
| config.py | ConfigItem |

### 2.8 core/ — 基础设施

| 文件 | 职责 |
|------|------|
| config.py | pydantic-settings 加载 .env，暴露配置对象 |
| errors.py | 统一错误码（1000-5999），自定义异常类，全局异常处理器 |
| logging_config.py | structlog 或标准 logging 配置 |
| retry.py | 指数退避重试装饰器，用于 MinerU/Embedding/LLM 调用 |

## 3. 依赖方向

```
routes → services → pipelines → clients
                 ↘ stores ↗
routes → models（所有层共享）
core（所有层共享）
```

**禁止**: clients 直接依赖 stores，stores 直接依赖 pipelines。

## 4. 启动流程

```python
# main.py
app = FastAPI(lifespan=lifespan)

async def lifespan(app):
    # 1. 加载配置 (.env → config)
    # 2. 初始化 Zvec (create_and_open / open)
    # 3. 初始化 SQLite (建表)
    # 4. 初始化 Redis 连接
    # 5. 初始化 BM25 索引
    # 6. 注册路由
    yield
    # 7. 关闭 Zvec
    # 8. 关闭 Redis 连接
```
