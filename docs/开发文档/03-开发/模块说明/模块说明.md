## 2. 各模块详细说明

### 2.1 api/ — API 路由层

**职责**: 请求校验、调用下层服务、格式化响应。不含业务逻辑。

| 路由文件 | 端点 | 功能 |
|----------|------|------|
| health_routes.py | `GET /health` | 探测 SQLite/Zvec/Redis 状态 |
| import_routes.py | `POST /import/start` | 上传 PDF，启动后台导入任务 |
| | `GET /import/status/{id}` | 查询导入进度 |
| | `GET /import/stream/{id}` | SSE 实时进度流 |
| query_routes.py | `POST /query` | SSE 流式问答 |
| paper_routes.py | `GET /papers` | 列出所有论文 |
| | `DELETE /papers/{id}` | 删除论文（跨三存储联动） |
| conversation_routes.py | `GET /conversations/{id}` | 获取对话历史 |
| | `DELETE /conversations/{id}` | 清除对话历史 |
| poll_routes.py | `POST /poll/sync` | WPS 插件内容同步 |

**依赖注入** (`deps.py`): 启动时初始化 4 个单例（SQLite、Zvec、Redis、BM25），通过 `get_*()` 函数注入到路由。

### 2.2 clients/ — 外部服务客户端

**职责**: 封装外部 API 调用，处理认证、重试、错误转换。

#### mineru_client.py — MinerU PDF 解析

- `MinerUClient.submit_task(file_path)` → 申请上传链接 → 上传 PDF → 返回 task_id
  - data_id 自动截断到 120 UTF-8 字节（API 限制 128 字节）
- `MinerUClient.poll_task(task_id)` → 自适应间隔轮询（3s → 5s → 10s）
- `MinerUClient.fetch_result(task_id)` → 下载 ZIP → 解压到 `data/papers/{id}/`
- `MinerUClient.submit_batch(file_paths)` → 并发提交多个文件
- 超时: 300s，配置项 `MINERU_POLL_INTERVAL` / `MINERU_TIMEOUT`

#### vlm_client.py — Kimi VLM 图片描述

- `VLMClient.describe_image(path)` — 单图描述
- `VLMClient.describe_batch(paths, batch_size=5)` — 批量描述，5 张/请求打包
  - 构造多 image content blocks + JSON 输出 prompt
  - JSON 解析失败时自动降级为逐个单图请求
  - `_parse_batch_response()` 先尝试 JSON 数组，失败按序号分割
- 必需 headers: `User-Agent: claude-code`, `anthropic-version: 2023-06-01`, `x-api-key`
- 超时: 120s（批量请求需要更长时间）

#### llm_client.py — DeepSeek LLM

- `LLMClient.chat(messages)` → 同步完整响应
- `LLMClient.chat_stream(messages)` → 异步流式迭代器
- OpenAI 兼容协议，用户自配 `LLM_BASE_URL` / `LLM_API_KEY` / `LLM_MODEL`

#### embedding_client.py — 硅基流动 Embedding

- `EmbeddingClient.embed(texts)` → 批量向量化
- `EmbeddingClient.embed_single(text)` → 单条
- 批量参数: 32 条/批，并发 5，最多重试 3 次
- 重试触发: HTTP 429 + RemoteProtocolError
- 模型: Qwen3-Embedding-8B，1536 维（不可更改，否则索引失效）

### 2.3 pipelines/ingestion/ — 导入管道

**6 阶段流水线**，由 `service.py` 编排:

```
PDF → [1] MinerU解析 → [2] VLM图片描述 → [3] 清洗 → [4] 切块 → [5] Embedding → [6] 存储
```

#### service.py — 管道编排

- `IngestionService.ingest_pdf(file_path, progress_callback)` — 主入口
- 去重: 文件内容 SHA256 哈希，重复导入跳过
- 进度: 通过 callback 报告当前阶段，写入 SQLite import_logs
- 标题推测: 从 MD 前 3 行提取，或取文件名

#### cleaner.py — MD 清洗

- `_detect_zones(md)` — 区域检测:
  - `cover_toc`: 摘要/Abstract 之前的内容（封面 + 目录）
  - `body`: 核心正文
  - `end_noise`: 参考文献之后（致谢、附录、答辩表等）
- `_find_first_chapter()` — 章节检测（支持多种格式）:
  - `第1章`/`第一章`、`一、`/`（二）`（中文序号）
  - `Chapter 1`、`# 1 标题`（英文/数字编号）
- `_strip_toc_entries()` — 去除目录条目（标题 + 页码后缀的行）
- `_filter_noise()` — 后处理：空表格、答辩表、学位评阅表
- 摘要匹配: 标题格式 `# 摘要` + 行内格式 `摘 要：xxx`
- 支持中英文论文，中英文区域标记关键词

#### image_handler.py — VLM 图片描述注入

- `inject_image_descriptions(md, images_dir, paper_dir, vlm)` — 主函数
- 扫描 `![](images/xxx.jpg)` → 调 VLM → 替换为 `![hash8|描述](images/xxx.jpg)`
- 缓存: `image_descriptions.json` 按论文目录存储，二次运行零 API 调用
- 并发: Semaphore(10) 控制批量组并发

#### chunker.py — 固定切块

- `chunk_text(chunks, chunk_size=1000)` — 主函数
- 1000 token/块，段落边界对齐（不切断段落）
- 超长段落 (>2000 token) 按句号/换行二次切分
- `estimate_tokens(text)` — 中文约 1.5 字/token 估算

### 2.4 pipelines/retrieval/ — 检索管道

#### service.py — RAG 问答服务

- `QAService.query(request)` → `AsyncIterator[dict]` SSE 事件流
- 流程: Embedding 查询 → Zvec 密集检索 → 拼接上下文 → LLM 流式生成
- Token 预算: 上下文不超过 30k tokens
- 来源标注: 每个 chunk 的 paper_id + section + page
- 四种模式: 纯提问 / 圈选辅助 / 纯续写 / 全量（按字段组合自动识别）
- 对话历史: 从 Redis 加载，拼接 system prompt

> 注: 当前为简化版检索（单路 Dense），P5 阶段将升级为三层架构。

### 2.5 stores/ — 存储层

| 存储 | 实现 | 关键操作 |
|------|------|----------|
| Zvec | 单 Collection `papers`, payload 过滤 | `insert_chunks()`, `query(topk, paper_id)`, `delete_paper()` |
| BM25 | jieba 分词 + rank_bm25, JSON 磁盘缓存 | `add_documents()`, `query()`, `delete_paper()` |
| SQLite | papers/import_logs/config 三表, SQLAlchemy | `get_session()`, `get_paper_count()`, `init()` |
| Redis | 对话历史 LIST + 轮询 STRING, TTL 1h | `add_message()`, `get_messages()`, `set_poll_cache()` |

**Redis 降级**: 所有 Redis 操作 try/except 包裹，不可用时返回空值/跳过。

**Zvec 注意事项**:
- 过滤语法: `paper_id = 'xxx'`（单等号，不是 `==`）
- 初始化: `open()` 失败后 `shutil.rmtree()` 再 `create_and_open()`
- ID 格式: `{paper_id}_{chunk_index}`

**BM25 注意事项**:
- 删除是全量重建索引（rank_bm25 不支持增量删除）
- 文档 ID 按 paper_id 前缀匹配删除

### 2.6 models/ — 数据模型

| 文件 | 模型 | 用途 |
|------|------|------|
| paper.py | `PaperInfo`, `PaperListItem` | 论文列表/详情响应 |
| query.py | `QueryRequest`, `SourceRef`, `QueryMetadata` | 问答请求/来源引用 |
| conversation.py | `ChatMessage`, `ConversationCreate`, `ConversationResponse` | 对话管理 |

### 2.7 core/ — 基础设施

#### config.py

pydantic-settings 从 `.env` 加载，支持 Field alias 兼容旧变量名。
固定配置（不可更改）: `EMBEDDING_MODEL`, `EMBEDDING_DIMENSION`。

#### errors.py

错误码分段:
- `1001` ParamError — 参数校验失败
- `2001` PaperNotFoundError
- `2002` PaperExistsError
- `2003` ImportFailedError
- `3000+` ExternalServiceError

  │                           ↑
  └──→ pipelines/service ─────┘
          │         │

## 4. 导入管道详细时序

```
用户上传 PDF
    │
    ▼
import_routes.py → 创建 task_id → asyncio.create_task(ingest_pdf)
    │
    ▼
[Stage 1] MinerU 解析 ───→ full.md + layout.json + images/
    │                       (自适应轮询 3s→5s→10s，超时 300s)
    ▼
[Stage 2] image_handler ──→ 扫描 ![](images/xxx) → VLM 批量描述
    │                       (5 张/请求，失败逐个 fallback，JSON 缓存)
    ▼
[Stage 3] cleaner ────────→ 区域检测 → 噪音过滤 → 语义块列表
    │                       (cover_toc / body / end_noise)
    ▼
[Stage 4] chunker ────────→ 固定 1000 token 切块，段落边界对齐
    │
    ▼
[Stage 5] embedding ──────→ 批量 32 条，并发 5，429/断连自动重试
    │
    ▼
[Stage 6] 存储 ───────────→ Zvec + BM25 + SQLite 三路写入
    │
    ▼
完成 → paper_id, chunk_count, status='completed'
```
