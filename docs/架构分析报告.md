## 一、正确的数据流架构

### 1.1 PDF 导入与处理流程

```
┌──────────────┐
│ 用户上传 PDF │
└──────┬───────┘
       ▼
┌─────────────────────────────────────────────────────────┐
│           MinerU API (远程解析服务)                      │
│  • 返回 JSON 结构 (content_list_v2.json)                 │
│  • 返回图片文件 (images/)                                │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│           后端：数据清洗层 (Cleaning Service)            │
│  • 过滤噪声类型 (page_header/footnote/aside/number)      │
│  • UI 噪声过滤 (Expand table、Copilot 占位符等)         │
│  • LaTeX 清洗 (去除格式命令，保留内容)                   │
│  • 表格 HTML → 管道文本                                  │
│  • 追踪 section 层级                                      │
│  • 输出: Chunk[] (包含图片路径)                          │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│           后端：VLM 图片描述 (Describer Service)         │
│  • 调用 Kimi Coding API (kimi-for-coding 模型)          │
│  • 伪装请求头: User-Agent: claude-code                  │
│  • 10 并发请求，带缓存                                    │
│  • 输出: Chunk[] (content 包含图片详细描述)              │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│           后端：混合切分策略 (Chunker Service)           │
│  • 优先语义切分 (按 section/paragraph)                   │
│  • 如果超过 embedding 上下文 (32k) → 平均切分            │
│  • 目标: 接近 24k/chunk，留 8k 缓冲做首尾重叠           │
│  • 示例: 200k 语义块 → 8 个 chunk，每块约 25k + 重叠     │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│           后端：Embedding 服务                           │
│  • 调用硅基流动 Qwen3-Embedding-8B (4096维)             │
│  • 批量处理 (batch_size=32)                              │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│           后端：Qdrant 向量库存储                        │
│  • 分布式隔离管理 (每篇论文独立 collection)              │
│  • 存储向量 + 元数据 (图片路径、描述、section 等)        │
└────────────────────┬────────────────────────────────────┘
                     ▼
              ┌─────────────┐
              │ 导入完成    │
              └─────────────┘
```

### 3.2 发现的架构问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **向量库错误** | 使用 ChromaDB，应改用 Qdrant | 分布式隔离管理困难 |
| **切分策略错误** | 简单按元素切分，应使用混合策略 | 长文档处理不佳 |
| **PDF 解析错误** | 使用 PyMuPDF 本地解析，应调用 MinerU API | 解析质量差 |
| **图片处理缺失** | 没有 VLM 图片描述环节 | 图片信息丢失 |
| **分层混乱** | services/ 和 modules/ 职责重叠 | 代码维护困难 |
| **仓储重复** | repositories/ 和 modules/*/repo.py 功能重复 | 数据访问不一致 |

    """混合切分策略"""
    for chunk in chunks:
        size = count_tokens(chunk.content)
        if size <= EMBEDDING_MAX_CONTEXT:

### 5.3 Qdrant 存储结构

```python
# qdrant_store.py
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct

client = QdrantClient(path="./qdrant_db")

# 每篇论文独立 collection
def get_collection_name(paper_id: str) -> str:
    return f"paper_{paper_id}".replace("-", "_")

def create_paper_collection(paper_id: str, vector_size: int):
    client.create_collection(
        collection_name=get_collection_name(paper_id),
        vectors_config=VectorParams(size=vector_size, distance=Distance.COSINE),
    )

def add_chunks(paper_id: str, chunks: list[Chunk], embeddings: list[list[float]]):
    points = [
        PointStruct(
            id=chunk.id,
            vector=embedding,
            payload={
                "content": chunk.content,
                "section": chunk.section,
                "page": chunk.page,
                "image_path": chunk.image_path,
                "chunk_type": chunk.chunk_type,
            }
        )
        for chunk, embedding in zip(chunks, embeddings)
    ]
    client.upsert(
        collection_name=get_collection_name(paper_id),
        points=points,
    )
```
