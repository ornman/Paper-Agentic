# agent_layer 审计总览

## 审计范围

- 设计基线：
  - `docs/backend/architecture.md`
  - `docs/frontend/重构说明.md`
- 代码范围：
  - `backend/app/agent_layer`
  - `backend/app/service_layer/api/assistant_routes.py`
  - `backend/app/service_layer/api/conversation_routes.py`
  - `backend/app/service_layer/bootstrap/container.py`

## 已验证事实

1. 当前对话入口仍是 `POST /api/v1/conversations/chat`，没有前端 V2 需要的 `POST /api/v1/query`。
2. 前端设置页请求的是 `/api/v1/conversations/list?limit=50`，而后端当前只定义了 `GET /api/v1/conversations`。
3. `written_context / selection` 已可写入 Redis，但回答主链没有读取它们。
4. `backend/app/agent_layer/planning/input_assembler.py:28` 存在语法错误，导致 `AnswerGenerator` 当前无法 import。
5. `AnswerGenerator` 当前发的是 `metadata / chunk / done / error`，而前端消费的是 `thinking / block / sources / done / error`。
6. `AnswerGenerator` 当前只输出纯文本流和简化来源信息，`SourceCard.file_path / import_time` 等关键字段未生成。
7. `ChatModel` 只吃全局 `BackendSettings`，请求级 `model / thinking` 没有接入运行时。
8. `compact.py` 已存在，但没有被任何对话入口调用。
9. `conversation_window` Redis 存储已实现，但当前回答主链没有使用。
10. `container.py` 仍在 import 旧 `app.data_layer.storage.* / indexing.* / normalization.*` 路径，和当前 `backend/app/data_layer` 目录不一致。
11. 最小导入验证已复现两个阻断：
   - `ModuleNotFoundError: No module named 'openai'`
   - `SyntaxError` 来自 `input_assembler.py`
12. 前端当前源码虽然声明了 `selection / draft` 字段，但真实发送时只传 `prompt / paper_ids / enable_rag / model / thinking`；WPS 选区当前只是自动填充输入框，还没有调用 `/assistant/selection`。

## 总体结论

- 当前 `agent_layer` 还不是设计文档中的 Agent runtime，而是一个旧式 `chat` SSE 入口加一个 `AnswerGenerator`。
- 主要缺口不在“模块数量不够”，而在“主链没有闭合”：
  - 输入快照没闭合
  - SSE 协议没闭合
  - 会话事实与运行态没闭合
  - request 级模型配置没闭合
- 现在最先要修的不是“回答质量”，而是“接口和运行时骨架”。

## 优先级建议

1. 先修 P0：
   - 新建 `/api/v1/query`
   - 修 `input_assembler.py` 语法错误
   - 处理 `openai` 缺依赖的启动失败
   - 修 `container.py` 旧路径导入
2. 再修 P1：
   - 接入 `written_context / selection`
   - 切换到 `thinking / block / sources / done / error`
   - 补齐结构化持久化
3. 最后补 P2：
   - retrieval loop
   - reflection
   - compact 自动触发
   - request 级模型能力与模型列表接口
