# agent_layer 测试设计总览

## 目标

- 测试不再只验证“函数能不能跑”，而是验证 Agent 层在真实索引、真实 SSE、真实模型配置下的链路行为。
- 测试分三层：
  1. 接口契约测试：校验 `/query`、`assistant context`、历史接口、SSE 事件协议。
  2. 模块行为测试：校验 snapshot、planning、session、compact、runtime 降级行为。
  3. 真实链路验证：用真实中英文论文、真实 Redis / SQLite / Chroma / BM25、真实 LLM 配置去跑整条回答链。

## 样本池

- 中文样本：`backend/tests/UnitTesting-Docling&&Anthropic'sPDF&DOCX/Testpaper-PDF-ZH-CN`，43 份 PDF。
- 英文样本：`backend/tests/UnitTesting-Docling&&Anthropic'sPDF&DOCX/Testpaper-PDF-EN`，111 份 PDF。

## 前置条件

1. 先用 `data_layer` 当前可导入链路构建一批真实索引样本。
2. 至少准备：
   - 2 篇中文论文
   - 2 篇英文论文
   - 每篇都已完成 chunk + embedding + indexing
3. Redis、SQLite、Chroma、BM25 必须用真实实例，不用 mock。
4. LLM 若配置存在，必须走真模型；若未配置，标记 `blocked`，不能用 mock 假装通过。

## ClaudeCode 执行规则

1. 统一使用 `uv run pytest`。
2. 真实链路测试必须发真实 HTTP 请求，解析真实 SSE 帧。
3. 对本轮审计已确认的 P0 缺陷，先固化红灯：
   - `/api/v1/query` 缺失
   - `input_assembler.py` 导入失败
   - `conversation/list` 路由缺失
4. 对 Redis 不可用、LLM 不可用这类场景，不用 mock 伪装成功；要验证真实降级输出。
5. 每轮测试都保留 artifacts，建议目录：
   - `backend/tests/_artifacts/agent_layer/<run_id>/`

## 执行阶段

1. Phase A：接口红灯固化。
2. Phase B：补齐 `/query` 后，跑通 SSE 协议和结构化回答最小链。
3. Phase C：接入 `written_context / selection`，验证 snapshot freeze。
4. Phase D：跑真实中英文论文问答链。
5. Phase E：跑长对话 compact 和历史回放。

## 当前基线测试处理建议

- 不建议在旧的 `conversation/chat` 测试上继续缝补。
- 建议新建测试根：
  - `backend/tests/agent_layer/`
  - `backend/tests/service_layer/query_api/`
