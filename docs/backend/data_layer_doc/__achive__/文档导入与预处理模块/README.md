# 文档导入与预处理模块

## 1. 模块定位

这个文件夹承接 [architecture.md](../architecture.md) 里“文档导入、预处理、归一化”那一层的实现文档。

总原则：

- `architecture.md` 只写全局目标、边界、约束、模块关系
- 具体到某个文档类型、某条路由、某个探针、某种增强方式，单独放在模块文档里
- 模块文档之间用相对路径互链，不再把实现细节堆回总架构文档

## 2. 这个模块管什么

这一层负责：

- 接收本地文档路径
- 做格式识别与轻量探针
- 做路由决策
- 执行转换 / 清洗 / 增强
- 统一产出可供后续索引消费的标准结果

统一产物契约至少包括：

- `markdown`
- `structured json`
- `extraction report`
- `anchors`

## 4. 当前已锁定的模块级结论

- 预处理模块不再默认绑定单一解析器
- `PDF` 与 `DOCX` 在数据契约层面同级支持
- `PDF` 当前默认方向是轻量探针 + 多策略路由 + API 增强
- 重模型能力不作为当前普通机器的默认本地链路
- 预处理后的统一产物，而不是某个具体工具的原始输出，才是数据层后续消费对象
- 当前这一轮先把 `VLM-first` 的视觉块增强主链定稳，`OCR` 作为下一阶段增强能力单独推进
