# ADR-005: 后端主链重构先于评测体系接入

## Status

Accepted

## Context

当前后端还处于混合态：

- 旧目录残留还没有冻结和清理
- 数据层、Agent 层、服务层还没有按目标边界重建完成
- 活动模型基线和示例配置也还在收口

如果这时先接评测、A/B 或 benchmark，会产生两个问题：

1. 测到的是混合态实现，不是目标主链
2. 调优结论会被旧目录、旧契约、旧默认值污染

## Decision

- 评测体系不取消，但明确后置
- 顺序固定为：
  - 先收敛活动链路与遗留目录边界
  - 再重建数据层
  - 再重建 Agent 层
  - 再重建服务层
  - 最后接黑盒、内部态、A/B 评测

## Consequences

正面：

- 调优对象变成真正的活动主链
- 避免在旧实现上浪费评测和分析成本
- 文档、代码、评测顺序重新一致

代价：

- 评测体系需要延后落地
- 当前阶段优先产出是重构骨架，不是评分报表
