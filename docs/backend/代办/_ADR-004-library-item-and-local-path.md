# ADR-004: 导入入口固定为 Windows 本地路径，并强制保留原文锚点

## Status

Accepted

## Context

这个项目当前的真实运行环境不是公网 SaaS，而是：

- Windows 本机
- 本地文件
- WPS 插件协作写作

如果继续在上传式接口、本地路径、远程 URL 之间摇摆，会把导入链、权限边界、回跳能力全弄乱。

同时，用户的核心诉求之一是：

- 回答必须引用原文
- 必须能回到原始 PDF 或 DOCX
- 至少做到段落级锚点

## Decision

- 第一版导入入口固定为 Windows 本地绝对路径
- 第一版只支持本地 `PDF / DOCX`
- 导入后必须保存统一原文锚点结构
- 宿主侧能否一步跳到该位置，不影响后端必须产出锚点

## Consequences

正面：

- 安全边界和输入契约清楚
- 后端可以围绕“本地文档 -> 结构化产物 -> 锚点 -> 回跳”稳定建模
- 不会再为了兼容上传式接口破坏主链

代价：

- 当前版本不追求通用远程部署接口
- 如果未来要支持上传或远程 URL，需要单独新增入口和安全设计
