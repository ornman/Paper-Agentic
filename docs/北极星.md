# 北极星 - WPS 论文写作辅助工具

> **创建日期**: 2026-03-23
> **状态**: 北极星

基于私有文献知识库的论文写作辅助工具，以 WPS 插件形式嵌入写作场景。

---

## 1. 愿景

让学术写作者拥有一个**能自动收集、整理、理解知识的 AI 助手**。

解决的核心问题：
- **信息收集难** - 找文献、筛文献耗时耗力
- **知识管理乱** - 文献分散、格式不统一、难以复用
- **检索效率低** - 找不到、找不准、找不到出处
- **生成不可信** - AI 容易编造不存在的内容

---

## 2. 产品定位

| 维度 | 定位 |
|------|------|
| **形态** | WPS 插件（侧边栏 TaskPane） |
| **用户** | 高校学生、科研工作者、知识工作者 |
| **场景** | 写作过程中的知识收集、整理、检索、生成 |
| **核心价值** | 从信息收集到可信生成的完整闭环 |

## 3. 能力版图

### 3.1 完整架构

```
DeepResearch ──→ SpiderMan ──→ 清道夫 ──→ RAG层 ──→ Agentic
   (搜索分析)      (智能爬取)     (格式清洗)    (检索增强)   (多智能体)
```

### 3.2 各层能力

| 模块 | 职责 | 能力 |
|------|------|------|
| **DeepResearch** | 自动联网搜索与分析 | 搜索引擎集成、信息源分析、爬取决策、为 SpiderMan 铺路 |
| **SpiderMan** | 智能爬虫 | 只要合法都能爬（视频、PDF、Word、图片、网页）、精准定位有价值信息 |
| **清道夫** | 本地数据处理 | 格式转换、数据清洗、结构化提取、为 RAG 铺路 |
| **RAG 层** | 检索增强生成 | 向量检索、GraphRAG、Neo4j、AdvanceRAG、模型微调支持 |
| **Agentic** | 多智能体架构 | 长期记忆、多 Agent 协作、自主决策 |

### 3.3 用户侧能力

- **知识收集** - 自动搜索 + 一键爬取 + 批量导入
- **知识管理** - 文献库管理 + 自动分类 + 标签系统
- **智能检索** - 语义检索 + 关键词检索 + 知识图谱查询
- **可信生成** - 基于私有知识库的问答与续写 + 来源溯源
- **写作辅助** - 格式排版 + 参考文献管理 + 查重预警

## 4. 阶段规划

### 4.1 阶段总览

| 阶段 | 核心 | 里程碑 |
|------|------|--------|
| **MVP** | AdvanceRAG 闭环 | PDF 数据清洗 + 检索增强生成 |
| **第二阶段** | 清道夫升级 | 清洗质量提升 + 多格式支持 |
| **第三阶段** | Agentic | 多智能体架构 + 长期记忆 |
| **第四阶段** | DeepResearch | 自动搜索 + SpiderMan 爬虫 |
| **第五阶段** | 打磨 | 知识图谱 + 细节优化 |

### 4.2 各阶段详解

#### MVP：AdvanceRAG 闭环
- **目标**：跑通 AdvanceRAG 流程
- **能力**：
  - PDF 导入
  - 数据清洗（MinerU）
  - 检索增强生成（向量 + BM25 + Rerank）
  - 问答 + 续写
  - 页码溯源

#### 第二阶段：清道夫升级
- **目标**：提升数据质量，扩展格式支持
- **能力**：
  - 清洗质量优化
  - 多格式支持（Word、PPT、图片、视频）
  - 本地化清洗（降低 API 成本）

#### 第三阶段：Agentic
- **目标**：多智能体架构
- **能力**：
  - 多 Agent 协作
  - 长期记忆
  - 自主决策

#### 第四阶段：DeepResearch
- **目标**：自动收集信息
- **能力**：
  - DeepResearch（自动联网搜索 + 分析）
  - SpiderMan（智能爬虫，合法即爬）
  - 完整数据流打通

#### 第五阶段：打磨
- **目标**：精细化 + 知识图谱
- **能力**：
  - 知识图谱（Neo4j / GraphRAG）
  - 体验优化
  - 性能调优

## 5. 技术演进

### 5.1 技术栈演进

| 层级 | MVP | 二阶段 | 三阶段 | 四阶段 | 五阶段 |
|------|-----|--------|--------|--------|--------|
| 前端 | WPSJS + Vue 3 | 同上 | 同上 | 同上 | 同上 |
| 后端 | FastAPI + LangChain | 同上 | 多智能体框架 | 同上 | 同上 |
| 向量库 | ChromaDB | 同上 | 同上 | 同上 | + Neo4j |
| 检索 | BM25 + 向量 + Rerank | 同上 | 同上 | 同上 | + GraphRAG |
| 清洗 | MinerU API | 本地化 + 多格式 | 同上 | 同上 | 同上 |
| 爬虫 | - | - | - | SpiderMan | 同上 |

### 5.2 技术选型原则

- **MVP**：能用 API 就用 API，快速闭环
- **二阶段**：核心能力本地化，提升质量
- **三阶段**：架构升级，支持多智能体
- **四阶段**：能力扩展，自动收集
- **五阶段**：精细化，知识图谱

## 6. 边界

### 6.1 产品边界
