## 四、硬编码路径检查

### 4.1 后端路径

**【代码事实】** `app/core/config.py`

```python
qdrant_path: str = Field(
    default="./data/qdrant",
    description="Qdrant 本地存储路径",
)
sqlite_db_path: str = Field(
    default="./data/app.db",
    description="SQLite 数据库路径",
)
bm25_index_path: str = Field(
    default="./data/bm25_index.json",
    description="BM25 索引文件路径",
)
```

**￥问题￥：相对路径问题**

**问题**：
- 使用`./data/`相对路径
- 依赖工作目录
- 不同启动方式路径不同

**影响**：
- 从不同目录启动会找不到数据
- 可能在错误位置创建数据目录

**示例**：
```bash
# 从backend目录启动
cd backend
python main.py  # ✅ 数据在backend/data/

# 从项目根目录启动
python project/MVP/backend/main.py  # ❌ 数据在根目录/data/
```

**建议**：
```python
from pathlib import Path

BASE_DIR = Path(__file__).parent.parent
DATA_DIR = BASE_DIR / "data"

qdrant_path: str = Field(
    default=str(DATA_DIR / "qdrant"),
)
```

### 4.2 前端路径

**【代码事实】** `src/services/api-client.ts`

```typescript
const API_BASE_URL = 'http://127.0.0.1:8000';
```

**￥问题￥：硬编码API地址**

**问题**：
- 开发环境和生产环境可能不同
- 无法动态配置

**建议**：
```typescript
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://127.0.0.1:8000';
```

## 五、网络环境差异

### 5.1 代理配置

**【全局配置】**（来自用户全局配置）：
- 默认使用系统代理（Clash）
- 不在代码中显式配置

**差异点**：
- ✅ 开发环境：有代理
- ❌ 生产环境：可能无代理

**影响**：
- 外部API依赖代理
- 代理故障导致全部不可用

**验证**：
```python
import requests

try:
    response = requests.get("https://api.kimi.com", timeout=5)
    print(f"代理可用: {response.status_code}")
except Exception as e:
    print(f"代理失败: {e}")
```

### 5.2 localhost访问

**后端**：
- 监听: `127.0.0.1:8000`
- 只允许本地访问

**前端**：
- WPS插件本地加载
- 访问 `http://127.0.0.1:8000`

**跨域**：
- FastAPI已配置CORS
- 允许所有来源（开发）

**【代码事实】** `app/main.py`

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 生产环境应限制
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 七、依赖差异

### 7.1 Python依赖

**开发环境**：
- 使用`uv`管理
- 所有依赖在`pyproject.toml`

- 所有依赖在`package.json`

**生产环境**：
- WPS内置V8引擎
- 不需要Node.js

**差异**：
- 开发时需要Node.js
- 生产时不需要

## 十、环境检查清单

### 10.1 启动前检查

```python
def check_environment():
    """环境检查."""
    issues = []
    
    # 1. 检查数据目录
    data_dir = Path(settings.qdrant_path).parent
    if not data_dir.exists():
        issues.append(f"数据目录不存在: {data_dir}")
    
    # 2. 检查API密钥
    if not settings.kimi_api_key:
        issues.append("KIMI_API_KEY未配置")
    if not settings.siliconflow_api_key:
        issues.append("SILICONFLOW_API_KEY未配置")
    
    # 3. 检查网络连接
    try:
        requests.get("https://api.kimi.com", timeout=5)
    except Exception:
        issues.append("无法连接到Kimi API（检查代理）")
    
    return issues
```
