### 沙箱限制

| 资源 | 限制 | 影响 | 解决方案 |
|------|------|------|---------|
| **网络请求** | 受浏览器 CORS 限制 | 无法直接调用 API | 需要后端代理 |

## 架构审查发现的问题

**￥问题 #20：环境区分自动开启 debug 存风险￥**

**维度**: 安全与合规
**严重性**: P1
**位置**: `core/config.py:272-274`

**问题描述**:
开发环境自动强制开启 debug，可能导致敏感信息泄露。

**代码证据**:
```python
# core/config.py:272-274
@property
def effective_debug(self) -> bool:
    return self.debug or self.is_development  # ⚠️ 自动开启 debug
```

**潜在影响**:
- 🔴 敏感信息泄露: debug 模式会打印详细错误信息
- 🔴 配置泄露: debug 模式可能暴露环境变量、API Key
- 🔴 性能下降: debug 模式有额外开销

**建议方向**:
1. 移除自动开启逻辑
2. 显式配置 `debug=true`
3. 生产环境强制校验 `debug=false`

**￥问题 #21：启动时缺少配置验证￥**

**维度**: 架构与设计
**严重性**: P2
**位置**: `main.py:47-52`

**问题描述**:
启动时只初始化数据库，不验证 API Key、数据目录等关键配置。

**代码证据**:
```python
# main.py:47-52
@app.on_event("startup")
async def startup():
    """启动时初始化数据库"""
    from app.repositories.sqlite_repo import get_engine
    get_engine()  # 触发建表
    print("数据库初始化完成")
    # ❌ 缺少:
    #    - API Key 验证
    #    - 数据目录验证
    #    - Qdrant 连接验证
```

**潜在影响**:
- 🔴 启动后才发现配置错误: 如 API Key 无效
- 🔴 运行时崩溃: 如数据目录无写权限
- 🔴 用户体验差: 发现问题时已部分初始化

**建议方向**:
添加完整的启动验证:
```python
@app.on_event("startup")
async def startup():
    """启动时初始化并验证"""
    # 1. 验证 API Key
    if not settings.deepseek_api_key:
        raise ValueError("DEEPSEEK_API_KEY is required")

    # 2. 验证数据目录
    data_dir = Path(settings.sqlite_db_path).parent
    if not data_dir.exists():
        data_dir.mkdir(parents=True)
    if not os.access(data_dir, os.W_OK):
        raise ValueError(f"Data directory not writable: {data_dir}")

    # 3. 验证 Qdrant 连接
    try:
        from app.stores.qdrant_store import QdrantStore
        store = QdrantStore()
        store.client.get_collections()  # 测试连接
    except Exception as e:
        raise ValueError(f"Qdrant connection failed: {e}")

    # 4. 初始化数据库
    from app.repositories.sqlite_repo import get_engine
    get_engine()
    print("✅ 所有验证通过")
```
