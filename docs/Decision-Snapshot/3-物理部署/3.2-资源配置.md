## 磁盘占用估算表

### 数据目录结构

```
data/
├── qdrant/                  # Qdrant 本地存储
│   └── {collection_name}/   # 每篇论文一个 collection

│       └── images/          # 提取的图片

    └── {filename}.pdf       # 上传的 PDF
```

**证据**: `core/config.py:177, 239`, `stores/qdrant_store.py:62-63`

### 磁盘占用模型

| 数据类型 | 单篇占用 | 10 篇 | 100 篇 | 500 篇 | 2000 篇 |
|---------|---------|-------|-------|-------|---------|
| **原始 PDF** | 2-10 MB | 50 MB | 500 MB | 2.5 GB | 10 GB |
| **MinerU JSON** | 500 KB - 2 MB | 10 MB | 100 MB | 500 MB | 2 GB |
| **提取图片** | 1-5 MB | 25 MB | 250 MB | 1.25 GB | 5 GB |
| **Qdrant Collection** | 1-3 MB | 20 MB | 200 MB | 1 GB | 4 GB |
| **SQLite 数据** | 10 KB | 100 KB | 1 MB | 5 MB | 20 MB |
| **合计（每篇）** | 4-20 MB | 105 MB | 1.05 GB | 5.25 GB | 21 GB |

**估算依据**:
- PDF: 平均 5 MB（学术论文）
- MinerU JSON: 约 PDF 的 10-20%
- 图片: 每篇论文约 5-10 张图片，每张 200-500 KB
- Qdrant Collection: 向量数据（1536 维 × 4 字节 × 100-500 chunks）

**证据**: `modules/ingestion/service.py:50-111`（导入流程）

### 清理机制

**当前状态**: ❌ **无自动清理**

**证据**: 扫描整个代码库，未发现定时清理任务

### 临时文件位置

| 位置 | 用途 | 大小 | 清理建议 |
|------|------|------|---------|
| `data/uploads/` | 临时上传文件 | 取决于上传频率 | ✅ 每天清理 |
| `data/papers/*/mineru.json` | MinerU 解析结果 | 500 KB - 2 MB/篇 | 🟡 按需清理 |
| `data/papers/*/images/` | 提取的图片 | 1-5 MB/篇 | 🟡 按需清理 |

### 清理建议

**方案 1: 定时清理**
```python
# 每天凌晨清理 uploads/
import shutil
from pathlib import Path
from datetime import datetime, timedelta

uploads_dir = Path(settings.sqlite_db_path).parent / "uploads"
if uploads_dir.exists():
    for file in uploads_dir.glob("*.pdf"):
        # 删除超过 1 天的文件
        if file.stat().st_mtime < (datetime.now() - timedelta(days=1)).timestamp():
            file.unlink()
```

**方案 2: 删除论文时同步清理**
```python
# api/v1/routes/library.py:178-197
@router.delete("/papers/{paper_id}", response_model=ApiResponse)
async def delete_paper(paper_id: str):
    # 1. 删除 Qdrant collection
    store.delete_paper(paper_id)

    # 2. 删除文件系统数据（已实现）
    papers_dir = Path(settings.sqlite_db_path).parent / "papers" / paper_id
    if papers_dir.exists():
        shutil.rmtree(papers_dir)  # ✅ 已实现
```

**证据**: `api/v1/routes/library.py:192-195`

## 架构审查发现的问题

**￥问题 #18：无临时文件自动清理机制￥**

**维度**: 可观测性与运维
**严重性**: P2
**位置**: 整体架构

**问题描述**:
`data/uploads/` 目录中的临时上传文件无自动清理机制，会累积占用磁盘空间。

**证据**:
```python
# api/v1/routes/library.py:41-48
upload_dir = Path(settings.sqlite_db_path).parent / "uploads"
upload_dir.mkdir(parents=True, exist_ok=True)
pdf_path = upload_dir / file.filename

with pdf_path.open("wb") as f:
    content = await file.read()
    f.write(content)
# ⚠️ 上传后不清理，即使导入成功
```

**潜在影响**:
- 🔴 磁盘空间泄漏: 上传越多，占用越多
- 🔴 用户隐私: 上传的 PDF 可能包含敏感信息

**建议方向**:
1. 导入成功后删除上传文件
2. 添加定时清理任务（如每天凌晨）
3. 设置文件过期时间（如 1 小时）

**￥问题 #19：Qdrant 内存优化配置不足￥**

**维度**: 架构与设计
**严重性**: P2
**位置**: `stores/qdrant_store.py:99-109`

**问题描述**:
虽然降低了 HNSW 参数，但 Qdrant 内存占用仍然很高（2000 篇论文约 6 GB），小内存机器可能无法运行。

**证据**:
```python
# stores/qdrant_store.py:99-109
self.client.create_collection(
    collection_name=collection_name,
    vectors_config=VectorParams(
        size=vector_size,  # 1536 维
        distance=Distance.COSINE,
        hnsw_config=HnswConfigDiff(
            m=settings.qdrant_hnsw_m,              # 12（已优化）
            ef_construct=settings.qdrant_hnsw_ef_construct,  # 64（已优化）
            full_scan_threshold=settings.qdrant_full_scan_threshold,  # 5000
        ),
    ),
)
```

**内存占用估算**（2000 篇论文）:
- 向量数据: 1,000,000 chunks × 6 KB = 6 GB
- HNSW 索引: 6 GB × 20% = 1.2 GB
- **总计**: 约 7.2 GB

**潜在影响**:
- 🔴 内存溢出: 小内存机器（< 8 GB）无法运行
- 🔴 性能下降: 内存不足导致频繁 swap

**建议方向**:
1. 使用量化向量（如 8-bit 量化）
2. 分片存储（按时间或类别）
3. 使用远程 Qdrant 服务（而非本地）
4. 定期清理旧论文（LRU 策略）
