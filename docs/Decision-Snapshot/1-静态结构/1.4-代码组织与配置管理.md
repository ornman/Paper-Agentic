def validate_pinned_model_contract(self) -> "Settings":
    """校验模型契约，防止向量维度和模型漂移污染索引。"""
    if self.embedding_model != PINNED_EMBEDDING_MODEL:

    if self.embedding_dimensions != PINNED_EMBEDDING_DIMENSIONS:

    if self.rerank_model != PINNED_RERANK_MODEL:
        raise ValueError(f"RERANK_MODEL 必须固定为 {PINNED_RERANK_MODEL}")

    return self

#### 配置校验

**硬约束**（`config.py:286-298`）：
```python
@model_validator(mode="after")
def validate_pinned_model_contract(self) -> "Settings":
    if self.embedding_model != PINNED_EMBEDDING_MODEL:
        raise ValueError(f"EMBEDDING_MODEL 必须固定为 {PINNED_EMBEDDING_MODEL}")
    # ...
    return self
```

**理由**：
- 模型漂移会导致向量库失效
- 维度不一致会导致检索失败
- 系统级契约不允许随意更改

#### 兼容层

**证据**：`config.py:300-341`
```python
@property
def llm_api_key(self) -> str:
    """兼容旧字段名，映射到 DeepSeek API Key。"""
    return self.deepseek_api_key

@property
def embedding_api_key(self) -> str:
    """兼容旧字段名，Embedding 统一复用 SiliconFlow API Key。"""

```

**目的**：平滑迁移，不破坏现有代码

## ￥问题￥ - 代码组织缺陷

### 1. ￥配置文件膨胀￥

**事实**：`config.py` 有 367 行，50+ 配置项

**问题**：
- 单文件承载过多职责
- 字段定义和验证逻辑混在一起
- 难以快速定位配置项

**建议**：
```
app/core/config/
├── base.py          # 基础配置
├── llm.py           # LLM 配置
├── vector_store.py  # 向量库配置
├── ingestion.py     # 导入配置
└── __init__.py      # 聚合导出
```

### 2. ￥前端状态机过于复杂￥

**事实**：`conversation.ts` 有 472 行

**问题**：
- 状态逻辑、SSE 处理、消息管理混在一起
- 5 种状态但只有 3 条有效路径
- 难以测试和维护

**建议**：
```typescript
// stores/conversation/
├── stateMachine.ts  # 状态机逻辑
├── sseHandler.ts    # SSE 处理
├── messageManager.ts # 消息管理
└── index.ts         # 主导出
```

### 3. ￥VLM 提示词硬编码￥

**事实**：`kimi_client.py:34-105` 有 105 行提示词

**问题**：
- 提示词无法在不修改代码的情况下调整
- 无法 A/B 测试不同提示词
- 多语言支持困难

**建议**：
```python
# app/prompts/vlm_image_description.md
你是一个专业的学术文献图表分析专家...
```

```python
# app/clients/kimi_client.py
def load_vlm_prompt() -> str:
    path = Path(__file__).parent.parent / "prompts" / "vlm_image_description.md"
    return path.read_text(encoding="utf-8")
```

### 4. ￥缺少配置验证工具￥

**事实**：只有启动时的硬约束校验

**问题**：
- 无法预先检查配置有效性
- API Key 过期才知道
- 无配置文档生成

**建议**：
```bash
uv run check-env  # 检查环境变量
uv run validate-config  # 验证配置有效性
uv run generate-config-docs  # 生成配置文档
```
