## ￥问题￥ - 架构缺陷

### 后端问题

1. **￥缺失的存储层隔离￥**
   - **事实**：`stores/` 目录只有 `qdrant_store.py`，缺少 BM25 索引存储
   - **位置**：`app/stores/`
   - **影响**：关键词检索依赖直接实现，无法统一管理

2. **￥VLM并发控制不完善￥**
   - **事实**：`describer.py:79` 使用 `asyncio.Semaphore(CONCURRENCY)` 但配置 `kimi_vlm_concurrency=10`
   - **位置**：`app/processing/describer.py:79`
   - **风险**：高并发时可能触发 Kimi API 限流

3. **￥错误传播不一致￥**
   - **事实**：`ingestion/service.py:113` 捕获所有异常转为 `IngestionError`
   - **位置**：`app/modules/ingestion/service.py:113-121`
   - **风险**：丢失原始错误堆栈，难以调试

### 前端问题

4. **￥状态机复杂度转移￥**
   - **事实**：`conversation.ts` 有472行，包含完整状态机逻辑
   - **位置**：`src/stores/conversation.ts:87-472`
   - **影响**：状态管理过于复杂，难以维护

5. **￥缺失的错误边界￥**
   - **事实**：所有组件都是 `try-catch` 在 store 层，组件层无错误边界
   - **位置**：`src/components/`
   - **风险**：一个组件崩溃可能导致整个应用白屏
