### Qdrant 向量库存储

**位置**：`app/stores/qdrant_store.py`【298行】

#### 分布式隔离架构

**设计决策**：每篇论文一个独立 Collection

**证据**：`qdrant_store.py:29-38`
```python
def _get_collection_name(paper_id: str) -> str:
    """获取 collection 名称（合法化处理）."""
    name = f"paper_{paper_id}".replace("-", "_").replace(" ", "_")
    # 移除非法字符
    import re
    name = re.sub(r"[^a-zA-Z0-9_-]", "", name)
    return name.lower()
```

**优势**：
- 清理简单：直接删 Collection（`qdrant_store.py:237-249`）
- 扩展性好：单 Collection 不会无限增长
- 隔离性强：不同论文数据完全隔离

#### Collection 配置（内存优化）

**证据**：`qdrant_store.py:100-110`
```python
self.client.create_collection(
    collection_name=collection_name,
    vectors_config=VectorParams(
        size=vector_size,
        distance=Distance.COSINE,
        hnsw_config=HnswConfigDiff(
            m=settings.qdrant_hnsw_m,           # 默认12（标准16）
            ef_construct=settings.qdrant_hnsw_ef_construct,  # 默认64（标准100）
            full_scan_threshold=settings.qdrant_full_scan_threshold,  # 默认5000
        ),
    ),
)
```

**参数含义**：
- `m=12`：每个向量连接的最大节点数（降低20%）
- `ef_construct=64`：构建索引时的搜索范围（降低36%）
- `full_scan_threshold=5000`：小于此值直接全扫描

**代价**：召回率可能轻微下降（<2%）

## ￥问题￥ - 存储架构缺陷

### 1. ￥VLM缓存残留风险￥

**事实**：`describer.py:84-88`
```python
except Exception as e:
    print(f"[WARN] VLM failed for {chunk.image_path}: {e}")
    return index, str(chunk.image_path), "description unavailable"
```

**问题**：
- VLM 失败时不清理缓存
- 下次启动会加载 `"description unavailable"`
- 无法区分"未描述"和"描述失败"

**建议**：
- 失败时删除缓存条目
- 或标记为 `{"status": "failed", "message": "..."}`
- 重试时重新调用 VLM

### 2. ￥Qdrant Collection 命名冲突￥

**事实**：`qdrant_store.py:31-36`
```python
name = f"paper_{paper_id}".replace("-", "_").replace(" ", "_")
name = re.sub(r"[^a-zA-Z0-9_-]", "", name)
```

**问题**：
- 不同 `paper_id` 可能生成相同 Collection 名
- 例如：`paper/test-1` 和 `paper/test_1` 都变成 `paper_test_1`

**风险**：
- 数据混淆
- Collection 覆盖

### 3. ￥SQLite 无外键约束￥

**事实**：`repository.py` 无外键定义

**问题**：
- `session.document_id` 可以指向不存在的文档
- 删除文档时不会级联删除关联会话
- 数据完整性无保障

### 4. ￥缺少软删除机制￥

**事实**：`qdrant_store.py:237-249`
```python
def delete_paper(self, paper_id: str) -> None:
    collection_name = _get_collection_name(paper_id)
    try:
        self.client.delete_collection(collection_name)
    except Exception:
        pass
```

**问题**：
- 删除操作不可逆
- 无法恢复误删数据
- 无审计日志

## 性能指标

### Qdrant 性能

**配置**（`app/core/config.py:189-207`）：
- `hnsw_m=12`：每向量 12 个连接（标准 16）
- `ef_construct=64`：构建时 64 个候选（标准 100）
- `full_scan_threshold=5000`：小于 5000 点直接全扫描

**预估性能**（基于 Qdrant 官方文档）：
- 单 Collection 查询：<10ms（1000 点）
- 全 Collection 查询：<100ms（100 论文）
