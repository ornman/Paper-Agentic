## 架构审查发现的问题

**￥问题 #7：配置类过大，维护成本高￥**

**维度**: 代码与实现
**严重性**: P2
**位置**: `core/config.py:15-367`

**问题描述**:
`Settings` 类有 367 行，包含大量配置和兼容逻辑，维护成本高。

**代码证据**:
```python
# core/config.py:15-367
class Settings(BaseSettings):
    # 17 个环境区分配置
    # 9 个旧字段兼容 @property
    # 1 个复杂校验器
    # 总计 367 行
```

**潜在影响**:
- 可读性差: 难以快速找到需要的配置
- 维护成本高: 添加新配置需要理解整段代码
- 兼容代码堆积: 旧字段兼容代码占 10%

**建议方向**:
1. 按功能拆分: `LLMConfig`, `EmbeddingConfig`, `QdrantConfig`, `IngestionConfig`
2. 使用继承或组合: `class Settings(LLMConfig, EmbeddingConfig, ...)`
3. 定期清理旧字段兼容代码（建议在 v2.0 移除）

---

**￥问题 #8：环境差异验证不足￥**

**维度**: 可观测性与运维
**严重性**: P2
**位置**: `core/config.py:256-274`

**问题描述**:
虽然支持 `development` / `staging` / `production` 环境区分，但缺乏环境差异验证。

**代码证据**:
```python
# core/config.py:256-274
@property
def is_development(self) -> bool:
    return self.environment in ("development", "dev")

@property
def is_production(self) -> bool:
    return self.environment in ("production", "prod")

@property
def is_staging(self) -> bool:
    return self.environment in ("staging", "stage")

@property
def effective_debug(self) -> bool:
    return self.debug or self.is_development
    # ⚠️ 开发环境强制开启 debug，可能不是预期行为
```

**潜在影响**:
- 🔴 调试信息泄露: 开发环境强制开启 debug，可能导致敏感信息泄露
- 🔴 配置错误: 生产环境误配置为 `development` 会强制开启 debug

**建议方向**:
1. 移除 `effective_debug` 的自动开启逻辑
2. 添加环境差异校验: 生产环境必须显式配置 `debug=false`
3. 启动时打印当前环境配置
