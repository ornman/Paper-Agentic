**￥问题￥1: 缺乏依赖注入容器**
- **位置**: 所有routes文件

- **建议**: 使用`fastapi.Depends`或`dependency_injector`

### 3.3 流式响应协议（SSE）

**【代码事实】事件格式** (`routes/query.py:62-77`):
```python
async def event_generator():
    async for event in service.ask_stream_with_rag(...):
        event_type = event["type"]  # "chunk" | "sources" | "rewrite" | "done" | "error"
        event_data = json.dumps(event["data"], ensure_ascii=False)
        yield f"event: {event_type}\ndata: {event_data}\n\n"
```

**事件类型定义** (`models/query.py:61-81`):

| 事件类型 | 数据结构 | 说明 |
|---------|---------|------|
| `rewrite` | `{"original": str, "rewritten": [str]}` | Query改写结果 |
| `sources` | `{"sources": [SourceItem]}` | 检索到的引用来源 |
| `chunk` | `{"content": str, "index": int}` | LLM生成的文本片段 |
| `done` | `{"total_tokens": int}` | 生成完成 |
| `error` | `{"code": int, "message": str}` | 错误信息 |

**SourceItem结构** (`models/query.py:52-59`):
```python
class SourceItem(BaseModel):
```

## 五、安全边界标注

### 5.1 认证/授权

**￥问题￥2: 无认证机制**
- **位置**: 所有API端点
- **问题**: 无API Key验证、无用户隔离
- **影响**: 任何人可访问/删除/导入数据
- **建议**:
  1. 添加`FastAPI Security`中间件
  2. 在`SessionORM`中添加`user_id`字段
  3. 在Repository层强制过滤用户数据

### 5.2 输入校验

**【代码事实】Pydantic自动校验**:
- ✅ 类型校验: 所有请求模型使用Pydantic
- ✅ 长度校验: `session_id: str` (无长度限制)
- ❌ **内容校验缺失**: `query: str` 无最大长度限制

**￥问题￥3: 缺少输入长度限制**
- **位置**: `models/query.py:39`
- **问题**: `query`字段无最大长度限制
- **风险**: 恶意用户提交10MB字符串导致OOM
- **建议**:
  ```python
  query: str = Field(max_length=10000)
  ```

### 5.3 输出脱敏

**【代码事实】敏感信息暴露**:
- ✅ API Key不返回给前端（存储在后端）
- ❌ **错误信息泄露**: `HTTPException`包含原始异常 (`routes/library.py:77`)
- ❌ **文件路径泄露**: `document_id`可能包含路径信息

**￥问题￥4: 错误信息过于详细**
- **位置**: `main.py:34-43`
- **问题**: 全局异常处理器返回`str(exc)`
- **风险**: 暴露内部实现细节（如文件路径、SQL语句）
- **建议**:
  ```python
  # 生产环境不返回详细错误
  message = "内部错误" if settings.is_production else str(exc)
  ```

### 5.4 CORS配置

**【代码事实】当前CORS策略** (`main.py:21-27`):
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],      # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**￥问题￥5: CORS过于宽松**
- **位置**: `main.py:24`
- **问题**: `allow_origins=["*"]` 允许任何域名访问
- **风险**: CSRF攻击、数据泄露
- **建议**:
  ```python
  allow_origins=[
      "http://localhost:5173",  # 开发环境
      "https://wps-plugin.cn",  # 生产环境
  ]
  ```

## 六、接口版本控制

### 6.1 当前版本策略

**【代码事实】URL路径版本**:
- 路径前缀: `/api/v1/`
- 无版本协商机制
- 无向后兼容保证

**￥问题￥6: 缺少版本迁移策略**
- **影响**: 修改字段会导致旧客户端崩溃
- **建议**:
  1. 维护`v1`和`v2`两套路由
  2. 使用`FastAPI`的APIRouter版本控制
  3. 在`ApiResponse`中添加`api_version`字段

### 6.2 弃用标记

**【代码事实】无弃用标记**:
- 所有端点均为v1
- 无`Deprecated`注解
- 无 Sunset HTTP头

**建议**: 在字段注释中标记`@deprecated`：
```python
class AskRequest(BaseModel):
    query: str
    # @deprecated: 使用context代替
    context_old: Optional[str] = None
```
