## 核心发现

### ✅ 架构优点

1. **模块化清晰**: 按领域拆分（ingestion/library/qa/retrieval），依赖方向单向
2. **抽象层完整**: VLM/LLM/Embedding/Rerank均有抽象接口，可替换实现
3. **分布式设计**: 每篇论文独立Qdrant collection，删除简单
4. **降级保护**: Query改写、RAG检索失败均有降级方案
5. **配置集中**: 所有配置在`core/config.py`，支持环境隔离

### ⚠️ 关键问题

**P0级别**（必须修复）:
1. **导入失败残留**: Qdrant部分写入后失败，不清理数据
2. **无启动校验**: API Key为空不会导致启动失败
3. **SQLite无并发保护**: 高并发时可能锁死
4. **静默错误无监控**: `print()`在生产环境丢失

**P1级别**（强烈建议）:
1. **Embedding串行**: 批量API未利用，导入慢
2. **无熔断机制**: 服务持续失败时继续重试
3. **CORS过于宽松**: `allow_origins=["*"]`存在安全风险
4. **废弃代码未清理**: `services_archive/`目录占用1500行

**P2级别**（优化建议）:
1. **大文件警告**: `mineru_client.py` 518行接近阈值
2. **无代理配置**: 无法通过代理访问API
3. **日志级别固定**: 生产环境无法调整

## 技术栈总结

| 层级 | 技术选型 | 版本 | 用途 |
|------|---------|------|------|
| **Web框架** | FastAPI | - | API服务器 |
| **向量库** | Qdrant | 本地存储 | 分布式向量检索 |
| **关系数据库** | SQLite | 3.x | 会话/元数据 |
| **关键词检索** | rank_bm25 + jieba | - | BM25索引 |
| **LLM** | DeepSeek | - | 主问答模型 |
| **Embedding** | 硅基流动 | Qwen3-8B | 文本向量化（1536维） |
| **Rerank** | 硅基流动 | Qwen3-8B | 结果重排序 |
| **VLM** | Kimi Coding API | - | 图片描述 |
| **PDF解析** | MinerU API | - | 远程解析服务 |
