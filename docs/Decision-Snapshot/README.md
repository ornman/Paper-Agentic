## 核心发现

### 架构优势

1. **分布式向量存储** - 每篇论文独立 Collection，清理简单，扩展性好
2. **抽象接口设计** - VLM/LLM/Embedding/Rerank 可替换，符合开闭原则
3. **前端大幅简化** - 从66文件简化到23文件，代码重复减少
4. **内存优化配置** - Qdrant m=12, ef_construct=64，内存占用降低30%

### 架构问题

1. **￥测试覆盖率为0￥** - 所有测试文件被删除，无回归测试
2. **￥前端状态机过于复杂￥** - conversation.ts 有472行，难以维护
3. **￥VLM 错误处理不完善￥** - 失败后永久缓存 `"description unavailable"`，无重试
4. **￥配置文件膨胀￥** - config.py 有367行，50+配置项
