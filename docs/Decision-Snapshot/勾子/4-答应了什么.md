## SSE 流式契约（重点）

`/api/v1/query/ask` 返回 `text/event-stream`，事件类型：

| 事件类型 | data 结构 | 说明 |
|----------|-----------|------|
| `chunk` | `{"content": str, "index": int}` | 文本增量块 |
| `sources` | `{"sources": [SourceItem]}` | 引用来源列表 |
| `rewrite` | `{"original": str, "rewritten": [str]}` | Query 改写信息 |
| `done` | `{"total_tokens": int}` | 流式完成 |
| `error` | `{"code": int, "message": str}` | 错误事件 |

前端 SSE 客户端实际处理的事件：`chunk`、`sources`、`done`、`error`。**前端未处理 `rewrite` 事件。**

## 契约风险

### 红色高风险

1. **前后端 API 端点不对齐**
   - 前端 `GET /api/v1/library/documents` 在后端不存在，会 404
   - 前端 `POST /api/v1/library/import` 发送 JSON body，后端同路径期望 UploadFile（multipart）
   - 影响：知识库管理功能完全无法工作

2. **query.py 缺少 import**
   - 路由文件 `query.py` 使用了 `asyncio.CancelledError` 和 `logger`，但文件头部没有 `import asyncio` 和 `import logging`
   - 影响：运行时 `NameError`，SSE 流的错误捕获分支会直接崩溃

3. **零测试覆盖**
   - 22 个 API 端点、6 个服务层、4 个数据处理模块全部没有自动化测试
   - 影响：任何重构都是盲改，回归问题无法自动发现

4. **health 路由硬编码 vector_db = "ok"**
   - 注释说 "ChromaDB 本地"，但实际已迁移到 Qdrant
   - 检查逻辑始终返回 "ok"，不会发现 Qdrant 实际不可用的情况

### 黄色中风险

5. **前端 SSE 不处理 rewrite 事件**
   - 后端 QAService 会发出 `type: "rewrite"` 事件，前端 SSE 客户端只处理 `chunk/sources/done/error`
   - 影响：Query 改写信息静默丢失，用户看不到改写过程

6. **CORS 全开放**
   - `allow_origins=["*"]`，开发阶段可以接受，但部署时必须收紧

7. **library 路由混合使用两套存储**
   - `/library/papers` 直接操作 `QdrantStore`，不经过 `LibraryService`
   - `/library/import` 和 `/library/import-path` 走 `LibraryService` + SQLite `DocumentORM`
   - 两套数据源之间没有关联：Qdrant 的 papers 和 SQLite 的 documents 是独立体系

8. **会话服务路由直接调用 sqlite_repo**
   - `session.py` 路由中 `create_session`/`list_sessions` 直接调用 `sqlite_repo` 函数
   - 但 `get_history`/`clear_history` 走 `SessionService`
   - 混合调用模式，不一致

9. **RetrieveResponse 定义了 rewritten_query 但实际未返回**
   - `RetrieveResponse` 模型有 `rewritten_query` 字段
   - 但 `retrieve_endpoint` 路由只返回 `{query, results, total}`，没有 `rewritten_query`
   - Pydantic 响应不会校验这个缺失（因为 response_model 未声明）

10. **前端 LibraryImportPayload 硬编码 index_mode='brute'**
    - 后端支持 `distributed` 和 `brute`，但前端锁死为 `brute`
    - 如果后端默认切换到 `distributed`，前端仍然发 `brute`
