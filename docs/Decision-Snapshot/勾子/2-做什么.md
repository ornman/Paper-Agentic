## 核心流程

### 1. PDF 导入流程

**入口**: `POST /api/v1/library/import` 或 `POST /api/v1/library/import-path`

**完整调用链**（6 阶段状态机）:

```
pending → parsing → cleaning → indexing → completed
                                           ↘ failed（任何阶段都可能回退）
```

1. **登记**: `LibraryService.import_pdf()` (`app/modules/library/service.py:63`)
   - 路径安全校验：拒绝 URL/UNC/符号链接/路径遍历/非 .pdf (`service.py:138-183`)
   - 创建 DocumentRecord（status=pending）写入 SQLite (`service.py:84-89`)
   - 调用 `IngestionService.ingest_document()` (`service.py:90`)

2. **MinerU 解析** (status: parsing → cleaning): `IngestionService.ingest_document()` (`app/modules/ingestion/service.py:50`)
   - 调用 `MineruClient.run_pdf_task(file_path)` (`ingestion/service.py:60`)
   - MineruClient 执行：申请上传链接 → 上传 PDF → 轮询任务状态 → 下载 ZIP 结果 → 解压提取 layout.json (`ingestion/mineru_client.py:48-67`)
   - 轮询策略：自适应间隔（初始 2s，最小 1s，最大 10s，指数退避 + 抖动）（`mineru_client.py:136-233`）
   - 校验 payload 最小结构（必须包含 `pages` 数组）（`ingestion/service.py:308-329`)

3. **清洗** (status: cleaning): `clean_mineru_payload()` (`app/modules/ingestion/cleaning.py:38`)
   - 从 payload 提取原始块（支持文本和图片）（`cleaning.py:111-136`）
   - 过滤规则：页眉页脚（跨页重复短行）、页码（正则匹配 4 种格式）、短噪音块（≤2 字符）、重复文本（`cleaning.py:273-298`）
   - 图片块：调用 KimiVLMClient 生成图片描述（`cleaning.py:301-395`）
   - 返回 `CleanedDocument`（frozen Pydantic model）（`ingestion/dto.py:34-46`)

4. **VLM 图片描述**: `_describe_images()` (`ingestion/service.py:177-197`)
   - 先构建基础 chunks，过滤出有图片路径的 chunks
   - 调用 `describe_chunks_async()`（`processing/describer.py:45`）
   - 并发控制：asyncio.Semaphore（默认 10 并发）（`describer.py:79`）
   - 缓存机制：`image_descriptions.json`（按图片路径去重）（`describer.py:24-43`）
   - 每个 chunk 的 content 被替换为 `[Image]\nDescription: {description}\nOriginal: {content}`（`describer.py:130`）

5. **混合切分** + **Embedding** + **Qdrant 存储** (status: indexing):
   - 切分：`chunk_by_semantic_units()` (`processing/chunker.py:205`)
     - 两个块合计 ≤ 32k token → 打包合并
     - 超过 32k，单个不超 → 分开，不重叠
     - 单个超过 32k → 按 24k 切分，首尾重叠接近 32k
     - Token 估算：中文*1.5 + 非中文*0.75（`chunker.py:28-37`）
   - Embedding：`EmbeddingClient.embed()` (`app/clients/embedding_client.py:64`)
     - 硅基流动 Qwen3-Embedding-8B，1536 维（`config.py:10-11` 固定契约）
     - 批量处理：按 batch_size 分批，asyncio.gather 并发（`embedding_client.py:86-98`）
     - 连接复用 + 速率限制 + 重试 3 次（`embedding_client.py:40-51`）
   - Qdrant 存储：`QdrantStore.add_chunks()` (`app/stores/qdrant_store.py:113`)
     - 分布式架构：每篇论文一个独立 collection（命名规则 `paper_{id}`）
     - 幂等写入：先删除同 paper_id 旧点，再批量 upsert（`qdrant_store.py:141-172`）
     - HNSW 参数：m=12, ef_construct=64, full_scan_threshold=5000（`config.py:222-239`）

6. **完成** (status: completed): 返回 document_id、title、status

**失败处理**:
- 任何阶段抛出 IngestionError 或未预期异常 → `_mark_failed()` 将记录推进到 failed 状态，写入 error_stage 和 error_message（`ingestion/service.py:339-347`）
- 恢复：`POST /api/v1/library/resume/{document_id}` → 检查 status==failed → 重置为 pending → 重新走完整导入链路（`library/service.py:92-128`）

### 2. RAG 问答流程

**入口**: `POST /api/v1/query/ask` → SSE 流式响应

**调用链**:

1. **路由层** (`app/api/v1/routes/query.py:23`)
   - 接收 `AskRequest`（session_id, query, context, resource_types, selected_papers）
   - 生成唯一 request_id（UUID4）（`query.py:69`）
   - 返回 `StreamingResponse(media_type="text/event-stream")`（`query.py:104-112`）

2. **QAService.ask_stream_with_rag()** (`app/modules/qa/service.py:79`)
   - SSE 事件类型：`rewrite`, `sources`, `chunk`, `done`, `error`

3. **Query 改写** (`app/services/query_rewrite_service.py:97`)
   - 4 种场景模板（`query_rewrite_service.py:22-88`）:
     - 场景 1：仅有 query → 直接改写为检索语句
     - 场景 2：仅有 written_content → 推断续写意图
     - 场景 3：written + selected → 70% selected + 30% written
     - 场景 4：written + selected + prompt → 50% prompt + 30% selected + 20% written
   - 调用 LLM（OpenAI 兼容协议）生成 1-3 个检索查询（`query_rewrite_service.py:116-128`）
   - 失败降级：改写失败时回退到原始 query（`query_rewrite_service.py:126-128`）
   - 改写结果不同时 yield `rewrite` 事件（`qa/service.py:129-136`）

4. **RAG 检索** (`app/modules/retrieval/service.py:36`)
   - **向量检索**：EmbeddingClient.embed_single(query) → QdrantStore.search_all(vector, limit=top_k*2)（`retrieval/service.py:65-69`）
   - **Rerank**：RerankClient.rerank(query, documents, top_k) → 按相关性重排序（`retrieval/service.py:72-87`）
   - **降级**：Embedding 失败时降级到 BM25 检索，返回 degraded=True 标记（`retrieval/service.py:98-122`）
   - 检索结果 yield `sources` 事件（`qa/service.py:153-157`）

5. **LLM 流式生成** (`qa/service.py:159-176`)
   - 构建消息列表：system prompt + 最近 20 条历史 + 当前消息（含 RAG 上下文）（`qa/service.py:29-62`）
   - RAG Prompt 模板：引用格式 `[来源:论文名 页码X]`，要求不编造内容（`qa/service.py:16-25`）
   - 调用 KimiLLMClient.chat_stream() 或 OpenAI 兼容 LLM（`qa/service.py:167`）
   - 带 retry_async 重试 3 次（`qa/service.py:167`）
   - 每个 chunk yield `chunk` 事件（`qa/service.py:169`）
   - 失败降级：yield `error` 事件（`qa/service.py:173-174`）

6. **保存消息** (`qa/service.py:178-185`)
   - 保存 user 消息和 assistant 消息到 SQLite
   - assistant 消息携带 sources JSON
   - yield `done` 事件，包含 total_tokens（`qa/service.py:187`）

### 3. 前端交互流程

**用户输入路径**（`frontend/src/components/layout/BottomActionBar.vue:36-53`）:

1. 用户在 textarea 输入 prompt → `conversationStore.sendPrompt(payload)`
2. payload 包含：session_id（前端生成的 UUID）、query、context（written_content 来自 WPS 轮询、selected_text 来自选区监听、prompt 来自输入框）
3. sendPrompt → `postAskInspirationStream()` → `fetch('/api/v1/query/ask')` with SSE
4. SSE 事件处理（`frontend/src/services/sse-client.ts:287-395`）:
   - `chunk` → `appendAssistantChunk()` → 增量更新 assistant message content
   - `sources` → `attachAssistantSources()` → 暂存或挂载来源卡片
   - `done` → `finishResponse()`
   - `error` → `markError()`
5. 消息状态机（`frontend/src/stores/conversation.ts:8`）: idle → requesting → streaming → done | error

**WPS 宿主交互**:
- AppShell mounted 时启动 `hostAdapter.startPolling(hostStore)`（5 秒周期）（`frontend/src/app/AppShell.vue:162`）
- 每次轮询读取 WPS ActiveDocument.Name 和 Content.Text → 更新 hostStore（`frontend/src/services/wps-host.ts:263-273`）
- 读取异常时保留上次快照，标记为 stale（`wps-host.ts:227-229`）
- 选区监听：350ms 轮询 WPS Selection.Text → 写入 conversationStore.selectionContext（`frontend/src/services/wps-selection.ts:111-113`）

## 数据流

### PDF 导入数据流

```
用户上传 PDF
    │
    ▼
routes/library.py (import_pdf / import_pdf_by_path)
    │  校验文件路径、创建上传目录
    ▼
LibraryService.import_pdf()          ← 创建 DocumentRecord(pending)
    │
    ▼
IngestionService.ingest_document()   ← 状态迁移: pending → parsing
    │
    ├──► MineruClient.run_pdf_task()
    │       │  submit_task(): 申请上传链接 → 上传 PDF
    │       │  poll_task(): 轮询直到 success/failed/timeout
    │       │  fetch_result_json(): 下载 ZIP → 解压 → 读取 layout.json
    │       ▼
    │   返回 {pages: [{page: int, blocks: [...]}]}
    │
    │   ← 状态迁移: parsing → cleaning
    │
    ├──► clean_mineru_payload()
    │       │  提取原始块 → 过滤噪音 → 识别图片块
    │       │  图片块 → KimiVLMClient.describe_image() → 生成描述
    │       ▼
    │   返回 CleanedDocument(blocks: list[CleanedBlock])
    │
    ├──► _describe_images()           ← 跳过条件: 无图片块
    │       │  并发调用 KimiVLMClient (Semaphore=10)
    │       │  缓存到 image_descriptions.json
    │       ▼
    │   返回 list[Chunk] (图片 content 被替换为 VLM 描述)
    │
    ├──► _chunk_chunks()
    │       │  SemanticChunk → chunk_by_semantic_units()
    │       │  混合策略: 打包 / 分开 / 切分+重叠
    │       ▼
    │   返回 list[dict] (text_chunks)
    │
    ├──► _embed_chunks()
    │       │  EmbeddingClient.embed(texts)
    │       │  硅基流动 API, batch_size=32, 并发=5
    │       ▼
    │   返回 list[list[float]] (1536 维向量)
    │
    │   ← 状态迁移: cleaning → indexing (Embedding 成功后才迁移)
    │
    └──► QdrantStore.add_chunks()
            │  创建/复用 collection (paper_{id})
            │  删除旧点 → 批量 upsert 新点
            ▼
        Qdrant 本地存储
```

持久化位置：
- SQLite (`data/app.db`): DocumentRecord 状态和元数据
- Qdrant (`data/qdrant/`): 向量和 payload
- 文件系统 (`data/papers/{task_id}/`): MinerU 解析结果（layout.json、images/）
- BM25 (`data/bm25_index.json`): 关键词索引（pickle）

```
前端 textarea 输入 prompt
    │
    ▼
BottomActionBar.handlePrimaryAction()
    │  构建 payload: {session_id, query, context: {written_content, selected_text, prompt}}
    ▼
conversationStore.sendPrompt()
    │  状态: idle → requesting
    │  追加 user action message
    ▼

    │
    ▼

    │
    ├──► QueryRewriteService.rewrite(query, context)
    │       │  选择场景模板 → LLM 改写 → 解析为 1-3 个查询
    │       ▼
    │   yield {type: "rewrite", data: {original, rewritten}}
    │
    ├──► RetrievalService.retrieve(rewritten_query)
    │       │
    │       ├──► EmbeddingClient.embed_single(query)     ← 失败则降级到 BM25
    │       ├──► QdrantStore.search_all(vector, limit)   ← 遍历所有 paper_* collection
    │       ├──► RerankClient.rerank(query, docs, top_k) ← 按 relevance_score 重排
    │       ▼
    │   yield {type: "sources", data: {sources: [...]}}
    │
    ├──► _build_messages(session_id, query, sources)
    │       │  system prompt + 最近 20 条历史 + RAG 上下文消息
    │       ▼
    │   消息列表
    │
    ├──► KimiLLMClient.chat_stream(messages)             ← retry_async(max_retries=3)
    │       ▼
    │   yield {type: "chunk", data: {content, index}}    ← 逐块返回
    │
    ├──► sqlite_repo.add_message("user", query)
    │   sqlite_repo.add_message("assistant", content, sources)
    │
    └──► yield {type: "done", data: {total_tokens}}
```

- `done` → conversationStore.finishResponse()
- `error` → conversationStore.markError()

## 文档状态机

状态定义（`app/modules/library/models.py:20-28`）：

```
pending ──→ parsing ──→ cleaning ──→ indexing ──→ completed
   │           │           │            │              │
   ↓           ↓           ↓            ↓              ↓
 failed ←─────┘ ←─────────┘ ←──────────┘              │
   │                                                    │
   ↓                                                    │
 pending (resume)                                       │
                                                        ↓
deleting ──→ deleted    ← 任何状态都可以进入 deleting ←──┘
```

合法迁移表（`models.py:62-71`）：
- pending → {parsing, failed, deleting}
- parsing → {cleaning, failed, deleting}
- cleaning → {indexing, failed, deleting}
- indexing → {completed, failed, deleting}
- completed → {indexing, deleting}
- failed → {pending, deleting}
- deleting → {deleted, failed}
- deleted → {} (终态)

关键设计决策：
- `completed → indexing` 已开放但 reindex_document() 当前拒绝 completed 状态重建索引（占位实现，`service.py:235-236`）
- 软删除：通过状态机推进 deleting → deleted，不物理删除数据库行（`service.py:202-210`）
- DocumentRecord 使用 frozen=True Pydantic model，状态迁移通过 `transition()` 返回新对象（不可变模式，`models.py:136-145`）

## 前端消息状态机

```
idle ──→ requesting ──→ streaming ──→ done
  │           │              │
  │           │              ↓
  │           │           error
  │           │
  └───────────┘  (reset)
```

状态定义（`frontend/src/stores/conversation.ts:8`）：
- `idle`: 初始状态或 reset 后
- `requesting`: 用户发起请求，等待 SSE 响应
- `streaming`: 收到第一个 chunk 或 sources
- `done`: 收到 done 事件或流自然结束
- `error`: 收到 error 事件或网络异常

关键约束：
- assistant message 只在首个 chunk 到达时创建（避免空白气泡）（`conversation.ts:228-243`）
- sources 可能先于 chunk 到达 → 暂存到 pendingAssistantSources → chunk 到达时一并挂载（`conversation.ts:137-139`）
- reset() 生成新 session_id（`conversation.ts:476`），隔离前后会话
