# 7-经历了什么 — 旅程图

## 需人工补充

⚠️ 代码无法推导用户旅程，需要人工补充以下内容：

### 典型用户场景
- **场景 1**：研究生撰写论文，需要引用管理 + 文献检索
  - 用户上传 PDF 论文 → 系统解析入库 → 用户提问 → 系统基于论文内容回答
- **场景 2**：论文写作过程中，需要查证某个概念或数据
  - 用户选中文本 → 右键查询 → 系统从已导入文献中检索相关段落
- **场景 3**（🔮 未来）：多模态资源管理
  - 用户导入视频/笔记 → 系统理解多类型内容 → 跨模态检索

### 用户流程
1. 打开 WPS → 侧边栏显示论文助手
2. 上传/导入论文 PDF → 等待解析完成
3. 在对话区提问 → 系统基于 RAG 生成回答（流式显示）
4. 查看引用来源 → 定位到论文原文位置
5. （🔮 未来）选择特定文献进行定向检索

### 从代码可推导的用户触点

| 触点 | 位置 | 说明 |
|------|------|------|
| WPS 侧边栏入口 | frontend/manifest.xml | WPS 插件注册 |
| 论文导入 | POST /api/v1/library/import | 上传 PDF 文件 |
| 论文导入（路径） | POST /api/v1/library/import-path | 通过本地路径导入 |
| 断点续传 | POST /api/v1/library/resume/{doc_id} | 导入中断后恢复 |
| 论文列表 | GET /api/v1/library/papers | 查看已导入论文 |
| 删除论文 | DELETE /api/v1/library/papers/{paper_id} | 移除论文及索引 |
| 问答对话 | POST /api/v1/query/ask (SSE) | 核心交互——提问并获取回答 |
| 检索测试 | POST /api/v1/query/retrieve | 单独测试检索效果 |
| 会话管理 | CRUD /api/v1/session/ | 对话历史管理 |

### 痛点与需求（从代码推判）
- **痛点**: 导入流程耗时长（MinerU 远程解析 + 多步处理），用户需等待
  - **对应方案**: 断点续传接口已实现（`/resume/{doc_id}`）
- **痛点**: 导入进度不透明
  - **对应方案**: 文档状态机（6 阶段）可查询，但前端未对接进度展示
- **痛点**: 无法指定特定论文回答
  - **对应方案**: 🔮 接口已预留 `selected_papers` 参数
