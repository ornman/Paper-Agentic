## 配置单例模式

```python
@lru_cache
def get_settings() -> Settings:
    return Settings()
```

- `config.py:276-279` 使用 `lru_cache` 实现单例
- 首次调用解析 `.env` + 环境变量，后续调用返回缓存
- **风险**：运行时修改环境变量不会反映到已缓存的实例
￥这个有什么解决方法吗？￥
￥
【问题】：`lru_cache` 缓存了 `Settings` 实例，运行时修改环境变量不会反映到已缓存的实例。

**代码证据**：`config.py:276-279`
```python
@lru_cache
def get_settings() -> Settings:
    return Settings()
```

**影响**：
- 用户修改 `.env` 文件 → 需要重启服务才能生效
- 运行时修改环境变量 → 不会反映到 `get_settings()` 的返回值

**解决方案**：

### 方案 A：清除缓存（推荐）

**改动位置**：`config.py`
```python
def reload_settings() -> Settings:
    """清除缓存并重新加载配置"""
    get_settings.cache_clear()  # ← 清除 lru_cache
    return get_settings()
```

**使用场景**：
- 用户修改 `.env` 后，调用 `reload_settings()`
- 单元测试中，每个测试用例前重新加载配置

### 方案 B：环境变量监听（复杂）

**改动位置**：新增 `app/services/config_watcher.py`
```python
import asyncio
from watchfiles import awatch

async def watch_env_file():
    async for changes in awatch(".env"):
        reload_settings()
```

**缺点**：
- 增加依赖（`watchfiles`）
- WPS 插件环境可能不支持文件监听

### 方案 C：运行时配置覆盖（灵活）

**改动位置**：`config.py`
```python
class Settings(BaseSettings):
    # ... 现有字段 ...
    
    # 运行时覆盖（仅用于测试）
    _overrides: dict[str, Any] = {}
    
    def __getattribute__(self, name: str):
        if name in object.__getattribute__(self, "_overrides"):
            return object.__getattribute__(self, "_overrides")[name]
        return object.__getattribute__(self, name)
    
    @classmethod
    def override(cls, **kwargs):
        """临时覆盖配置（仅用于测试）"""
        instance = cls()
        instance._overrides = kwargs
        return instance
```

**使用场景**：
```python
# 测试中
settings = Settings.override(llm_temperature=0.0)
```
