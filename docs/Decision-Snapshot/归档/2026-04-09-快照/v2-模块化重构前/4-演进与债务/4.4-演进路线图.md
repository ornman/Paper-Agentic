# 4.4 演进路线图

> 生成时间: 2026-04-08
> 分析方法: 🔮 未来扩展标记汇总 + 架构瓶颈识别

## 能力阶梯

### 能做（已实现）

- PDF 导入：MinerU 解析 → 清洗 → VLM 描述 → 切分 → Embedding → Qdrant
- 分布式检索：多 Collection 并行搜索 + RRF 融合 + Rerank
- RAG 问答：Query 改写（4 场景） + 检索 + 流式输出
- 会话管理：创建/列表/删除/历史
- 配置管理：环境变量 + 模型契约锁定 + 脱敏查询
- 健康检查：组件可用性检测

### 勉强能做（有限制）

- 批量导入：串行执行，无进度追踪，无断点续传
- 删除论文：同步 `shutil.rmtree` 阻塞事件循环
- 错误反馈：全局 500 + 错误信息字符串，无结构化错误码
- 并发检索：无 Semaphore 限制，论文多了会打爆 Qdrant

### 不能做（未实现 / 🔮 标记）

- **用户自选文献**：接口已预留（`selected_papers` 参数），Collection 过滤逻辑有骨架代码，但未实现
- **用户自选数据类型**：接口已预留（`resource_types` 参数），无实现
- **导入进度查询**：无状态追踪，前端无法知道导入到哪了
- **导入断点续传**：中断后只能删除重来
- **任务队列**：MinerU 同步阻塞，无异步任务机制
- **用户认证**：所有 API 无需认证
- **多模态扩展**：只支持 PDF，视频/文档/笔记未支持

## 扩展点实现优先级

基于依赖关系排序：

```
优先级 1 (基础能力补全):
  ├── 用户自选文献 (selected_papers)     ← 提升检索准确性
  │   └── 依赖: Collection 过滤逻辑 (retrieval_service.py:239)
  │
  ├── 导入进度追踪 + 异步任务            ← 解决 MinerU 阻塞
  │   └── 依赖: 任务状态持久化 + 前端轮询接口
  │
  └── 检索并发限制 (Semaphore)           ← 防止 Qdrant 过载
      └── 依赖: retrieval_service.py 改造

优先级 2 (质量提升):
  ├── VLM 描述 Schema Validation         ← 防止幻觉数据进入向量库
  │   └── 依赖: describer.py 输出校验
  │
  ├── 结构化错误体系                     ← 替代全局 500
  │   └── 依赖: errors.py → 路由层映射
  │
  └── 新旧架构统一                       ← 消除 modules/ 残留
      └── 依赖: 路由切换 + 旧代码删除

优先级 3 (产品完善):
  ├── 用户自选数据类型 (resource_types)
  ├── 用户认证
  ├── 多模态支持
  └── 知识图谱集成
```

## 架构瓶颈清单（不改会卡住的点）

| 瓶颈 | 当前状态 | 卡住条件 | 影响 |
|------|---------|---------|------|
| **MinerU 同步阻塞** | 全同步实现 | 论文 > 10 篇时导入体验极差 | 导入期间所有请求被卡 |
| **无任务队列** | 无 | 导入 + 问答并发 | 导入阻塞问答 |
| **无限并发检索** | 无 Semaphore | 论文 > 500 篇 | Qdrant 过载 |
| **无导入状态追踪** | 无 | 任何导入操作 | 用户不知道导入是否完成 |
| **VLM 无输出校验** | 无 | 任何图片描述 | 幻觉数据静默入库 |
| **新旧架构并存** | modules/ 残留 | 任何维护工作 | 认知负载增加，改一处忘一处 |
| **httpx 短生命周期** | 每次新建 | 高并发场景 | 连接无法复用，性能浪费 |
| **DeepSeek 默认超时 5s** | 未配置 | 网络抖动 | 问答频繁超时 |
