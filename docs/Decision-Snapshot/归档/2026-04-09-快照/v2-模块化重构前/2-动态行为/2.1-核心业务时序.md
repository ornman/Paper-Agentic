## PDF 导入端到端时序

```mermaid
sequenceDiagram
    participant User as 用户
    participant API as library.py
    participant WF as IngestWorkflow
    participant MinerU as MinerUClient
    participant Clean as cleaner.py
    participant VLM as describer.py
    participant Kimi as Kimi API
    participant Chunk as chunker.py
    participant Emb as EmbeddingService
    participant SF as SiliconFlow
    participant Q as QdrantStore

    User->>API: POST /library/import (PDF)
    API->>API: 保存文件到 uploads/ (同步写入)
    API->>WF: ingest_pdf(pdf_path)

    Note over WF,MinerU: 阶段1: PDF 解析 (同步阻塞，最长 300s)
    WF->>MinerU: parse_pdf(pdf_path)
    MinerU->>MinerU: submit_pdf() [HTTP POST, timeout=120s]
    loop 轮询 (每5s)
        MinerU->>MinerU: time.sleep(5) [阻塞事件循环]
        MinerU->>MinerU: 查询状态 [HTTP GET, timeout=30s]
    end
    MinerU->>MinerU: 下载结果 [HTTP GET, timeout=120s]
    MinerU-->>WF: 解析结果 (JSON + 图片)

    Note over WF,Clean: 阶段2: 数据清洗
    WF->>Clean: clean_paper(data)
    Clean-->>WF: Chunk[]

    Note over WF,Kimi: 阶段3: VLM 图片描述 (并发10, timeout=120s)
    WF->>VLM: describe_chunks(chunks)
    VLM->>VLM: Semaphore(10)
    par 并发描述 (最多10个)
        VLM->>Kimi: describe_image() [HTTP, timeout=120s]
        Kimi-->>VLM: 图片描述文本
    end
    VLM-->>WF: Chunk[] + 图片描述

    Note over WF,Chunk: 阶段4: 混合语义切分
    WF->>Chunk: chunk_by_semantic_units(chunks)
    Chunk-->>WF: SemanticChunk[]

    Note over WF,SF: 阶段5: Embedding (串行, timeout=60s)
    WF->>Emb: generate_embeddings(chunks)
    loop 每个 batch
        Emb->>SF: embed(texts) [HTTP, timeout=60s]
        SF-->>Emb: vectors
    end
    Emb-->>WF: vectors

    Note over WF,Q: 阶段6: Qdrant 存储
    WF->>Q: store_chunks(paper_id, chunks, vectors)
    Q-->>WF: 存储完成

    WF-->>API: result
    API-->>User: {"code": 0, "data": result, "message": "导入成功"}

    alt 异常: 任意阶段失败
        WF--xAPI: Exception
        API-->>User: {"code": 9001, "message": "内部错误：{具体信息}"}
    end
```

## RAG 问答端到端时序

```mermaid
sequenceDiagram
    participant User as 用户
    participant API as query.py
    participant QA as qa_service_rag
    participant RW as query_rewrite
    participant DS as DeepSeek
    participant RS as RetrievalService
    participant Q as QdrantStore
    participant Emb as EmbeddingClient
    participant SF as SiliconFlow
    participant Rerank as RerankClient
    participant Kimi as Kimi LLM
    participant DB as SQLite

    User->>API: POST /query/ask (AskRequest)
    API->>API: 打印调试信息

    Note over QA,RW: 步骤1: Query 改写 (timeout: 未配置)
    QA->>RW: rewrite_query(query, context)
    RW->>DS: chat(messages) [OpenAI协议]
    DS-->>RW: 改写后的查询
    RW-->>QA: queries []

    alt 改写失败
        RW--xQA: Exception
        Note over QA: 降级: 使用原始查询 [query]
    end

    Note over QA,DB: 步骤2: 加载历史
    QA->>DB: get_messages(session_id)
    DB-->>QA: 历史消息

    Note over QA,Rerank: 步骤3: RAG 检索
    QA->>RS: retrieve(query, top_k)
    RS->>Emb: embed_single(query) [timeout=60s]
    Emb->>SF: API 调用
    SF-->>Emb: query_vector
    RS->>Q: list_papers()
    Q-->>RS: paper_ids []

    par 并行搜索所有 Collection (无并发限制)
        RS->>Q: search(paper_1, vector, top_k)
        RS->>Q: search(paper_2, vector, top_k)
        RS->>Q: search(paper_N, vector, top_k)
    end

    Note over RS: RRF 融合 (k=60)
    RS->>Rerank: rerank(query, docs, top_k) [timeout=60s]
    Rerank->>SF: API 调用
    SF-->>Rerank: ranked results
    RS-->>QA: sources []

    alt 检索失败
        RS--xQA: Exception
        Note over QA: 降级: sources=[], 回退纯问答
    end

    Note over QA,Kimi: 步骤4: 流式回答
    QA->>QA: _build_messages(history, query, sources)
    loop SSE 事件流
        QA->>Kimi: chat_stream(messages) [timeout=120s]
        Kimi-->>QA: 文本块 (假流式: 先完整请求再切块)
        QA->>User: SSE event: chunk
    end
    QA->>User: SSE event: sources
    QA->>User: SSE event: done

    QA->>DB: add_message(session_id, role, content, sources)

    alt LLM 失败
        QA--xAPI: Exception
        API->>User: SSE event: error {"code": 9001, "message": "..."}
    end
```

### 关键追问

**如果某个节点挂起 10 秒，上游是报错还是死等？**
￥自动重试，重试3次后熔断，给出报错原因￥
￥
【建议】：当前代码**没有自动重试机制**，所有外部调用都是"一次失败即报错"。

**代码证据**：`embedding_client.py:84`
```python
async with httpx.AsyncClient(timeout=60.0) as client:
    response = await client.post(...)
    response.raise_for_status()  # ← 失败直接抛异常，无重试
```

**影响**：
- 网络抖动 → 请求失败 → 用户看到错误
- API 限流（429）→ 请求失败 → 无重试，需手动重试
- 超时 → 请求失败 → 无重试，需手动重试

**建议实现**（🔮 未来扩展）：
1. **重试策略**：指数退避（1s → 2s → 4s），最多3次
2. **熔断机制**：连续失败 N 次后，暂停该服务调用一段时间（如 30s）
3. **错误分类**：
   - 可重试：网络错误、超时、429 限流
   - 不可重试：401 认证失败、404 资源不存在、400 参数错误
4. **用户提示**：重试失败后，给出明确错误原因（"API 限流，请稍后重试" vs "论文不存在"）

**优先级**：P1（影响用户体验）
￥
