## 最需要看的

- 🔴 **[5-哪里会断] MinerU 同步阻塞整个导入链路** — `MinerUTask.poll()` 用 `time.sleep()` 轮询，在 async 路由里卡死事件循环。大 PDF 5-10 分钟解析期间，整个服务不可用。
- 🔴 **[5-哪里会断] SSE 流无超时保护** — 前端 `sse-client.ts` 的 `fetch()` 和 `ReadableStream` 都没有设置超时，后端挂起时前端永远转圈。
- 🔴 **[4-答应了什么] 零测试覆盖** — 71 个 Python 文件，测试只覆盖 `test_ingestion_pipeline.py` 和 `test_index_modes.py` 两个集成测试文件。21 个 API 端点无任何自动化验证。
- 🔴 **[5-哪里会断] CORS 通配符** — `main.py:21` `allow_origins=["*"]`，任何域都可以调用 API。
- 🔴 **[5-哪里会断] Kimi API 伪装 User-Agent** — `config.py:146` 注释明确写了"必须伪装 User-Agent: claude-code"，依赖对方的反爬漏洞，随时可能静默失效。
- ⚠️ **[1-有什么] 双 LLM 客户端冗余** — `kimi_client.py` 和 `vlm_client.py` 都封装了 Kimi，QA 服务实际用的是 KimiLLMClient 而非 LangChain。
- ⚠️ **[3-为什么] LangChain 声明了但几乎没用** — `pyproject.toml` 有 `langchain>=1.2.13`，但代码中未发现实际 import。
- ⚠️ **[2-做什么] 检索链路只走向量+Rerank** — `rank-bm25` 在依赖里但 `RetrievalService` 只用了向量检索 + Rerank，BM25 检索未接入。

## 一句话定性

RAG 学术写作助手，Python 后端 (FastAPI + LangChain + DeepSeek + Qdrant + MinerU) + Vue 3 前端 + WPS 插件。MVP 阶段，导入链路可用但 QA 链路的 BM25 检索和 LangChain 未实际集成。测试覆盖近乎为零。核心风险集中在导入链路的同步阻塞和外部 API 的脆弱依赖。
