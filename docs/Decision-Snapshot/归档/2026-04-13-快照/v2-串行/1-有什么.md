# 1-有什么 — 结构全貌

**一句话**：前后端分离的 RAG 学术写作助手，后端 FastAPI + Vue 3 前端 + WPS 插件，本地部署。

## 物理结构

```
project/MVP/
├── backend/                  Python (FastAPI), 71 文件
│   ├── app/
│   │   ├── main.py           入口，53 行
│   │   ├── api/v1/routes/    5 个路由模块 (health/session/query/config/library)
│   │   ├── clients/          6 个外部 API 客户端 (MinerU/Kimi/VLM/Embedding/Rerank/LLM)
│   │   ├── core/             配置/错误/日志/重试
│   │   ├── models/           Pydantic 数据模型
│   │   ├── modules/          业务编排 (ingestion/library/qa/retrieval/session)
│   │   ├── processing/       数据处理 (chunker/cleaner/describer)
│   │   ├── repositories/     SQLite + BM25 持久化
│   │   ├── stores/           Qdrant 向量存储
│   │   └── services_archive/ 废弃的旧服务
│   └── tests/                2 个测试文件
├── frontend/                 Vue 3 + TypeScript, 25 文件
│   ├── src/
│   │   ├── app/AppShell.vue  主布局 (324 行)
│   │   ├── components/       8 个 Vue 组件 (对话/布局/浮层)
│   │   ├── services/         API 客户端 + SSE + WPS 集成
│   │   ├── stores/           Pinia 状态 (conversation/host/ui)
│   │   └── types/            TypeScript 类型定义
│   └── wps-plugin/           WPS 插件入口
└── docker-compose.yml        基础设施编排
```

## 技术栈（来自 pyproject.toml + package.json 事实）

**后端**：FastAPI | LangChain 1.2+ | OpenAI SDK | Qdrant Client | rank-bm25 | jieba | SQLAlchemy | Pydantic v2 | httpx | uvicorn

**外部 API**：DeepSeek (主 LLM) | SiliconFlow (Embedding + Rerank) | Kimi Coding (VLM + Chat) | MinerU (PDF 解析)

**前端**：Vue 3 | Pinia | Vite | TypeScript

**插件**：WPS JS SDK (侧边栏 + 圈选)

## 值得注意

- **`services_archive/`** — 4 个被废弃的旧服务仍然在代码库里，没有清理
- **双客户端问题** — `kimi_client.py`(381行) 和 `vlm_client.py`(184行) 都封装 Kimi API，职责重叠
- **Embedding/Rerank 模型锁死** — `config.py` 用 `PINNED_*` 常量和 `model_validator` 强制固定 Qwen3-Embedding-8B (1536维) 和 Qwen3-Reranker-8B，防止向量维度漂移
