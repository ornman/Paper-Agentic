# 5-哪里会断

**论文助手** 是一个 WPS 侧边栏插件：TypeScript 前端 + FastAPI 后端。核心链路是 PDF 导入（MinerU → 清洗 → 切分 → Embedding → Qdrant）+ RAG 问答（SSE 流式）。

## 🔴 CORS 全开

`app/main.py:22` 里 `allow_origins=["*"]`，注释说"开发阶段允许所有来源"。任何网页都能跨域调用后端 API。MVP 阶段产物留到了生产。

## 🔴 MinerU 同步阻塞 async 路由

`mineru_client.py:46` 的 `run_pdf_task` 是同步方法——轮询 MinerU 状态时用 `time.sleep` 阻塞线程（`self.sleep_fn(self.poll_interval)`）。但调用它的 `ingestion/service.py:60` 在 `async def ingest_document` 中直接调用。FastAPI 会把它扔进线程池，但默认线程池大小有限（通常 40），大量并发 PDF 导入会耗尽线程池，所有 async 路由卡死。

## 🔴 SSE 流无超时

`sse-client.ts:349` 的 `readSseStream` 没有 read timeout。`while (!terminalEventReached)` 循环中 `await reader.read()` 会无限等待。如果后端挂了但不关连接（半开连接），前端永远卡住，用户看到对话永远在"生成中"。没有 retry，没有 fallback。

## 🔴 API Key 默认空字符串

`config.py:39,74,149` — `deepseek_api_key`、`siliconflow_api_key`、`kimi_api_key` 默认都是 `""`。配置校验不阻止启动（`model_validator` 只锁了模型名和维度）。应用会正常起来，但任何实际请求都会在运行时炸掉。用户看到的错误信息取决于第一个被调用的 API 返回什么，不一定是"请配置 API Key"。

## ⚠️ 配置硬锁住了迁移路

`config.py:286-298` 的 `model_validator` 把 Embedding 模型（Qwen3-Embedding-8B）、维度（1536）、Rerank 模型锁死。一旦需要换模型（比如 Qwen3 → Qwen4），必须改代码重新部署。历史向量也会因为维度变化全部失效——这是设计意图，但意味着没有迁移策略。

## ⚠️ Kimi API 灰色通道

`kimi_client.py:23-29` — 使用 `https://api.kimi.com/coding/v1` 端点，伪装 `User-Agent: claude-code`。这是 Kimi 提供给 Claude Code 的免费 API 通道。如果 Kimi 关闭这个端点或检测到滥用，VLM 图片描述和备用 LLM 问答会全部失效。

## ⚠️ 重复的错误处理模板

`library.py` 里 6 个路由每个都有相同的 `try/except IngestionError/Exception` 模板（`import_pdf`、`import_pdf_by_path`、`resume_import`），复制粘贴了 3 遍。错误处理逻辑改一处就要改三处。

## 依赖链脆弱点

```
用户上传 PDF
  → library.py (FastAPI 路由)
    → LibraryService.import_pdf (async)
      → IngestionService.ingest_document (async)
        → MineruClient.run_pdf_task (同步阻塞！)
          → httpx 同步请求 MinerU API
            → 轮询等待（time.sleep）
              → 下载 ZIP → 解压 → 读 layout.json
        → cleaning.py (async 清洗)
        → chunker.py (切分)
        → embedding_client → SiliconFlow API
        → qdrant_store → 本地 Qdrant
```

任何一环挂了，整个导入失败。`resume_import` 支持断点续传，但只有 `IngestionError` 能恢复，`httpx` 网络错误不会记录进度。

## 最高风险

1. **MinerU 同步阻塞 async 路由** — 并发导入会卡死整个服务
2. **CORS 全开** — 任何网站可调用
3. **SSE 无超时** — 前端会永远卡住
4. **API Key 默认空** — 启动不报错，用时才炸
