## API 接口契约

### HTTP 端点（16 个）

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/v1/health` | GET | 健康检查 |
| `/api/v1/session/` | POST | 创建会话 |
| `/api/v1/session/` | GET | 列出会话（分页） |
| `/api/v1/session/{id}` | GET | 获取会话 |
| `/api/v1/session/{id}/title` | PUT | 更新标题 |
| `/api/v1/session/{id}` | DELETE | 删除会话 |
| `/api/v1/session/{id}/history` | GET | 获取历史 |
| `/api/v1/session/{id}/history` | DELETE | 清空历史 |
| `/api/v1/query/ask` | POST | 流式问答（SSE） |
| `/api/v1/query/retrieve` | POST | 纯检索 |
| `/api/v1/library/import` | POST | 上传导入 PDF |
| `/api/v1/library/import-path` | POST | 路径导入 PDF |
| `/api/v1/library/resume/{id}` | POST | 断点续传 |
| `/api/v1/library/papers` | GET | 列出论文 |
| `/api/v1/library/papers/{id}` | GET/DELETE | 论文详情/删除 |
| `/api/v1/library/status` | GET | 系统状态 |
| `/api/v1/config/` | GET | 获取配置 |
| `/api/v1/config/test` | POST | 测试连接 |
| `/api/v1/config/models` | GET | 推荐模型 |
| `/api/v1/config/export_logs` | POST | 导出日志 |
| `/api/v1/config/reload` | POST | 热加载配置 |

### 数据模型契约

- **AskRequest**: `session_id`, `query`, `context?` (written_content, selected_text, prompt), `resource_types?`, `selected_papers?`
- **RetrieveRequest**: `query`, `top_k?`, `resource_types?`, `selected_papers?`
- **ApiResponse**: `{code, data, message}` 统一信封
- **SSE 事件**: `chunk` / `sources` / `rewrite` / `done` / `error`

### 文档状态机

```
pending → parsing → cleaning → indexing → completed
                                          → failed（可恢复）
               任意阶段 → failed
```

## 问题

1. **零测试** — 21 个 HTTP 端点、6 阶段导入链路、状态机转换，全靠手工验证

3. **错误处理不一致** — `library.py` 路由有完整的 `IngestionError` / `Exception` 分支，`query.py` 只 `except Exception`，`session.py` 只抛 `HTTPException`
