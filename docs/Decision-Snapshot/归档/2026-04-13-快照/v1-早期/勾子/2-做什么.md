# 2-做什么

**论文助手** 做两件事：**PDF 导入** 和 **RAG 问答**。

## PDF 导入链路

```
用户上传 PDF
  → POST /api/v1/library/import（或 /import-path）
    → LibraryService.import_pdf
      → IngestionService.ingest_document
        → 1. MinerU 解析（同步阻塞，轮询等待）
        → 2. 清洗（页眉页脚/页码/噪音/去重）
        → 3. VLM 图片描述（Kimi，并发）
        → 4. 语义切分（打包 + 分割）
        → 5. Embedding（SiliconFlow，批量+重试）
        → 6. Qdrant 存储（每篇论文独立 collection）
```

支持断点续传：`POST /library/resume/{document_id}`，只从 `failed` 状态恢复。

## RAG 问答链路

```
用户提问
  → POST /api/v1/query/ask
    → QAService.ask_stream_with_rag
      → 1. Query 改写（Kimi）
      → 2. 检索（向量检索 + Rerank）
      → 3. 构建 RAG Prompt（历史 + 检索结果）
      → 4. 流式生成（Kimi，SSE）
      → 5. 保存消息到 SQLite
```

前端通过 `sse-client.ts` 的 `postAskInspirationStream` 读取 SSE 流。

## 其他端点

- `GET /query/retrieve` — 纯检索（非流式）
- `POST /library/import-path` — 本地路径导入
- `GET /library/papers` — 列出已入库论文
- `DELETE /library/papers/{id}` — 删除论文（Qdrant + 文件系统）
- `CRUD /session/*` — 会话管理
- `GET /config/` — 配置查询（敏感字段脱敏）
- `POST /config/test` — 测试 LLM 连通性
- `POST /config/reload` — 热加载配置

## 前端

WPS 侧边栏插件，通过 WPS JS SDK 与宿主通信。`sse-client.ts` 处理 SSE 流，`wps-selection.ts` 获取用户选区文本，`wps-host.ts` 轮询 WPS 文档正文。API 安全限制：只允许连接 `127.0.0.1` / `localhost` / `[::1]`。
