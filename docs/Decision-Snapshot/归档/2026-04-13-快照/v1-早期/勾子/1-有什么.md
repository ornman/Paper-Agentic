# 1-有什么

**论文助手** — WPS 侧边栏插件，中等规模。后端 60 个 Python 文件（~3000 行核心代码）+ 前端 12 个 TypeScript 文件。Python + TypeScript。版本 1.0.0-mvp。

## 技术栈

- **后端**: FastAPI + SQLite + Qdrant（本地向量库）+ httpx
- **前端**: Vue 3 + Pinia + Vite，WPS JS SDK 侧边栏
- **外部 API**: DeepSeek（主问答）、SiliconFlow（Embedding + Rerank）、MinerU（PDF 解析）、Kimi（VLM 图片描述 + 备用 LLM）

## 模块结构

```
app/
├── api/v1/routes/      5 个路由模块（health, session, query, config, library）
├── clients/            6 个外部 API 客户端（llm, embedding, rerank, mineru, vlm, kimi）
├── core/               配置管理、错误定义、重试机制、日志
├── modules/
│   ├── ingestion/      PDF 导入编排 + 清洗 + MinerU 客户端
│   ├── library/        文档库 CRUD + 状态机
│   ├── qa/             RAG 问答 + 流式输出
│   ├── retrieval/      混合检索（向量 + Rerank）
│   └── session/        会话管理
├── processing/         切分（chunker）、清洗（cleaner）、描述（describer）
├── repositories/       SQLite + BM25 仓储
├── stores/             Qdrant 向量库
└── services_archive/   5 个旧服务（已废弃但未删除）
```

## 注意

- `services_archive/` 保留了旧版服务的代码，未清理
- 前端只有 12 个文件，非常精简——UI 组件被大幅简化过（`7000ee1`）
- 配置管理用 `pydantic-settings` + `model_validator` 硬锁了 Embedding 模型和维度
