## 从代码能读到的设计决策

### 模型契约硬锁（config.py:286-298）

`model_validator` 把 Embedding 模型、维度、Rerank 模型锁死为常量。注释说："这些值属于系统契约的一部分，后续索引、检索、向量库结构都依赖它们。" 一旦模型漂移，历史向量全部失效。这是有意的刚性设计，不是疏忽。

### Kimi 伪装 Claude Code（kimi_client.py:23-29）

API 端点是 `https://api.kimi.com/coding/v1`，请求头伪装 `User-Agent: claude-code`，使用 Anthropic 消息格式。这是 Kimi 提供给 Claude Code 的免费 API，产品设计上依赖了这个灰色通道。

### 同步 MinerU 在 async 上下文（ingestion/service.py:60）

`mineru_client.run_pdf_task` 是同步方法，在 `async def ingest_document` 中直接调用。FastAPI 会把它扔进线程池，但默认线程池有限。这是 MVP 阶段的速度妥协——完整方案应该是 `asyncio.to_thread` 或异步 HTTP。

### 前端从"全功能"砍到"最小可用"（commit 7000ee1）

前端经历了"清理前端冗余功能，保留WPS插件核心组件"的重构。当前 12 个 TS 文件是精简后的结果，不是一开始就这么少。

### services_archive/ 未清理

`app/services_archive/` 保留了 5 个旧版服务文件（embedding_service, ingest_service, qa_service, qa_service_rag, retrieval_service）。代码注释说"Task 3"、"Task 4"，说明开发是按任务推进的，旧代码留着当参照。
