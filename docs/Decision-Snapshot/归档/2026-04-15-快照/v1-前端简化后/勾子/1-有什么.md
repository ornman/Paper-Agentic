## 核心数据流

### 1. 文献导入流程
```
PDF → MinerU（外部） → content_list_v2.json
  ↓
cleaner.clean_paper() (清洗噪声、提取文本)
  ↓
chunker.chunk_by_semantic_units() (语义切分)
  ↓
EmbeddingClient.embed() (向量化)
  ↓
QdrantStore.add_chunks() (存储)
```

**证据**:
- 入口: `library.py` (API 路由)
- 清洗: `cleaner.py:203-269`
- 切分: `chunker.py:205-310`
- 存储: `qdrant_store.py:113-158`

### 2. RAG 问答流程
```
用户提问 → AskRequest (api/v1/routes/query.py:24)
  ↓
QAService.ask_stream_with_rag()
  ├─ QueryRewriteService.rewrite() (查询改写)
  ├─ RetrievalService.retrieve() (混合检索)
  │   ├─ 向量检索 (Qdrant)
  │   ├─ BM25 检索 (本地)
  │   └─ RRF 融合
  ├─ 构建消息（含历史 + RAG 上下文）
  └─ KimiLLMClient.chat_stream() (流式生成)
      ↓
    SSE 事件流 (chunk/sources/rewrite/done/error)
```

**证据**:
- 主流程: `qa/service.py:79-187`
- 检索: `modules/retrieval/service.py`
- SSE 格式: `models/query.py:61-80`

### 3. 前端 WPS 集成流程
```
WPS 环境
  ↓
WpsHostAdapter.startPolling() (5秒轮询)
  ↓
读取 wps.WpsApplication().ActiveDocument
  ↓
HostStore syncSnapshot() (状态更新)
  ↓
ContextPreview 显示（已写内容 + 圈选文本 + 提示词）
  ↓
用户触发 askInspiration/sendPrompt
  ↓
SSE 流式接收 → MessageList 渲染
```

**证据**:
- 轮询: `wps-host.ts:284-306`
- 状态管理: `stores/conversation.ts:87-472`
- SSE 接收: `sse-client.ts:287-361`

## 状态机设计

### 前端对话状态（ConversationStore）
**证据**: `stores/conversation.ts:8-172`

```
状态流转:
idle → requesting → streaming → done
  ↓                   ↓
error ←───────────────┘
```

**状态定义**:
- `idle`: 初始状态
- `requesting`: 请求已发出，等待响应
- `streaming`: 正在接收 SSE 流
- `done`: 正常完成
- `error`: 发生错误

### 后端 SSE 事件类型
**证据**: `models/query.py:61-80`

```
事件类型:
- chunk:     文本片段 {"content": str, "index": int}
- sources:   引用来源 {"sources": [...]}
- rewrite:   查询改写 {"original": str, "rewritten": [str]}
- done:      完成事件 {"total_tokens": int}
- error:     错误事件 {"code": int, "message": str}
```
