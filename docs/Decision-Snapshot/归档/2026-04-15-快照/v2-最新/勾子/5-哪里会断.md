## 高危风险（🔴）

### 1. MinerU 客户端同步阻塞事件循环

**文件**: app/modules/ingestion/mineru_client.py
**触发**: time.sleep() 轮询阻塞事件循环
**影响**: 全局。所有 HTTP 请求无法响应
**修复**: ✅ P0-4 自适应轮询（异步）

### 2. asyncio.run() 在已有事件循环中崩溃

**文件**: app/clients/kimi_client.py:324, 380
**触发**: 同步函数被异步路由间接触发
**影响**: RuntimeError，导入/问答链路中断
**状态**: ⚠️ 代码仍存在

### 3. Embedding 429 限流导致无限递归

**文件**: app/clients/embedding_client.py:189-193
**触发**: 持续返回 429，递归无上限
**影响**: 栈溢出，导入流程挂起
**修复**: ✅ P0-3 并发控制 + 速率限制

### 4. Qdrant 先删后写导致数据丢失窗口

**文件**: app/stores/qdrant_store.py:141-172
**触发**: 上传中途失败，旧数据已删
**影响**: 受影响论文检索完全失效
**修复**: ✅ P0-1 幂等写入 + 状态迁移时机调整

### 5. API Key 硬编码泄露

**文件**: backend/.env
**触发**: 仓库被推送到公开平台
**影响**: 攻击者可用泄露 Key 调用所有 API
**建议**: 立即轮换所有 Key
￥只是本地开发，没有外部仓库￥

## 中等风险（⚠️）

### 6. 每次请求新建服务实例（无依赖注入）

**文件**: app/api/v1/routes/query.py, library.py
**触发**: 每次 HTTP 请求都重新创建
**影响**: TCP 连接不复用，资源浪费

### 7. BM25 索引 pickle 反序列化安全风险

**文件**: app/repositories/bm25_repo.py:168
**触发**: pickle 文件被恶意篡改
**影响**: 远程代码执行
**建议**: 改用 JSON 序列化

### 8. SSE 流式请求无超时保护

**文件**: frontend/src/services/sse-client.ts, backend/app/api/v1/routes/query.py
**触发**: LLM 服务挂起
**影响**: 前端永久停留在请求中状态
**修复**: ✅ P1-2 前端已添加重试

### 9. SQLite 写入竞争（StaticPool 单连接瓶颈）

**文件**: app/repositories/sqlite_repo.py:39
**触发**: 高并发下写入操作互相阻塞
**影响**: 数据库锁定错误
**修复**: ✅ P0-2 WAL 模式 + 线程安全

### 10. 检索降级到 BM25 时 payload 不完整

**文件**: app/modules/retrieval/service.py:103-113
**触发**: Embedding 服务不可用
**影响**: 来源卡片显示占位文本
**修复**: ✅ P1-1 BM25 降级策略（部分实现）

### 11. CORS 完全开放

**文件**: app/main.py:22
**触发**: allow_origins=["*"]
**影响**: CSRF 攻击风险
**建议**: 生产环境限制域名
