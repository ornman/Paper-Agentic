## 用户场景（基于代码推断）

### 场景一：首次使用

1. 启动后端：`python main.py` 或 `uvicorn app.main:app --reload`
2. 启动前端：`pnpm dev` 或 `wpsjs debug`
3. 前端加载后，自动连接 `http://127.0.0.1:8000`
4. 用户点击 WPS 插件"打开助手"，进入主界面

### 场景二：导入第一篇 PDF

1. 用户点击"导入文献"
2. 选择本地 PDF 文件（或输入路径）
3. 前端调用 `POST /api/v1/library/import-path`
4. 后端执行 6 阶段导入流程
5. 前端轮询状态：pending → parsing → cleaning → indexing → completed

**待补充**：
- [ ] 用户如何知道导入进度？
- [ ] 导入失败时怎么提示？
- [ ] 可以同时导入多篇吗？
￥前端不管，导入失败抛出错误，之后我们后端稳定后，再说￥
### 场景三：提问与获取灵感

1. 用户在 WPS 中选中一段文字（可选）
2. 点击"获取灵感"按钮
3. 前端调用 `POST /api/v1/query/ask`（SSE）
4. 后端执行 RAG 检索 + LLM 生成
5. 前端实时流式显示回答

**待补充**：
- [ ] 用户看到哪些中间状态？
- [ ] 来源卡片如何展示？
- [ ] 可以追问吗？
￥前端之后再说￥
### 场景四：查看历史会话

1. 用户点击"历史记录"
2. 前端调用 `GET /api/v1/session/`
3. 显示会话列表
4. 点击某个会话，加载历史消息

**待补充**：

## 痛点与解决方案（需用户访谈确认）

### 推断痛点

| 痛点 | 可能原因 | 验证方式 |
|------|----------|----------|
| 导入慢 | MinerU 轮询间隔长 | 用户反馈 |
| 问答卡顿 | KimiLLMClient 伪流式 | 用户反馈 |
| 检索不准 | BM25 降级时 payload 不完整 | 用户反馈 |
| 重连失败 | SSE 无重试机制（已修复） | 用户反馈 |

### 已知优化

- ✅ SSE 重试机制（P1-2）
- ✅ 缓冲区限制（P2-2）
- ✅ Embedding 并发（P0-3）
- ✅ 自适应轮询（P0-4）
