# 4-答应了什么？——契约与边界

## API 契约

### 统一响应格式

所有 JSON API 响应均采用统一的 ApiResponse 包装（定义于 project/MVP/backend/app/models/base.py）。

### API 路由清单

所有路由前缀为 /api/v1，聚合于 project/MVP/backend/app/api/v1/router.py。

#### 1. 健康检查 (/api/v1/health/)
- GET / - 返回 {status, version, components}

#### 2. 配置管理 (/api/v1/config/)
- GET / - 获取脱敏后的当前配置
- POST /test - 测试组件连通性
- GET /models - 返回推荐模型列表
- POST /export_logs - 导出日志到桌面
- POST /reload - 清除配置缓存并重新加载

#### 3. 会话管理 (/api/v1/session/)
- POST / - 创建会话
- GET / - 分页列出会话
- GET /{session_id} - 获取会话详情
- PUT /{session_id}/title - 更新会话标题
- DELETE /{session_id} - 删除会话
- GET /{session_id}/history - 获取历史消息
- DELETE /{session_id}/history - 清空历史消息

#### 4. 文献库 (/api/v1/library/)
- POST /import - 上传 PDF 文件触发导入链路
- POST /import-path - 指定本地路径导入 PDF
- POST /resume/{document_id} - 断点续传失败的导入
- GET /papers - 列出所有已入库论文
- GET /papers/{paper_id} - 获取单篇论文详情
- DELETE /papers/{paper_id} - 删除论文
- GET /status - 获取系统状态

#### 5. 问答与检索 (/api/v1/query/)
- POST /ask - 流式问答（SSE + RAG）
- POST /retrieve - 纯检索接口（非流式）

### 错误处理

业务错误码体系（定义于 project/MVP/backend/app/models/base.py）：
- 1000-1999: LLM
- 2000-2999: 检索
- 3000-3999: 导入
- 4000-4999: 会话
- 9000-9999: 系统

## 前端契约

### 前端安全边界

- 本机回环地址白名单：127.0.0.1, localhost, [::1], ::1
- 协议相对地址直接拒绝
- 默认 API 地址：http://127.0.0.1:8000

### SSE 重试契约

- 最多 3 次重试
- 指数退避：1s → 2s → 4s
- 缓冲区上限 1MB，单帧上限 100KB

## 风险清单

| 🔴 | Embedding 维度硬约束 | 1536 vs 4096 矛盾 |
| 🔴 | 配置系统不强制 API Key 非空 | 启动时不拒绝 |
| 🔴 | 测试文件硬编码 API Key | test_kimi_api.py |
| ⚠️ | MinerU base_url 不一致 | v1 vs v4 |
| ⚠️ | 前端索引模式固定为 brute | 后端支持 distributed |
