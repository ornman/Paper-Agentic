```
框架层：FastAPI 0.115.12 + Uvicorn
数据层：SQLAlchemy 2.0 + SQLite（元数据）、Qdrant（向量）、BM25（关键词）
外部服务：DeepSeek（LLM）、硅基流动（Embedding + Rerank）、Kimi（VLM）
HTTP 客户端：httpx 0.28.1（异步）
中文分词：jieba 0.42.1
```

### 前端（Vue 3 + TypeScript）

```
框架：Vue 3.4.0 + Pinia 2.1.0
构建：Vite 5.0.0
部署：WPS 插件（TaskPane）
SSE：原生 fetch + ReadableStream
```

## 核心流程

### PDF 导入流程（6 阶段）

```
pending → parsing(MinerU) → cleaning(清洗) → VLM描述 → chunking(切分) → Embedding → indexing(Qdrant) → completed
                                                           ↓
                                                        failed
```

**关键风险**：
- 🔴 MinerU 同步阻塞事件循环（已修复：P0-4）
- 🔴 Qdrant 先删后写导致数据丢失窗口（已修复：P0-1）
- ⚠️ VLM 描述超时 120s

### RAG 问答流程

```
Query改写(LLM) → 检索(向量+BM25+Rerank) → LLM流式生成 → SSE推送 → 保存消息
```

**关键风险**：
- 🔴 KimiLLMClient 伪流式（先完整获取再分块）
- ⚠️ BM25 降级时 payload 不完整
- ⚠️ 历史消息无截断保护

## 架构特征

### 领域模块化

```
app/
├── modules/
│   ├── ingestion/   # PDF 导入
│   ├── library/     # 文档库
│   ├── qa/          # 问答
│   ├── retrieval/   # 检索
│   └── session/     # 会话
```

### 分布式 Collection 策略

**每篇论文独立 Qdrant Collection**：
- ✅ 删除原子性（直接删 collection）
- ✅ 错误隔离（单篇失败不影响其他）
- ✅ 扩展性预留（video_xxx, note_xxx）

### 双 LLM 客户端体系

| 客户端 | 用途 | 流式支持 |
|--------|------|----------|
| LLMClient (DeepSeek) | Query 改写 | 真流式 |
| KimiLLMClient | QA 问答主力 | 伪流式 |

## 高危风险汇总（🔴 优先处理）

### 1. MinerU 同步阻塞（已修复 ✅）

**文件**：`modules/ingestion/mineru_client.py`
**触发**：`time.sleep()` 轮询阻塞事件循环
**修复**：P0-4 自适应轮询（异步）

### 2. asyncio.run() 崩溃风险（已部分修复）

**文件**：`clients/kimi_client.py:324,380`
**触发**：同步函数在异步路由中被调用
**状态**：代码仍存在，需清理

### 3. Embedding 无限递归（已修复 ✅）

**文件**：`clients/embedding_client.py:189-193`
**触发**：429 错误时递归重试无上限
**修复**：P0-3 并发控制 + 速率限制

### 4. Qdrant 数据丢失窗口（已修复 ✅）

**文件**：`stores/qdrant_store.py:141-172`
**触发**：先删后写，中途失败导致清空
**修复**：P0-1 幂等写入 + 状态迁移时机调整

### 5. API Key 泄露（需人工处理）

**文件**：`backend/.env`
**风险**：真实 API Key 已提交到仓库
**建议**：立即轮换所有 Key
