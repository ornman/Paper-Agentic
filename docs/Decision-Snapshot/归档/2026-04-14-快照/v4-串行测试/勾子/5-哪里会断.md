# 5-哪里会断 — 风险图

## 🔴 高风险断点

### 1. MinerU PDF 解析超时与轮询阻塞
**证据**: `app/clients/mineru_client.py:32-37`
```python
while True:
    if time.time() - start > timeout:
        raise TimeoutError(f"MinerU task {self.task_id} timeout after {timeout}s")
    resp = httpx.get(self.status_url, timeout=30)
    # ... sleep interval
```

**风险级别**: 🔴 Critical
**断点场景**:
- 大文件 PDF（>50 页）超过 300 秒默认超时
- MinerU 服务端无响应时，轮询持续阻塞
- 同步轮询占用事件循环，导致其他请求等待

**连锁反应**:
1. 导入任务状态卡在 "parsing"
2. 前端无超时机制，请求永久挂起
3. 数据库记录进入 "failed" 状态，但无错误详情

**缺乏保护**:
- 无请求级超时配置
- 无服务端健康检查
- 无客户端断路器

**连锁反应**:

2. Qdrant 无数据，检索返回空结果
3. 用户需重新上传 PDF，重复清洗工作

**缺乏保护**:
- 无部分成功处理机制
- 无批次级断点续传
