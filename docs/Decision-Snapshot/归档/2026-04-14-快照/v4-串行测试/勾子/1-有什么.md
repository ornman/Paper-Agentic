### 4. 业务模块 (app/modules/)

#### 4.1 导入模块 (ingestion)

**证据**: `backend/app/modules/ingestion/service.py:50-125`

**6 阶段管道**:
```
1. MinerU 解析 → 获取 PDF 文本+图片结构
2. 清洗 → 归一化为正文块（blocks）
3. VLM 图片描述 → 为图片生成文本描述
4. 混合切分 → 按语义单元打包/切分
5. Embedding → 批量向量化
6. Qdrant 存储 → 写入向量库
```

**关键常量** (backend/app/core/config.py:209-230):
- `chunk_max_context`: 32000 token（Embedding 上限）
- `chunk_target_size`: 24000 token（切分目标）
- `chunk_overlap_buffer`: 8000 token（首尾重叠）
- `chunk_pack_threshold`: 32000 token（打包阈值）

**错误处理**: 每个阶段失败会标记 `error_stage` 和 `error_message` (backend/app/modules/library/models.py，未读取但路由证实)

---

#### 4.2 检索模块 (retrieval)

**证据**: `backend/app/modules/retrieval/service.py:27-82`

**二阶段检索**:
```
阶段1: 向量检索
  - 召回 top_k * 2（默认 60 条）
  - 搜索所有 Collection

阶段2: Rerank
  - 如果召回数 > top_k，使用 Rerank 重新排序
  - 返回 top_k（默认 10 条）
```

**预留扩展** (backend/app/modules/retrieval/service.py:39-50):
- `resource_types`: 资源类型过滤（论文/笔记/视频）
- `selected_papers`: 用户自选文献 ID 列表

---

#### 4.3 问答模块 (qa)

**证据**: `backend/app/modules/qa/service.py:79-187`

**RAG Prompt 模板** (backend/app/modules/qa/service.py:16-26):
```
你是一个学术写作助手，帮助用户完成论文写作。

以下是从文献库检索到的相关内容：
{retrieved_context}

请基于上述文献内容回答用户问题。
- 回答时请引用来源，格式为 [来源:论文名 页码X]
- 如果检索内容不足以回答，请明确说明
- 不要编造文献中没有的内容

用户问题：{query}
```

**流式输出**:
- 返回 `AsyncGenerator[dict]`
- 事件类型: `chunk`, `sources`, `rewrite`, `done`, `error`

---

#### 4.4 文本处理 (processing)

**切分策略** (backend/app/processing/chunker.py:84-310):

| 场景 | 策略 |
|------|------|
| 两块合计 ≤ 32k | 打包一起 |
| 两块合计 > 32k，单个都 ≤ 32k | 分开，不用重叠 |
| 单块 > 32k | 平均切分到 24k，首尾重叠接近 32k |

## 数据流

### 1. 导入流程

```
用户提交 PDF 路径
  ↓
POST /api/v1/library/import
  ↓
IngestionService.ingest_document()
  ↓
[1] MinerU 解析 → {pages: [{blocks: [...]}]}
  ↓
[2] 清洗 → CleanedDocument {blocks: [...]}
  ↓
[3] VLM 图片描述 → Chunk {content, image_path}
  ↓
[4] 混合切分 → TextChunk {content, section}
  ↓
[5] Embedding → [[float, ...], ...]
  ↓
[6] Qdrant 存储 → Collection "paper_{id}"
  ↓
DocumentRecord {status: "completed"}
```

---

### 2. 问答流程

```
用户点击"获取灵感"
  ↓
conversationStore.askInspiration()
  ↓
POST /api/v1/query/ask (SSE)
  ↓
QAService.ask_stream_with_rag()
  ↓
[1] Query 改写 → [原始, 改写1, 改写2]
  ↓
[2] RAG 检索 → {results: [{score, payload}]}
  ↓
[3] 发送 sources 事件 → attachAssistantSources()
  ↓
[4] LLM 流式生成 → onChunk() → appendAssistantChunk()
  ↓
[5] 发送 done 事件 → finishResponse()
  ↓
消息列表增量渲染
```
