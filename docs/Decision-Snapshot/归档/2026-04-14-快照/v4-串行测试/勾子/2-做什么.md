# 2-做什么 — 行为图

本文档基于代码事实描述论文助手系统的核心行为。

## 系统行为概览

论文助手是一个基于 RAG（检索增强生成）的学术写作辅助系统，核心行为分为三条主链路：
1. **文献导入链路**：PDF → 解析 → 清洗 → 切分 → Embedding → 存储
2. **问答检索链路**：Query → 改写 → 检索 → Rerank → 生成
3. **会话管理链路**：会话创建 → 消息流式交互 → 历史管理

async def import_pdf(self, file_path: str, index_mode: str = "distributed", title: str = ""):
    """登记 PDF 并立即触发导入链路."""
    normalized_pdf_path = self._validate_import_pdf_path(file_path)
    created_record = self.import_document(file_path=normalized_pdf_path, title=title, index_mode=index_mode)
    return await self._get_ingestion_service().ingest_document(created_record)
```

**状态流转**：
1. `pending` → `parsing`：MinerU 解析开始
2. `parsing` → `cleaning`：数据清洗
3. `cleaning` → `indexing`：Embedding + Qdrant 存储
4. `indexing` → `completed`：导入完成
5. 任何阶段 → `failed`：失败记录错误阶段和消息

**证据**：
- 状态定义在 `DocumentRecord` 模型中
- `transition()` 方法执行状态迁移 (models.py 需人工补充)
- 失败时记录 `error_stage` 和 `error_message`

async def ingest_document(self, record: DocumentRecord) -> DocumentRecord:
    # 阶段1: MinerU 解析
    mineru_payload = self.mineru_client.run_pdf_task(current_record.file_path)

    # 阶段2: 清洗
    cleaned_document = await self._clean_document(current_record, mineru_payload)

    # 阶段3: VLM 图片描述
    if image_count > 0:
        chunks_with_images = await self._describe_images(current_record, mineru_payload)

    # 阶段4: 混合切分
    text_chunks = self._chunk_chunks(current_record, chunks_with_images)

    # 阶段5: Embedding
    embeddings = await self._embed_chunks(text_chunks)

    # 阶段6: Qdrant 存储
    self.qdrant_store.add_chunks(current_record.document_id, final_chunks, embeddings)
```

### 1.4 文本切分策略

**文件**：`project/MVP/backend/app/processing/chunker.py:205-310`

```python
def chunk_by_semantic_units(chunks: list[SemanticChunk]) -> list[TextChunk]:
    """按语义单元切分.

    策略：
    1. 先尝试打包（两个不超过 32k）
    2. 如果超过 32k：
       - 单个不超过 → 分开，不用重叠
       - 单个超过 → 平均切分 + 重叠
    """
```

**策略详解**：
- **打包阈值**：32k tokens (settings.chunk_pack_threshold)
- **最大上下文**：32k tokens (settings.chunk_max_context)
- **目标切分**：24k tokens (settings.chunk_target_size)
- **重叠缓冲**：8k tokens (settings.chunk_overlap_buffer)

**切分逻辑** (chunker.py:234-294)：
1. 两个块合计 ≤ 32k → 打包一起
2. 两个块合计 > 32k，但单个都 ≤ 32k → 分开，不用重叠
3. 任一单个 > 32k → 按 24k 切分，首尾重叠接近 32k

**证据**：
- `pack_chunks_if_fit()` 实现打包 (chunker.py:84-135)
- `split_long_chunk()` 实现切分 (chunker.py:138-202)
- Token 估算：中文×1.5 + 英文×0.75 (chunker.py:29-37)

async def rewrite(self, query: str, context: Optional[QueryContext] = None) -> list[str]:
    """根据输入场景改写查询

    Returns:
        改写后的查询列表（1-3 个）
    """
    prompt = self._select_prompt(query, context)
    messages = [
        {"role": "system", "content": "你是学术检索查询改写专家，输出简洁的检索语句。"},
        {"role": "user", "content": prompt},
    ]

    queries = self._parse_queries(raw_output, query)

**四种场景** (query_rewrite_service.py:130-177)：
1. **仅 prompt**：提取核心概念，去口语化
2. **仅已写内容**：推断续写意图，生成检索 query
3. **已写 + 圈选**：已写 30% + 圈选 70%
4. **已写 + 圈选 + prompt**：已写 20% + 圈选 30% + prompt 50%

**证据**：

### 2.3 混合检索

**文件**：`project/MVP/backend/app/modules/retrieval/service.py:27-82`

```python
async def retrieve(self, query: str, top_k: int = 10, ...) -> dict[str, Any]:
    """混合检索：向量检索 + Rerank.

    Returns:
        检索结果
    """
    # 阶段1: 向量检索
    query_vector = await self.embedding_client.embed_single(query)
    vector_results = self.qdrant_store.search_all(
        query_vector=query_vector,
        limit=top_k * 2,  # 召回更多，Rerank 后筛选
    )

    # 阶段2: Rerank
    if len(vector_results) > top_k:
        rerank_texts = [r["payload"]["content"] for r in vector_results]
        rerank_scores = await self.rerank_client.rerank(
            query=query,
            documents=rerank_texts,
            top_k=top_k,
        )
        # 按 Rerank 结果重新排序
        reranked_results = []
        for doc_index, score in rerank_scores:
            result = vector_results[doc_index].copy()
            result["rerank_score"] = score
            reranked_results.append(result)
        final_results = reranked_results[:top_k]
    else:
        final_results = vector_results[:top_k]
```

**证据**：
- 向量检索召回 `top_k * 2` 条（默认 20）
- Rerank 使用 Qwen3-Reranker-8B (硅基流动)
- 检索范围：所有论文 collection（暂未实现 selected_papers 过滤）

### 2.4 RAG 生成

**文件**：`project/MVP/backend/app/modules/qa/service.py:79-187`

```python
async def ask_stream_with_rag(self, session_id: str, query: str, use_rag: bool = True, ...):
    """流式问答（带 RAG + Query 改写）

    Yields:
        {"type": "chunk", "data": {"content": str, "index": int}}
        {"type": "sources", "data": {"sources": [...]}}
        {"type": "rewrite", "data": {"original": str, "rewritten": [str, ...]}}
        {"type": "done", "data": {"total_tokens": int}}
    """
    # 1. Query 改写
    rewritten_queries = await self.query_rewrite_service.rewrite(query)
    if rewritten_queries != [query]:
        yield {"type": "rewrite", "data": {"original": query, "rewritten": rewritten_queries}}

    # 2. RAG 检索
    if use_rag:
        retrieval_result = await self.retrieval_service.retrieve(rewritten_queries[0], top_k=10, ...)
        sources = retrieval_result["results"]

    # 3. 发送引用来源
    if sources:
        yield {"type": "sources", "data": {"sources": sources}}

    # 4. 构建消息（含历史和 RAG 上下文）
    messages = _build_messages(session_id, query, sources)

    # 5. 流式生成
    async for chunk in await retry_async(self.llm_client.chat_stream)(messages):
        full_response.append(chunk)
        yield {"type": "chunk", "data": {"content": chunk, "index": index}}
        index += 1

    # 6. 保存消息
    add_message(session_id, "user", query)
    add_message(session_id, "assistant", assistant_content, sources=sources if sources else None)
```

**Prompt 模板** (qa/service.py:16-26)：
```
你是一个学术写作助手，帮助用户完成论文写作。

以下是从文献库检索到的相关内容：
{retrieved_context}

请基于上述文献内容回答用户问题。
- 回答时请引用来源，格式为 [来源:论文名 页码X]
- 如果检索内容不足以回答，请明确说明
- 不要编造文献中没有的内容

用户问题：{query}
```

**证据**：
- 历史消息取最近 20 条（qa/service.py:43-44）
- RAG 上下文格式：`[来源:{paper} 页码{page}]\n{content}` (qa/service.py:50-53)
- LLM 使用 Kimi Coding API (clients/kimi_client.py)
