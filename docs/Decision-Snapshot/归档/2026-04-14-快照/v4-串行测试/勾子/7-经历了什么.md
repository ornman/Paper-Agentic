# 7-经历了什么 — 旅程图

## 需人工补充

⚠️ 代码无法推导用户旅程，需要人工补充以下内容：

### 典型用户场景
- **论文写作场景**: 用户在撰写学术论文时，需要查找相关文献、整理思路、生成初稿
- **文献综述场景**: 用户需要快速了解某个领域的研究现状，整合多篇论文的核心观点
- **论文润色场景**: 用户已有初稿，需要改进表达、调整结构、增强逻辑性

### 用户流程
1. 用户打开论文助手 Web 应用
2. 输入研究主题或上传已有论文片段
3. 系统检索相关文献并生成回答/建议
4. 用户浏览结果，进行追问或调整
5. 导出或保存有用内容到论文中

### 痛点与需求
- **痛点**: 学术论文写作耗时长，文献检索效率低，思路整理困难
  - **对应方案**: 智能检索 + RAG 生成，快速获取相关文献和观点

- **痛点**: 传统搜索引擎（Google Scholar、知网）只能检索，不能整合生成
  - **对应方案**: LangChain + OpenAI 生成式回答，直接输出可用内容

- **痛点**: 论文质量要求高，普通 AI 回答缺乏学术严谨性
  - **对应方案**: Qdrant 向量检索 + BM25 关键词匹配，双重保证相关性

### 用户旅程关键节点
- **发现阶段**: 用户如何发现这个工具？（通过推荐、搜索、社交平台？）
- **首次使用**: 第一次使用时的困惑和惊喜点
- **持续使用**: 哪些功能让用户愿意长期使用？
- **分享推荐**: 用户是否愿意推荐给他人？为什么？

### 待补充的定性信息
- 用户画像：谁在使用？（本科生、研究生、研究员？）
- 使用频率：偶尔使用 vs 日常工具？
- 满意度指标：哪些功能最受好评？哪些需要改进？
- 竞品对比：用户之前用的是什么工具？为什么切换？
