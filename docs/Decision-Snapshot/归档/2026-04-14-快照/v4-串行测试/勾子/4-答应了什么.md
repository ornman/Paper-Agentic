# 4-答应了什么 — 契约/测试

## 系统承诺的核心契约

### 1. 模型契约（不可变硬约束）

**证据**: `backend/app/core/config.py:8-12,103-133`

```python
# 固定模型常量
PINNED_EMBEDDING_MODEL = "Qwen/Qwen3-Embedding-8B"
PINNED_EMBEDDING_DIMENSIONS = 1536
PINNED_RERANK_MODEL = "Qwen/Qwen3-Reranker-8B"
```

**承诺内容**:
- Embedding 模型固定为 `Qwen/Qwen3-Embedding-8B`，维度固定为 `1536`
- Rerank 模型固定为 `Qwen/Qwen3-Reranker-8B`
- 任何模型或维度漂移都会在启动时通过 `validate_pinned_model_contract()` 拦截
- 这些是"系统硬约束"，不是"推荐值"，一旦改变会导致历史向量、检索结果和索引结构失去一致性

**测试覆盖**: `backend/tests/core/test_config_contract.py:22-39`

### 2. 错误处理契约（统一错误结构）

**证据**: `backend/app/core/errors.py:33-91`

**承诺内容**:
- 所有业务错误必须继承 `DomainError` 基类
- 错误对象必须暴露统一字段: `code`, `stage`, `message`, `detail`
- 错误必须支持 pickle/deepcopy 序列化（通过 `__reduce__` 协议）
- 错误必须满足 Python 标准异常文本契约（`args`, `str(err)`）

**错误阶段分类**:
- `ConfigError`: 配置阶段错误 (stage="config")
- `IngestionError`: 导入阶段错误 (stage="ingestion")
- `IndexingError`: 索引阶段错误 (stage="indexing")
- `RetrievalError`: 检索阶段错误 (stage="retrieval")
- `QAError`: 问答阶段错误 (stage="qa")

**测试覆盖**: `backend/tests/core/test_error_contract.py:8-113`

### 3. 文档状态机契约

**证据**: `backend/app/modules/library/models.py:20-71`

**合法状态集合**:
```python
DOCUMENT_STATUSES = (
    "pending", "parsing", "cleaning", "indexing",
    "completed", "failed", "deleting", "deleted"
)
```

**状态迁移规则**:
- `pending` → `parsing` | `failed` | `deleting`
- `parsing` → `cleaning` | `failed` | `deleting`
- `cleaning` → `indexing` | `failed` | `deleting`
- `indexing` → `completed` | `failed` | `deleting`
- `completed` → `indexing` | `deleting`
- `failed` → `pending` | `deleting`
- `deleting` → `deleted` | `failed`
- `deleted` → (终态)

**索引模式**:
- `brute`: 暴力搜索（无索引）
- `distributed`: 分布式多 Collection（每篇论文独立 Collection）
- `parent_child`: 父子 Collection（已废弃，保留用于兼容性）

**测试覆盖**: `backend/tests/library/test_document_state_machine.py:34-48`

### 6. 安全边界契约

**本地路径校验** (`frontend/src/services/api-client.ts:46-109`):
- API 基地址必须为本机回环地址 (`127.0.0.1`, `localhost`, `[::1]`, `::1`)
- 协议相对地址 (`//host`) 被明确拒绝
- 非本机地址一律回退到默认本机地址
- 导入接口只接受本地 PDF 文件路径，拒绝 URL 和 UNC 网络路径

**测试覆盖**: `backend/tests/ingestion/test_ingestion_pipeline.py:457-501`

### 7. 索引构建契约

**证据**: `backend/tests/indexing/test_index_modes.py:237-294`

**brute 模式承诺**:
- 必须产出 500~1000 token 的重叠切块
- 连续中文正文即使没有空格，也必须稳定切成 500~1000 token 窗口
- chunk.text 不能退化成逐字空格串（保持原始文本）

**parent_child 模式承诺**:
- 必须同时产出父块与子块
- 子块必须支持回挂父块（通过 `parent_chunk_id`）
- 所有子块的 `parent_chunk_id` 必须在父块 ID 集合中

**双索引写入承诺**:
- 必须同时写入 Chroma（向量）与 BM25（全文）
- 任一写入失败必须回滚已写入的数据
- 删除时必须同步删除两个索引
- BM25 保存失败时保留旧索引

**维度守卫承诺**:
- 写入前必须拦截错误的 embedding 模型与维度
- embedding 请求体必须显式携带 `dimensions` 字段（避免服务端默认值漂移）

### 10. 用户友好错误契约

**证据**: `backend/app/core/error_messages.py:9-163`

**错误信息结构**:
```python
{
    "code": str,           # 错误代码
    "user_message": str,   # 用户友好的错误描述
    "suggestion": str,     # 解决建议
    "severity": str,       # critical | error | warning | info
    "context": Any         # 额外上下文（可选）
}
```

**错误代码映射**（部分）:
- `missing_api_key`: API Key 未配置
- `mineru_parse_failed`: PDF 解析失败
- `cleaned_document_empty`: PDF 清洗后无有效内容
- `no_relevant_results`: 未找到相关内容
- `llm_generation_failed`: AI 回答生成失败

**HTTP 状态码映射**:
- `critical` → 500
- `error` → 400
- `warning` → 200（警告级别仍返回成功，但在数据中标注）
- `info` → 200
