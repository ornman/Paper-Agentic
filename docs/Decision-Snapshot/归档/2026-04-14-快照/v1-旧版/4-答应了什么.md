## 模型固定契约

`config.py` 硬编码三组不可变常量，启动时 `model_validator(mode="after")` 强制校验：

```python
PINNED_EMBEDDING_MODEL = "Qwen/Qwen3-Embedding-8B"
PINNED_EMBEDDING_DIMENSIONS = 1536
PINNED_RERANK_MODEL = "Qwen/Qwen3-Reranker-8B"
```

**校验行为**：
- `embedding_model != PINNED_EMBEDDING_MODEL` → 启动抛 ValueError
- `embedding_dimensions != PINNED_EMBEDDING_DIMENSIONS` → 启动抛 ValueError
- `rerank_model != PINNED_RERANK_MODEL` → 启动抛 ValueError

**为什么是硬约束**：一旦模型名或维度漂移，历史向量、检索结果和索引结构都会失去一致性。这是最严格的自保机制。

**兼容层**：config.py 保留旧字段名属性（`llm_api_key` → `deepseek_api_key`、`embedding_api_key` → `siliconflow_api_key` 等），避免现有客户端同时崩掉。

## 外部依赖契约

4 个外部 API，无任何 fallback：

| 服务商 | 用途 | API Key 配置 | Key 默认值 | 失效后果 |
|--------|------|-------------|-----------|----------|
| DeepSeek | 主 LLM 问答（deepseek-chat） | `DEEPSEEK_API_KEY` / `LLM_API_KEY` | `""` | QA 完全不可用 |
| SiliconFlow | Embedding（Qwen3-Embedding-8B）+ Rerank（Qwen3-Reranker-8B） | `SILICONFLOW_API_KEY` / `EMBEDDING_API_KEY` / `RERANK_API_KEY` | `""` | 导入 + 检索不可用 |
| Kimi | VLM 图片描述 + Chat（kimi-for-coding），需伪装 User-Agent: claude-code | `KIMI_API_KEY` | `""` | 图片描述 + QA 不可用 |
| MinerU | PDF 解析（提交→轮询→拉取 JSON） | `MINERU_API_KEY` | `""` | PDF 导入不可用 |

**共同风险**：
- 所有 Key 默认为空字符串，未配置时启动不报错，运行时才炸
- 无 fallback 机制：任一服务商宕机，对应功能直接不可用
- 无速率限制/熔断机制（配置层面有 `siliconflow_timeout`，但无自动重试/降级）
- Kimi API 有伪装要求（User-Agent: claude-code），若 Kimi 封堵该 User-Agent 则图片描述和 QA 同时失效
