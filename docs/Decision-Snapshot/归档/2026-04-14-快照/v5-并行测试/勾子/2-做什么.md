# 2-做什么 — 行为图

## 核心行为链路

### 1. 文献导入链路（Library → Ingestion → Qdrant）

**入口**：`POST /api/v1/library/import-path`
**位置**：`project/MVP/backend/app/api/v1/routes/library.py:89-144`

**完整流程**：
```
用户提交 PDF 路径
    ↓
LibraryService.import_pdf()  // service.py:63-90
    ↓ 创建文档记录（pending 状态）
    ↓ 转交 IngestionService
IngestionService.ingest_document()  // ingestion/service.py:50-124
    ↓
┌─────────────────────────────────────────────┐
│ 状态机转换：pending → parsing → cleaning →  │
│              indexing → completed            │
└─────────────────────────────────────────────┘
    ↓
阶段1：MinerU 解析（parsing）
    → MineruClient.run_pdf_task()
    → 输出：pages + blocks 结构
    ↓
阶段2：清洗（cleaning）
    → clean_mineru_payload()
    → 输出：CleanedDocument（blocks 列表）
    ↓
阶段3：VLM 图片描述（可选）
    → describe_chunks_async()
    → 为图片块生成文本描述
    ↓
阶段4：混合切分（chunking）
    → chunk_by_semantic_units()
    → 输出：text_chunks 列表
    ↓
阶段5：Embedding
    → EmbeddingClient.embed()
    → 输出：向量列表
    ↓
阶段6：Qdrant 存储
    → QdrantStore.add_chunks()
    → 每篇论文独立 collection
```

**断点续传支持**：
- `POST /api/v1/library/resume/{document_id}`
- 从 failed 状态恢复，重置为 pending 重新执行

**证据**：
- `project/MVP/backend/app/modules/library/service.py:63-90` - import_pdf 流程
- `project/MVP/backend/app/modules/ingestion/service.py:50-124` - ingest_document 六阶段
- `project/MVP/backend/app/api/v1/routes/library.py:147-197` - 断点续传接口

### 2. RAG 问答链路（Query → Retrieval → QA）

**入口**：`POST /api/v1/query/ask`
**位置**：`project/MVP/backend/app/api/v1/routes/query.py:23-87`

**完整流程**：
```
用户提问（含上下文）
    ↓
QAService.ask_stream_with_rag()  // qa/service.py:79-187
    ↓
阶段1：Query 改写
    → QueryRewriteService.rewrite()
    → 输出：1-3 个改写后的查询
    ↓
阶段2：RAG 检索
    → RetrievalService.retrieve()
    → 2.1 向量检索（QdrantStore.search_all）
    → 2.2 Rerank（RerankClient.rerank）
    → 输出：top_k 个相关片段
    ↓
阶段3：构建消息
    → _build_messages()
    → 系统提示 + 历史 + RAG 上下文 + 用户问题
    ↓
阶段4：流式生成
    → LLMClient.chat_stream()
    → SSE 流式输出
    ↓
阶段5：保存消息
    → add_message()
    → 用户消息 + 助手回复 + 引用来源
```

**SSE 事件类型**：
- `rewrite`：查询改写结果
- `sources`：引用来源列表
- `chunk`：流式文本片段
- `done`：完成
- `error`：错误

**证据**：
- `project/MVP/backend/app/modules/qa/service.py:79-187` - ask_stream_with_rag 主流程
- `project/MVP/backend/app/modules/retrieval/service.py:27-82` - retrieve 混合检索
- `project/MVP/backend/app/api/v1/routes/query.py:62-77` - SSE 事件生成

### 3. Query 改写策略（4种场景）

**位置**：`project/MVP/backend/app/services/query_rewrite_service.py`

**场景判断**：
```
┌─────────────────┬───────────────┬───────────────┬─────────────────┐
│ 已写内容        │ 圈选文本      │ 用户 prompt   │ 策略            │
├─────────────────┼───────────────┼───────────────┼─────────────────┤
│ ✗              │ ✗             │ ✓             │ 场景1：直接改写  │
│ ✓              │ ✗             │ ✗             │ 场景2：推断续写  │
│ ✓              │ ✓             │ ✗             │ 场景3：圈选重点  │
│ ✓              │ ✓             │ ✓             │ 场景4：完整输入  │
└─────────────────┴───────────────┴───────────────┴─────────────────┘

权重分配：
- 场景1：prompt 100%
- 场景2：已写内容 100%
- 场景3：已写 30% + 圈选 70%
- 场景4：已写 20% + 圈选 30% + prompt 50%
```

**证据**：
- `project/MVP/backend/app/services/query_rewrite_service.py:97-178` - rewrite 场景判断

## 数据流关键节点

### 1. 向量检索（Qdrant）

**位置**：`project/MVP/backend/app/stores/qdrant_store.py`

**策略**：
- 每篇论文独立 collection（`paper_{paper_id}`）
- 分布式检索：遍历所有 collection 并合并结果
- 按分数排序后返回 top_k

**内存优化配置**：
- `m`：12（默认 16）
- `ef_construct`：64（默认 100）
- `full_scan_threshold`：5000（默认 10000）

**证据**：
- `project/MVP/backend/app/stores/qdrant_store.py:29-299` - Qdrant 完整实现

---

### 2. 文本切分策略

**位置**：`project/MVP/backend/app/processing/chunker.py`

**混合切分规则**：
```
两个语义块合计 ≤ 32k → 打包一起
合计 > 32k：
  - 单个 ≤ 32k → 分开，无重叠
  - 单个 > 32k → 平均切分到 24k，首尾重叠接近 32k
```

**证据**：
- `project/MVP/backend/app/processing/chunker.py:1-100` - 切分策略说明

---

### 3. 错误处理与降级

**重试机制**：
- VLM 图片描述：`@retry_async(max_retries=3)`
- Embedding：`@retry_async(max_retries=3)`

**降级策略**：
- Query 改写失败 → 使用原始查询
- RAG 检索失败 → 回退到纯问答
- VLM 描述失败 → 跳过图片块
- SSE 无 done 事件 → 非 error 状态视为完成

**证据**：
- `project/MVP/backend/app/core/retry.py` - 重试装饰器
- `project/MVP/backend/app/modules/qa/service.py:122-127` - Query 改写降级
- `project/MVP/backend/app/modules/qa/service.py:138-150` - RAG 检索降级
- `project/MVP/frontend/src/stores/conversation.ts:363-369` - SSE 兜底
