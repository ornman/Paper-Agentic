## 核心行为概述

论文助手是一个**基于 RAG 的学术写作辅助系统**，核心行为围绕"文献导入→检索→问答"的数据流展开。

### 系统定位（main.py:14-18）
```python
app = FastAPI(
    title="论文写作助手 API",
    version="1.0.0-mvp",
    description="基于 RAG 的学术写作辅助系统",
)
```

### 一、文献导入行为

#### 1.1 导入入口（library.py:35-87）

**事实**：
- 入口：`POST /api/v1/library/import`（上传方式）或 `POST /api/v1/library/import-path`（路径方式）
- 完整流程：`MinerU 解析 → 清洗 → VLM 描述 → 切分 → Embedding → Qdrant 存储`
- 证据：library.py:39 注释 `完整流程：MinerU 解析 → 清洗 → VLM 描述 → 切分 → Embedding → Qdrant 存储`

**状态机**（library.py:90-91）：
- 入口参数 `index_mode` 支持 `"distributed"`（默认）或 `"brute"`
- 文档状态流转：`pending → parsing → cleaning → indexing → completed`

**断点续传**（library.py:147-153）：
- 入口：`POST /api/v1/library/resume/{document_id}`
- 功能：从失败阶段自动恢复并继续执行

**错误处理**（library.py:62-86）：
- 使用 `IngestionError` 区分业务错误
- 通过 `format_user_error` 提供用户友好的错误信息
- 错误信息包含：code、user_message、suggestion、severity

#### 1.2 导入编排（ingestion/service.py:50-124）

**六阶段流程**（ingestion/service.py:58-106）：

| 阶段 | 行为 | 证据 |
|------|------|------|
| 1. MinerU 解析 | 调用 MinerU 客户端解析 PDF | ingestion/service.py:59-62 |
| 2. 清洗 | 清洗 MinerU 结果为正文块 | ingestion/service.py:67-75 |
| 3. VLM 图片描述 | 为图片生成 VLM 描述（带重试） | ingestion/service.py:78-88 |
| 4. 混合切分 | 按语义单元切分文本 | ingestion/service.py:91-93 |
| 5. Embedding | 为 chunks 生成向量（带重试） | ingestion/service.py:98-100 |
| 6. Qdrant 存储 | 存储向量和元数据 | ingestion/service.py:103-106 |

**图片描述策略**（ingestion/service.py:176-195）：
- 先过滤出有图片的 chunks
- 并发调用 VLM（可配置并发数）
- 使用 `@retry_async(max_retries=3)` 重试机制

#### 1.3 文本切分行为（chunker.py:205-310）

**混合切分策略**（chunker.py:1-8 注释）：
```
1. 如果两个语义块合计不超过 32k → 打包一起
2. 如果超过 32k：
   - 情况1：超过但单个不超过 → 分开，不用重叠
   - 情况2：超过且单个也超过 → 平均切分到 24k，首尾重叠接近 32k
```

**配置参数**（config.py:209-230）：
- `chunk_max_context`: 32000（Embedding 模型最大上下文）
- `chunk_target_size`: 24000（目标切分大小）
- `chunk_overlap_buffer`: 8000（首尾重叠缓冲）
- `chunk_pack_threshold`: 32000（两个语义块打包阈值）

#### 1.4 向量存储行为（qdrant_store.py:66-158）

**分布式存储**（qdrant_store.py:2-3 注释）：
```
每篇论文独立 collection，支持分布式隔离管理。
```

**Collection 命名**（qdrant_store.py:29-37）：
- 格式：`paper_{paper_id}`（合法化处理）
- 只能包含字母、数字、下划线、连字符

**内存优化策略**（qdrant_store.py:100-109）：
```python
hnsw_config=HnswConfigDiff(
    m=settings.qdrant_hnsw_m,           # 默认 12（每个向量连接的最大节点数）
    ef_construct=settings.qdrant_hnsw_ef_construct,  # 默认 64（构建索引时的搜索范围）
    full_scan_threshold=settings.qdrant_full_scan_threshold,  # 默认 5000（全扫描阈值）
)
```

**存储结构**（qdrant_store.py:136-152）：
```python
payload={
    "content": chunk.content,

    "page": chunk.page,
```

## 风险标注

### 🔴 高风险

1. **向量维度漂移**（config.py:286-298）
   - 风险：Embedding 模型变更导致历史向量失效
   - 缓解：启动期强校验，禁止配置漂移

2. **本机 API 白名单绕过**（api-client.ts:46-109）
   - 风险：误配置可能导致本地 PDF 路径泄露到外部主机
   - 缓解：强制本机白名单校验

3. **VLM 并发过载**（config.py:170-174）
   - 风险：大量图片同时描述可能击穿 API 限速
   - 缓解：可配置并发数（默认 10）

### ⚠️ 中风险

1. **轮询固定周期**（wps-host.ts:10）
   - 风险：5 秒轮询可能不适合所有场景
   - 缓解：导出为常量，便于后续调整

2. **Query 改写失败降级**（query_rewrite_service.py:126-128）
   - 风险：改写失败时直接用原始查询，可能影响检索质量
   - 缓解：记录日志，便于监控

3. **历史消息数量**（qa/service.py:43）
   - 风险：固定保留最近 20 条，长对话可能丢失早期上下文
   - 缓解：可配置化（需人工补充）
