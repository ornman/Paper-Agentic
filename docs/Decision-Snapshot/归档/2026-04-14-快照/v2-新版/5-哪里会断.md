# 5-哪里会断 — 风险点分析

> 基于代码事实的断点识别，标注风险级别和证据来源。

## 🔴 高风险断点（可能导致系统不可用）

### 1. MinerU 轮询超时死锁

**证据**: `app/clients/mineru_client.py:29-52`
```python
while True:
    if time.time() - start > timeout:
        raise TimeoutError(...)
    resp = httpx.get(self.status_url, timeout=30)  # 同步阻塞
```

**风险**:
- 轮询循环使用同步 `httpx.get` 和 `time.sleep`
- 调用方 `IngestionService.ingest_document` (line 60) 在异步上下文中直接调用 `run_pdf_task`
- 长时间 PDF 解析会阻塞整个 FastAPI 事件循环

**影响范围**: 整个导入流程，所有并发请求

**修复方向**: 改为异步轮询或后台任务队列

---

### 2. Qdrant 内存溢出风险

**证据**: `app/core/config.py:190-207` + `app/stores/qdrant_store.py:100-111`
```python
qdrant_hnsw_m: int = Field(default=12, ge=4, le=64)
qdrant_hnsw_ef_construct: int = Field(default=64, ge=10, le=200)
qdrant_full_scan_threshold: int = Field(default=5000, ge=1000, le=50000)
```

**风险**:
- 默认配置假设本地运行场景，内存优化参数固定
- 远程 Qdrant 或大规模数据时可能出现 OOM
- `create_paper_collection` 没有验证 collection 是否已存在时的点数

**影响范围**: 向量存储阶段，多文档导入场景

**修复方向**: 配置分层 + 动态参数调整

---

### 3. Embedding API 无限重试

**证据**: `app/clients/embedding_client.py:29-30` + `app/core/retry.py:54-107`
```python
@retry_async(max_retries=3)
async def embed(self, texts: list[str]) -> list[list[float]]:
```

**风险**:
- SiliconFlow API 故障时会指数退避重试（1s → 2s → 4s）
- 批量调用时可能放大失败影响
- `embedding_batch_size=32` 时单次失败会损失 32 个文本

**影响范围**: Embedding 阶段，大批量导入

**修复方向**: 熔断机制 + 失败批次隔离

---

### 4. VLM 并发限制缺失

**证据**: `app/processing/describer.py:21` + `app/core/config.py:170-174`
```python
CONCURRENCY = settings.kimi_vlm_concurrency  # 默认 10
kimi_vlm_concurrency: int = Field(default=10, ge=1)
```

**风险**:
- VLM API 限流时没有降级策略
- `_describe_one` 失败只打印警告，不中断流程
- 大量图片时可能全部返回 "description unavailable"

**影响范围**: 图片描述阶段，含图表的学术论文

**修复方向**: 指数退避 + 失败计数阈值

---

### 5. BM25 索引重建性能

**证据**: `app/repositories/bm25_repo.py:27-44`
```python
def add(self, doc_id: str, document: str):
    self._doc_ids.append(doc_id)
    self._corpus.append(document)
    tokens = list(jieba.cut(document))
    self._tokenized.append(tokens)
    self._bm25 = BM25Okapi(self._tokenized)  # 每次都重建整个索引
```

**风险**:
- 每次添加文档都重建整个 BM25 索引
- 文档数量增长时性能退化 O(n²)
- 当前 `bm25_index.json` 存在但未在检索流程中使用

**影响范围**: 关键词检索阶段，大规模文献库

**修复方向**: 延迟重建 + 批量写入

---

## ⚠️ 中风险断点（影响用户体验）

### 6. 前端 SSE 连接中断处理

**证据**: `frontend/src/services/sse-client.ts:287-361`
```typescript
async function readSseStream(body, handlers) {
  const reader = body.getReader();
  // 没有 read timeout 机制
  while (!terminalEventReached) {
    const { done, value } = await reader.read();
  }
}
```

**风险**:
- SSE 长时间无响应时会永久挂起
- 网络抖动时无法自动重连
- 用户可能看到"请求中"状态持续不变

**影响范围**: 流式问答用户体验

**修复方向**: 读超时 + 心跳检测

---

### 7. 流式输出状态不一致

**证据**: `frontend/src/stores/conversation.ts:260-283`
```typescript
function appendAssistantChunk(chunkText: string) {
  const messageId = ensureStreamingAssistantMessage();
  updateAssistantMessage(messageId, (message) => ({
    ...message,
    content: `${message.content}${chunkText}`,
  }));
}
```

**风险**:
- chunk 到达顺序不保证时可能出现乱序
- `sources` 事件早于 `chunk` 时会暂存，但暂存逻辑复杂
- 多个并发请求时 `activeAssistantMessageId` 可能混淆

**影响范围**: 流式输出显示

**修复方向**: 请求 ID 绑定 + 顺序校验

---

### 8. 环境变量校验延迟

**证据**: `app/core/config.py:286-298`
```python
@model_validator(mode="after")
def validate_pinned_model_contract(self) -> "Settings":
    if self.embedding_model != PINNED_EMBEDDING_MODEL:
        raise ValueError(...)
```

**风险**:

- 但错误消息对非技术用户不友好
- 兼容层属性（line 302-341）掩盖了字段漂移问题

**影响范围**: 应用启动，配置迁移

**修复方向**: 分层校验 + 迁移指南

### 9. 文件路径注入风险

**证据**: `frontend/src/services/api-client.ts:76-109`
```typescript
function normalizeConfiguredApiBaseUrl(configuredBaseUrl: string): string {
  // 有 allowlist 校验，但只在导入路径场景生效
  if (configuredBaseUrl.startsWith('//')) {
    return DEFAULT_API_BASE_URL
  }
}
```

**风险**:
- 导入接口携带本地路径，但校验逻辑复杂
- 如果 `VITE_API_BASE_URL` 配置错误，可能泄露本地路径
- 当前 `importLibraryPdf` 没有验证路径合法性

**影响范围**: 隐私安全

**修复方向**: 路径脱敏 + 服务端校验

### 10. SQLite 并发写入限制

**证据**: `app/repositories/sqlite_repo.py` (未在读取范围内，需补充)

**风险**:
- SQLite 同一时间只允许一个写事务
- 多文档并发导入时可能出现锁等待
- `get_engine()` 在 `startup` 事件中调用，但没有连接池配置

**影响范围**: 批量导入性能

**修复方向**: WAL 模式 + 队列化写入
