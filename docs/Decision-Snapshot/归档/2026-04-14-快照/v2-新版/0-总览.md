## 最需要看的

### 🔴 [5-哪里会断] MinerU 同步阻塞会卡死整个服务
`mineru_client.py:29-52` 轮询使用同步 `httpx.get`，在异步 FastAPI 上下文中直接调用会阻塞事件循环。

**修复方向**: 改为后台任务队列或异步轮询。

---

### 🔴 [4-答应了什么] Embedding 模型固定契约一旦漂移会破坏所有历史索引
`config.py:286-298` 强制校验模型名称和维度为 `Qwen3-Embedding-8B`/1536，无迁移方案。

**风险**: 升级模型需手动重建所有索引，无法快速切换。

---

### 🔴 [3-为什么] 分布式架构在论文数 >1000 时性能会退化
每篇论文独立 Qdrant Collection，跨论文检索需遍历所有 Collection，无批量优化。

**证据**: `qdrant_store.py:196-235` 的 `search_all()` 遍历逻辑。

---

### ⚠️ [1-有什么] Embedding 配置维度（1536）与 API 实际（4096）不匹配
`config.py:103` 配置 1536 维，但 `embedding_client.py:38` 发现 API 实际返回 4096 维。

**风险**: 向量空间可能不一致。

---

## 一句话定性

基于 RAG 的学术写作辅助系统，**71 个 Python 文件 + 28 个 Vue 组件**，6 阶段导入链路（MinerU → 清洗 → VLM → 切分 → Embedding → 存储），采用"一论文一 Collection"分布式架构，当前以 WPS 插件为主界面。
