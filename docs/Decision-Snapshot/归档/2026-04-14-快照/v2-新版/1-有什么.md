**类型**：基于 RAG 的学术写作助手
**架构**：前后端分离，本地部署

## 数据流（事实）

### PDF导入流程（6阶段）

```
PDF → MinerU解析 → 清洗 → VLM图片描述 → 混合切分 → Embedding → Qdrant存储
```

证据：`backend/app/modules/ingestion/service.py:50-114`

各阶段：
1. **parsing**：MinerU API 异步轮询解析
2. **cleaning**：去除页眉页脚、脚注、噪音
3. **VLM描述**：Kimi VLM 为图片生成描述
4. **chunking**：混合语义切分（打包/拆分/重叠）
5. **embedding**：批量向量化
6. **indexing**：Qdrant 存储向量

状态机：`pending → parsing → cleaning → indexing → completed`
证据：`backend/app/modules/library/models.py:62-71`

### 问答流程（带RAG）

```
用户输入 → Query改写 → 混合检索 → Rerank → LLM生成 → SSE流式输出
```

证据：`backend/app/modules/qa/service.py:79-187`

检索策略：
- **向量检索**：Qdrant 全 collection 搜索
- **Rerank**：对 Top-K 结果重排序
- **RRF 融合**：⚠️ 代码中有注释但未实现

证据：`backend/app/modules/retrieval/service.py:27-82`

### Query改写（4场景）

| 场景 | 输入 | 权重分配 | 证据 |
|------|------|----------|------|
| 1 | 仅有 prompt | prompt 100% | `backend/app/services/query_rewrite_service.py:21-33` |
| 2 | 仅有已写内容 | 推断续写意图 | `backend/app/services/query_rewrite_service.py:35-48` |
| 3 | 已写 + 圈选 | 已写30% + 圈选70% | `backend/app/services/query_rewrite_service.py:50-66` |
| 4 | 已写 + 圈选 + prompt | 已写20% + 圈选30% + prompt50% | `backend/app/services/query_rewrite_service.py:68-88` |

## 索引策略（事实）

### Qdrant 存储模式

**分布式多 Collection**：
- 每篇论文独立 Collection
- Collection 命名：`paper_{document_id}`（合法化处理）
- 支持单篇检索和全库检索

证据：`backend/app/stores/qdrant_store.py:29-38,66-111`

内存优化配置（可调）：
- `hnsw_m`：12（默认16）
- `ef_construct`：64（默认100）
- `full_scan_threshold`：5000（默认10000）

证据：`backend/app/core/config.py:189-207`

### 文本切分策略

**混合语义策略**：
1. 两个块合计 ≤32k → 打包
2. 超过32k但单个不超过 → 分开，无重叠
3. 单个超过32k → 切分到24k，首尾重叠接近32k

证据：`backend/app/processing/chunker.py:84-203`

### 关键词检索（BM25）

- 分词：jieba
- 索引：`rank_bm25.BM25Okapi`
- 持久化：pickle 序列化
- 路径：`./data/bm25_index.json`

证据：`backend/app/repositories/bm25_repo.py:12-18,90-98`

⚠️ **风险**：BM25 索引未集成到主检索流程
证据：`backend/app/modules/retrieval/service.py` 仅有向量检索和 Rerank

## 数据模型（事实）

### 统一响应格式

```json
{
  "code": 0,
  "data": {...},
  "message": "success"
}
```

证据：`backend/app/models/base.py:22-26`, `frontend/src/services/api-client.ts:1-5`

### 文档状态机

合法状态：`pending, parsing, cleaning, indexing, completed, failed, deleting, deleted`

状态迁移规则（部分）：
- `pending` → `parsing, failed, deleting`
- `parsing` → `cleaning, failed, deleting`
- `completed` → `indexing, deleting`

证据：`backend/app/modules/library/models.py:20-29,62-71`

### 索引模式

- `brute`：暴力搜索（无索引）
- `distributed`：分布式多 Collection（每篇论文独立）
- `parent_child`：父子 Collection（已废弃）

证据：`backend/app/modules/library/models.py:32-39`

## 安全机制（事实）

### 前端 API 基地址校验

本机白名单：`127.0.0.1, localhost, [::1], ::1`

拒绝：
- 协议相对地址（`//host`）
- 非本机地址
- 非纯 origin（带 path/search/hash）

证据：`frontend/src/services/api-client.ts:44-109`

### Kimi API 伪装

强制 User-Agent：`claude-code`

证据：`backend/app/clients/kimi_client.py:24,29`

### PDF 路径校验

只接受本地 PDF 文件：
- 拒绝 URL/file URI/UNC 网络路径
- 必须以 `.pdf` 结尾
- 文件必须存在

证据：`backend/app/modules/library/service.py:138-173`
