# 7-经历了什么 — 用户旅程

## 需人工补充

代码无法推导用户旅程。需要人工补充以下信息：

### 1. 论文导入流程

- [ ] 用户如何触发导入？（拖拽？点击上传？WPS 内部触发？）
- [ ] 导入进度如何展示？（进度条？百分比？阶段提示？）
- [ ] 导入失败如何处理？（重试？错误提示？部分导入？）
- [ ] 导入后的论文如何呈现？（列表？卡片？搜索？）

### 2. 查询交互流程

- [ ] 用户如何发起查询？（输入框？语音？WPS 选区？）
- [ ] Query 改写如何触发？（自动？用户确认？）
- [ ] 检索结果如何展示？（来源列表？高亮？排序？）
- [ ] 用户如何选择参考文档？（自动？手动？多选？）

### 3. 问答输出流程

- [ ] AI 回答如何展示？（流式？一次性？Markdown 渲染？）
- [ ] 来源引用如何呈现？（脚注？内联？悬浮？）
- [ ] 用户如何追问？（续写按钮？重新输入？上下文保持？）
- [ ] 回答不满意如何处理？（重新生成？反馈？人工介入？）

### 4. WPS 集成场景

- [ ] 用户如何在 WPS 中打开侧边栏？（ ribbon 按钮？快捷键？）
- [ ] WPS 选区如何传递给后端？（实时？提交按钮？）
- [ ] AI 回答如何插入 WPS？（替换选区？追加？粘贴板？）
- [ ] 用户如何在 WPS 和侧边栏间切换？（焦点管理？同步滚动？）

### 5. 配置与设置

- [ ] 用户如何配置 API Key？（环境变量？配置文件？UI 界面？）
- [ ] 模型切换是否对用户可见？（固定模型？用户可选？）
- [ ] 主题/字体设置如何持久化？（本地存储？云端同步？）

### 6. 错误与异常

- [ ] API 失败时用户看到什么？（错误码？友好提示？重试按钮？）
- [ ] 网络断开如何处理？（自动重连？离线模式？）
- [ ] Qdrant 连接失败如何降级？（纯 BM25？错误提示？）

---

## 建议收集方法

1. **用户访谈**：与实际用户面对面沟通，观察操作流程
2. **录屏分析**：收集用户使用过程的录屏，分析真实行为路径
3. **日志分析**：分析后端 API 调用日志，推断用户操作序列
4. **前端埋点**：添加用户行为追踪（点击、停留时间、页面跳转）
