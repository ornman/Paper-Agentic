## 静默失效风险（不报错但产出错误结果）

## 长时间运行任务中断分析

### PDF 导入中断风险

**流程时长分析**:
```
ingest_document() 的 6 个阶段：

1. MinerU 解析  ── 5-10 分钟 ── 🔴 中断风险: 高
2. 数据清洗     ── 秒级       ── 🟢 中断风险: 低
3. VLM 描述    ── 数分钟     ── 🟡 中断风险: 中
4. 混合切分     ── 秒级       ── 🟢 中断风险: 低
5. Embedding   ── 数十秒     ── 🟢 中断风险: 低
6. Qdrant 存储  ── 秒级       ── 🟢 中断风险: 低
```

### 中断点残留数据分析

| 中断点 | 残留数据 | 清理逻辑 | 风险等级 |
|--------|---------|---------|---------|
| MinerU 解析中断 | 服务器端任务可能仍在执行 | **❌ 无清理** | 🔴 **高** |
| VLM 描述中断 | 部分 chunks 有描述，部分没有 | **❌ 无清理** | 🟡 **中** |
| Qdrant 存储前中断 | 内存数据丢失，无持久化影响 | ✅ 无需清理 | 🟢 **低** |
| Qdrant 存储中中断 | Collection 可能不完整 | **❌ 无清理** | 🟡 **中** |
| 任意阶段后再次导入 | 取决于 paper_id 是否冲突 | **⚠️ 可能创建重复 Collection** | 🟡 **中** |

### 关键场景推演

**场景**: 用户在导入过程中关闭 WPS，再次打开系统显示什么？

**模型推断**:
```
前端断连后，后端 ingest_pdf() 仍继续执行
（FastAPI 的 SSE 连接断开不自动取消后台任务）

结果分支：
├─ 任务成功完成 → Qdrant 存在完整 Collection
│  └─ 用户再次打开 → 系统不知道导入已完成（无状态追踪）
│
└─ 任务中途失败 → Qdrant 存在不完整 Collection
   └─ 用户再次导入 → paper_id 冲突或创建重复 Collection
```

**风险**: 无状态追踪 + 无清理机制 → 数据一致性问题

## 级联失效分析

### 故障传播链

```
MinerU API 故障
  └─ PDF 导入失败
      └─ 向量库无新数据
          └─ 检索结果陈旧
              └─ 回答质量下降

Kimi API 故障
  └─ VLM 描述失败
      └─ 图片信息丢失
          └─ 检索召回率下降
              └─ 回答不完整
  └─ RAG 问答失败
      └─ 只能纯 LLM 问答
          └─ 无文献引用
```

### 无熔断机制风险

**当前状态**:
- 所有外部调用均无限流保护
- 故障会持续重试（依赖 `retry_async`）
- 可能触发 API 限流或 IP 封禁

**建议**:
```python
# 🔮 未来：添加熔断器
from circuitbreaker import circuit

@circuit(failure_threshold=5, recovery_timeout=60)
async def call_mineru_with_circuit_breaker(...):
    return await self.mineru_client.run_pdf_task(...)
```
