### 1. PDF 导入完整时序

**位置**：`app/modules/ingestion/service.py:50-122`

#### 六阶段流程

```
[用户上传PDF]
    ↓
[1. MinerU解析] 10-30s
    ├─ POST /mineru/tasks
    ├─ 轮询 /mineru/tasks/{task_id}
    └─ GET /mineru/tasks/{task_id}/download
    ↓
[2. 清洗数据] 1-3s
    ├─ 读取 content_list_v2.json
    ├─ 过滤噪声（page_header, page_footnote）
    ├─ 提取文本、图片、表格
    └─ 生成 Chunk[]
    ↓
[3. VLM图片描述] 5-60s
    ├─ 统计图片数量
    ├─ 加载缓存（image_descriptions.json）
    ├─ 并发调用 Kimi VLM（最多10个）
    └─ 更新 Chunk[].content
    ↓
[4. 混合切分] 1-2s
    ├─ 打包小 chunks（≤32k）
    ├─ 切分大 chunks（>32k）
    └─ 生成 TextChunk[]
    ↓
[5. Embedding] 5-15s
    ├─ 批量调用 SiliconFlow（32个/批）
    └─ 生成 float[1536][]
    ↓
[6. Qdrant存储] 1-3s
    ├─ 创建 Collection（paper_{id}）
    ├─ 上传向量+Payload
    └─ 返回 completed
```

### 2. RAG 问答时序

**位置**：`app/modules/qa/service.py`

#### 完整流程

```
[用户提问]
    ↓
[1. Query改写] 1-3s
    ├─ 分析 Query 类型
    ├─ 应用改写策略
    └─ 生成改写后 Query
    ↓
[2. 向量检索] 0.1-0.5s
    ├─ Embedding Query
    ├─ Qdrant.search_all()
```

### 3. VLM 图片描述时序

**位置**：`app/processing/describer.py:45-138`

#### 异步并发流程

```
[遍历Chunks]
    ↓
[过滤有图片的Chunks]
    ↓
[加载缓存]
    ├─ 读取 image_descriptions.json
    └─ 跳过已描述的图片
    ↓
[并发调用VLM] (Semaphore=10)
    ├─ Task 1: describe_image(fig1.png)
    ├─ Task 2: describe_image(fig2.png)
    ├─ ...
    └─ Task 10: describe_image(fig10.png)
    ↓
[收集结果]
    ↓
[更新缓存]
    └─ 写入 image_descriptions.json
    ↓
[更新Chunks]
    └─ content = "[Image]\nDescription: ...\nOriginal: ..."
```

#### 关键代码路径

**缓存读取**

**证据**：`describer.py:63-75`
```python
cache = _load_cache(paper_dir)

to_describe: list[tuple[int, Chunk]] = []
for i, chunk in enumerate(chunks):
    if not chunk.image_path:
        continue
    abs_path = paper_dir / chunk.image_path
    if not abs_path.exists():
        continue
    rel_path = str(chunk.image_path)
    if rel_path not in cache:
        to_describe.append((i, chunk))
```

**并发调用**

**证据**：`describer.py:78-108`
```python
semaphore = asyncio.Semaphore(CONCURRENCY)

async def _describe_one(index: int, chunk: Chunk) -> tuple[int, str, str]:
    abs_path = paper_dir / chunk.image_path
    try:
        description = await vlm_client.describe_image(abs_path)
        return index, str(chunk.image_path), description
    except Exception as e:
        print(f"[WARN] VLM failed for {chunk.image_path}: {e}")
        return index, str(chunk.image_path), "description unavailable"

async def _run_batch():
    tasks = [
        _describe_one(i, chunk)
        for i, chunk in to_describe
    ]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    return [
        r for r in results
        if not isinstance(r, Exception)
    ]

print(f"    VLM: {len(to_describe)} images to describe ({CONCURRENCY} concurrent)...")

results = asyncio.run(_run_batch())
for index, rel_path, description in results:
    cache[rel_path] = description

_save_cache(paper_dir, cache)
```

## ￥问题￥ - 时序问题

### 1. ￥VLM 描述失败无重试￥

**事实**：`describer.py:84-88`
```python
try:
    description = await vlm_client.describe_image(abs_path)
    return index, str(chunk.image_path), description
except Exception as e:
    print(f"[WARN] VLM failed for {chunk.image_path}: {e}")
    return index, str(chunk.image_path), "description unavailable"
```

**问题**：
- 一次失败后永久缓存 `"description unavailable"`
- 无重试机制
- 无错误分类（临时性 vs 永久性错误）

**影响**：
- 网络抖动导致永久失败
- API 限流导致批量失败
- 用户体验差

### 2. ￥MinerU 轮询间隔固定￥

**事实**：`mineru_client.py` 轮询间隔固定为 5 秒

**问题**：
- 无法根据任务状态动态调整
- 小文件等待时间过长
- 大文件轮询过于频繁

**建议**：
```python
# 自适应轮询
poll_interval = min(5, elapsed_time / 10)  # 最小1秒，最大5秒
```

### 3. ￥Qdrant 批量上传无错误恢复￥

**事实**：`qdrant_store.py:155-158`
```python
self.client.upsert(
    collection_name=collection_name,
    points=points,
)
```

**问题**：
- 单个点失败导致整批失败
- 无部分成功恢复
- 无断点续传

**影响**：
- 大文件导入时风险高
- 网络不稳定时容易失败
- 浪费已处理的数据

### 4. ￥SSE 流中断无恢复￥

**事实**：前端 SSE 客户端无重连逻辑

**问题**：
- 网络抖动导致流中断
- 用户需要手动刷新
- 已消耗的 token 浪费

**证据**：`src/services/sse-client.ts` 无 `retry` 逻辑
