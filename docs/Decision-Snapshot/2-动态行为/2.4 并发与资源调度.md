**【代码事实】** `app/processing/describer.py:79-103`

```python
# 并发调用 VLM
if to_describe:
    semaphore = asyncio.Semaphore(CONCURRENCY)
    
    async def _describe_one(index: int, chunk: Chunk) -> tuple[int, str, str]:
        abs_path = paper_dir / chunk.image_path
        try:
            description = await vlm_client.describe_image(abs_path)
            return index, str(chunk.image_path), description
        except Exception as e:
            print(f"[WARN] VLM failed for {chunk.image_path}: {e}")
            return index, str(chunk.image_path), "description unavailable"
    
    async def _run_batch():
        tasks = [
            _describe_one(i, chunk)
            for i, chunk in to_describe
        ]
        results = await asyncio.gather(*tasks, return_exceptions=True)
        return [
            r for r in results
            if not isinstance(r, Exception)
        ]
    
    print(f"    VLM: {len(to_describe)} images to describe ({CONCURRENCY} concurrent)...")
    results = asyncio.run(_run_batch())
```

**【模型推断】**：
- ✅ 使用Semaphore限制并发数
- ✅ 失败不中断其他请求
- ⚠️ 缺乏超时控制

## 二、同步阻塞点分析

### 2.1 导入流程中的同步调用

**【代码事实】** `app/modules/ingestion/service.py:78`

```python
# 阶段3: VLM 图片描述
if image_count > 0:
    chunks_with_images = asyncio.run(self._describe_images(...))
```

**￥问题￥：同步函数内调用asyncio.run()**

**问题**：
- `ingest_document`是同步函数，内部调用`asyncio.run()`
- 每次调用会创建新的事件循环，无法与现有异步代码集成
- 可能阻塞FastAPI的异步工作线程

**影响**：
- 高并发时可能导致线程池耗尽
- 无法利用FastAPI的异步特性

**建议**：
- 导入服务改为全异步
- FastAPI路由使用`async def`

### 2.2 Embedding的同步调用

**【代码事实】** `app/modules/ingestion/service.py:96`

```python
# 阶段5: Embedding
embeddings = asyncio.run(self._embed_chunks(text_chunks))
```

**同样的阻塞问题**：
- `_embed_chunks`是异步函数，但用`asyncio.run()`同步调用
- 批量Embedding时可能长时间阻塞

```python
def _load_cache(paper_dir: Path) -> dict[str, str]:
    """加载图片描述缓存."""
    cache_path = paper_dir / CACHE_FILENAME
    if cache_path.exists():
        try:
            return json.loads(cache_path.read_text(encoding="utf-8"))
        except (json.JSONDecodeError, IOError):
            return {}
    return {}
```

### 3.2 Embedding资源调度

**批量策略**：


**【代码事实】** `app/clients/embedding_client.py`

```python
async def embed(self, texts: list[str]) -> list[list[float]]:
    """批量文本向量化."""
    if not texts:
        return []
    
    if len(texts) <= self.batch_size:
        return await self._embed_batch(texts)
    
    # 分批处理
    results = []
    for i in range(0, len(texts), self.batch_size):
        batch = texts[i:i + self.batch_size]
        batch_results = await self._embed_batch(batch)
        results.extend(batch_results)
    
    return results
```

**【模型推断】**：
- ✅ 自动分批，避免单次请求过大
- ✅ 批量大小可配置（默认32）

## 四、FastAPI并发模型

### 4.1 异步路由

**【代码事实】** `app/api/v1/routes/library.py`

```python
@router.post("/import")
async def import_document(
    request: DocumentImportRequest,
    service: IngestionService = Depends(get_ingestion_service),
) -> DocumentRecord:
    """导入文档."""
    record = DocumentRecord(
        title=request.title,
        file_path=request.file_path,
        index_mode=request.index_mode,
        tags=request.tags,
    )
    
    result = service.ingest_document(record)
    return result
```

**￥问题￥：异步路由调用同步服务**

**问题**：
- 路由是`async def`，但`service.ingest_document`是同步函数
- 内部多次`asyncio.run()`会阻塞事件循环

**影响**：
- 其他请求无法并发处理
- 可能导致超时

**建议**：
- 服务改为全异步
- 使用`background_tasks`处理长时间导入

```python
async def ask_question(
```



3. **使用BackgroundTasks**
   ```python
   @router.post("/import")
   async def import_document(
       request: DocumentImportRequest,
       background_tasks: BackgroundTasks,
       service: IngestionService = Depends(),
   ):
       task_id = str(uuid.uuid4())
       background_tasks.add_task(service.ingest_document, task_id, request)
       return {"task_id": task_id}
   ```
