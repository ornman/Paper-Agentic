# 设置页重组方案

> 状态：待实施
> 背景：用户反馈"设置里面的功能太乱了"，需要基于后端 AskRequest 实际能力重新规划

## 4 个分区

### 1. 模型配置
- API URL（文本输入）
- API Key（密码输入）
- 模型选择（**下拉选择**，调用后端 `/models` 接口获取列表，替换当前文本输入）
- 思考模式开关（修复 `toggleThinking()` 缺失 bug）

### 2. 对话行为（后端已支持，前端未暴露）
- 反思模式开关 → 对应 `AskRequest.reflection: bool`
- RAG 检索开关 → 对应 `AskRequest.enable_rag: bool`

### 3. 主题与界面
- 深色/浅色主题切换
- 字体大小（Store 已有 `fontSize`，缺 UI）

### 4. 存储管理（Store 已有方法，缺 UI 入口）
- 显示存储用量 → `estimateStorageUsage()`
- 清理缓存按钮 → `clearCache()`
- 导出数据按钮 → `exportData()`

## 从设置页移除
- **历史会话列表** → 移到 ChatView 侧边栏或单独页面
- **文献库管理** → 独立 `/library` 页面

## 清理（Store 里定义了但没人用，后端 per-request 也不接受）
- 移除 `chunkSize`、`retrievalCount`、`temperature`、`contextLength`

## 涉及文件
- `frontend/src/views/SettingsView.vue` — 重写模板和逻辑
- `frontend/src/stores/settings.ts` — 添加 toggleThinking、主题字段、清理未用字段
- `frontend/src/stores/conversation.ts` — sendPrompt 里传递 reflection 和 enable_rag
- `frontend/src/services/sse-client.ts` — AskRequestPayload 加 reflection 和 enable_rag 字段
- `frontend/src/router/index.ts` — 可选：添加 /library 路由
