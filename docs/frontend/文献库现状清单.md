## 状态管理（Pinia Store）

### Library Store (`stores/library.ts`)

**核心状态：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `papers` | `PaperItem[]` | 论文列表 |
| `trashedPapers` | `PaperItem[]` | 回收站列表 |
| `loading` | `boolean` | 加载状态 |
| `error` | `string \| null` | 错误信息 |
| `selectedPaperIds` | `string[]` | 已选论文 ID |
| `importQueue` | `ImportQueueItem[]` | 导入队列 |
| `importError` | `string \| null` | 导入错误 |

**关键 Actions：**

| 方法 | 说明 |
|------|------|
| `loadPapers()` | 从 API 获取论文列表 |
| `removePaper(id)` | 软删除（进回收站） |
| `loadTrashedPapers()` | 加载回收站 |
| `restorePaperFromTrash(id)` | 恢复论文 |
| `permanentDeleteFromTrash(id)` | 永久删除 |
| `importFiles(files)` | 批量导入（含队列管理、去重、进度轮询） |
| `resumeImports()` | 刷新后恢复中断的导入 |
| `monitorImportStatus(taskId, queueIdx)` | 长轮询导入进度（1s 间隔，最多 30 次连续失败） |
| `retryQueueItem(index)` | 重试失败的导入项 |

**持久化：** 导入队列通过 localStorage (`paper-agentic-import-queue`) 跨刷新保持。

### UI Store (`stores/ui.ts`)

与文献库相关的状态：
- `readerOpen` / `readerPaperId` — PDF 阅读器开关和当前论文
- `sidebarOpen` / `sidebarTab` — 侧边栏开关和当前 tab（history/library）

## 设计决策

### 界面布局：保留原版（main 分支）

重做时**保留 main 分支的界面布局结构**，不做大的布局变更。原版布局特点：

```
┌──────────────────────────────────┐
│  搜索栏 [🔍 输入框] [清除] [排序▾]  │  ← sticky header
│  年份筛选 ▾   作者筛选 ▾   [上传]  │
├──────────────────────────────────┤
│  ☐ 全选    共 N 篇               │
│ ┌──────────────────────────────┐ │
│ │ ☐ 论文标题 (高亮)             │ │
│ │   作者 · 年份 · 页数          │ │
│ │   [关键词] [关键词]           │ │
│ │                [👁] [🗑]      │ │
│ └──────────────────────────────┘ │
│  ... more cards ...              │
│  共 N 篇论文，已选 N 篇          │  ← library-summary
├──────────────────────────────────┤
│  导入队列（内联渲染）             │  ← footer，非独立组件
│  导入错误横幅                    │
│  [回收站 N]    [批量删除]         │  ← 底部操作栏
└──────────────────────────────────┘

拖拽覆盖层 + 删除确认弹窗（内联 Teleport）
```

原版与 worktree 版的主要结构差异：

| 区域 | 原版（main） | worktree 版 |
|------|-------------|------------|
| 导入队列 | 内联 HTML，直接写在 LibraryPanel 里 | 拆为 `ImportQueueList` 组件 |
| 删除确认 | 内联 `<Teleport>` + 手写弹窗 | 拆为 `DeleteConfirmDialog` 组件 |
| 回收站 | 无独立组件，内联逻辑 | 拆为 `TrashPanel` 组件 |
| 单文件导入 | 有 `importing / importFileName / importPercent / importStep` 状态 | 已移除，统一走 importQueue |
| 底部操作栏 | 回收站按钮 + 批量删除按钮 | 同原版 |

## 重做时需要关注的要点

1. **Bug #1（CSS 丢失）和 #2（重试进度丢失）必须在新版本中修复**
2. **保留原版界面布局**，在此基础上修复 Bug 和优化代码质量
3. **导入流程**是核心路径（上传→队列→轮询→持久化），重做时需保留全部逻辑
4. **搜索系统**（Fuse.js + 过滤 + 排序 + 高亮）可复用，已是独立 composable
5. **删除流程**（软删除→回收站→永久删除 + 确认弹窗）链路较长，需保持
6. **类型安全**：API 层的强转需要改为运行时校验（zod 或手动验证）
7. **死代码清理**：SSE 接口、`findSimilar` 未接入的部分
