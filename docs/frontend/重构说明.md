## 1. 变更概览

### 1.1 设计理念

基于 Anthropic [frontend-design](https://github.com/anthropics/skills/blob/main/skills/frontend-design/SKILL.md) 设计原则，对前端进行了完整重构：

- **无侧边栏**：聊天区域获得全部可用宽度
- **护眼暖色调**：`#f4f1ea` 底色 + `PingFang SC` 字体
- **块级渲染**：后端返回 JSON → ContentRenderer 逐块渲染（非原始 markdown）
- **DeepSeek 风格引用**：小药丸 `¹` 标注 + hover 1s 预览 + click WPS 打开 PDF
- **可折叠思考**：AI 思考过程可展开/折叠
- **模型配置**：API Key / URL 输入 → 拉取模型列表 → 选择 → 思考开关

### 1.2 架构对比

| | V1（旧） | V2（新） |
|---|---|---|
| 主布局 | App.vue 含侧边栏 + 聊天面板 | ChatView 全宽，无侧边栏 |
| 侧边栏 | HistorySidebar 内嵌历史+文献 | 跳转独立 `/settings` 页面 |
| AI 消息 | 原始 markdown → v-html | JSON blocks → ContentRenderer → 结构化组件 |
| 引用标注 | `[1][2]` 按钮式 | DeepSeek 风格小药丸 + `·` 末尾标记 |
| 悬停预览 | 无 | hover 1s → 弹出 PDF 名+时间+摘录 |
| 打开 PDF | 无 | click → WPS ShellExecute |
| 思考过程 | 无 | 可折叠 ThinkingSection |
| 模型配置 | 无 | ModelSelector 面板 |
| CSS 变量 | `--claude-*` 与 `--color-*` 混用 | 统一 `--color-*` 设计系统 |
| 路由 | 无 | vue-router Hash 路由 |
| 图标 | emoji | Heroicons SVG |

## 2. 后端 API 需求文档

### 2.1 已实现（V1 兼容，无需改动）

这些接口在 V1 后端已存在，V2 前端直接沿用：

#### 文献管理

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|---|---|---|---|---|
| `GET` | `/api/v1/papers` | 获取论文列表 | — | `{ papers: PaperItem[] }` |
| `DELETE` | `/api/v1/papers/{paper_id}` | 删除论文 | — | `{ message: string }` |
| `POST` | `/api/v1/import/start` | 上传 PDF 开始导入 | `multipart/form-data: file` | `{ task_id, status }` |
| `GET` | `/api/v1/import/status/{task_id}` | 轮询导入状态 | — | `ImportStatus` |
| `GET` | `/api/v1/import/stream/{task_id}` | SSE 导入进度 | — | `text/event-stream` |
| `GET` | `/api/v1/papers/{paper_id}/open` | 获取 PDF 路径 | — | `{ file_path: string }` |

#### 对话系统

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|---|---|---|---|---|
| `GET` | `/api/v1/conversations/list` | 历史会话列表 | `?limit=50` | `ConversationSummary[]` |
| `GET` | `/api/v1/conversations/{session_id}` | 回放历史会话 | — | `{ messages: [...] }` |

#### 系统

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/v1/health` | 健康检查 |

**PaperItem 类型：**
```typescript
interface PaperItem {
  paper_id: string
  title: string
  authors: string
  file_path: string
  file_hash: string
  chunk_count: number
  total_pages: number
  import_time: string

```

### 2.2 必须实现（V2 新增，前端已有完整调用逻辑）

#### ✅ 核心：SSE 对话流式接口（重写事件协议）

**`POST /api/v1/query`**

请求体不变：
```json
{
  "session_id": "v1-session-xxx",
  "prompt": "用户提问...",
  "paper_ids": ["paper_1", "paper_2"],
  "enable_rag": true,
  "model": "deepseek-chat",
  "thinking": true
}
```

响应改为多类型 SSE 事件流：

```
event: thinking
data: {"text": "好的，用户询问Transformer...", "time_ms": 3200}

event: thinking
data: {"text": "→ 检索关键词：Transformer architecture", "time_ms": 3200}

event: block
data: {"type": "paragraph", "text": "Transformer架构的核心创新在于...", "citations": [{"sourceId": "src_1"}]}

event: block
data: {"type": "heading", "level": 2, "text": "核心创新"}

event: block
data: {"type": "citation_block", "text": "在Attention Is All You Need中提到...", "sourceIds": ["src_1"]}

event: block
data: {"type": "list", "ordered": false, "items": ["自注意力机制...", "位置编码..."]}

event: block
data: {"type": "table", "headers": ["模型", "参数量"], "rows": [["Transformer", "65M"]]}

event: block
data: {"type": "code", "language": "python", "code": "import torch\n..."}

event: sources
data: [{"id":"src_1","title":"Attention Is All You Need","paper_id":"...","file_path":"D:\\papers\\attention.pdf","page":3,"import_time":"2026-05-20","content":"We propose a novel network architecture..."}]

event: done
data: {}
```

**SSE 事件协议说明：**

| 事件名 | 触发时机 | data 类型 | 前端处理 |
|---|---|---|---|
| `thinking` | 模型思考过程中逐段发送 | `{ text: string, time_ms: number }` | 追加到 `ThinkingSection` 可折叠区域 |
| `block` | 思考结束后，逐块发送回答内容 | `ContentBlock`（见下文） | 追加到 `ContentRenderer`，按 block.type 分发渲染 |
| `sources` | 回答结束后一次性发送 | `SourceCard[]` | 更新底部 `SourcePanel` 卡片网格 |
| `done` | 流结束 | `{}` | 标记 `status='done'` |
| `error` | 发生错误 | `{ message: string }` | 显示错误提示 |

**ContentBlock 类型：**

```typescript
// 段落（可含引用）
{ type: "paragraph", text: string, citations?: { sourceId: string }[] }

// 标题
{ type: "heading", level: 1 | 2 | 3 | 4, text: string }

// 列表
{ type: "list", ordered: boolean, items: string[] }

// 引用段落（标记了某篇论文的引用来源）
{ type: "citation_block", text: string, sourceIds: string[] }

// 表格
{ type: "table", headers: string[], rows: string[][] }

// 代码块
{ type: "code", language: string, code: string }

// 分隔线
{ type: "divider" }
```

**SourceCard 类型（需要后端在 sources 事件中返回完整字段）：**

```typescript
{
  id: string           // 来源唯一标识，对应 citation_block 的 sourceIds
  paper_id?: string
  title: string        // PDF 文件名
  page?: number        // 页码
  section?: string     // 章节名
  file_path?: string   // 本地 PDF 绝对路径（用于 WPS 打开）
  local_path?: string  // 本地路径备选
  content?: string     // 原文摘录（用于悬停预览）
  import_time?: string // 论文导入时间（用于预览卡片右上角）
}
```

#### ⚠️ 重要：SourceCard 必须包含 `import_time`

前端预览卡片在右上角显示 `2026-05-20` 这样的日期，需要后端在 `sources` 事件返回的数据中附带 `import_time` 字段。这个字段在论文导入时已知（`PaperItem.import_time`），需要在组装 SourceCard 时一并透传。

#### 模型列表

**`POST /api/v1/models`**（暂未实现，前端调用逻辑已就绪）

## 4. 开发调试

### 浏览器调试

```bash
cd frontend
npm run dev          # → http://localhost:3904
```

- 浏览器下 `window.wps` 不存在，click 引用的 PDF 打开回退为 `window.open(file://...)`
- axios 请求 `localhost:8000`，需后端 CORS 放行或 vite proxy

### WPS 插件打包

```bash
npm run build        # → dist/
cd dist
npx wpsjs debug      # 注册到 WPS
```

- 构建时自动内联 taskpane.html 的 JS/CSS 为 IIFE
- WPS 内嵌浏览器不支持 ES Module，内联后兼容
- 如遇 CORS 问题，在 vite.config.ts 中配置 proxy

## 6. 引用交互链路详解

```
用户提问 → POST /api/v1/query (SSE)
  ↓
thinking 事件 → ThinkingSection 展示思考过程（可折叠）
  ↓
block 事件 → ContentRenderer 逐块渲染
  ↓ (citation_block 类型)
BlockCitation 渲染：
  ├─ 左边框 3px rgba(#0066cc, 0.3)
  ├─ 段落正文
  └─ 末尾 · （浅灰，hover → 蓝色）
        ↓ hover 1s
      CitationPreview 弹出：
        ┌──────────────────────────────┐
        │ Attention Is All You Need    │  2026-05-20
        │ 第 3 页                      │
        ├──────────────────────────────┤
        │ "We propose a novel network  │
        │  architecture..."            │
        └──────────────────────────────┘
        ↓ click
      wps.OAAssist.ShellExecute(file_path) → WPS 打开本地 PDF
  ↓
sources 事件 → SourcePanel 底部卡片网格
  ↓
done 事件 → status = 'done'
```
