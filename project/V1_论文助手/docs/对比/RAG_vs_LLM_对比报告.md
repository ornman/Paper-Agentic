## 结论

1. **延迟**: RAG 多出检索+向量化环节，首字延迟平均多 1-2s（排除 Q2 异常值后）
   - Q2 的 48s TTFF 是 DeepSeek API 偶发慢响应，非 RAG 检索导致
   - 正常情况下 RAG TTFF 约比纯 LLM 多 0.5-1.5s（向量化 + Zvec 检索开销）
2. **Token 消耗**: RAG 输入 tokens 大幅增加（含 30000 token 上下文），输出 tokens 也更多（回答更详细）
3. **回答质量**: RAG 能引用具体论文来源，回答有据可查；纯 LLM 回答泛泛而谈、缺乏学术支撑
4. **适用场景**: 通用问答用纯 LLM 即可；涉及论文内容的问题必须用 RAG
5. **性价比**: RAG 用 ~70 倍的输入 tokens 换取了可引用、可溯源的专业回答，对于学术场景值得
