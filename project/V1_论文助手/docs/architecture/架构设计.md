## 数据流

### 导入管道（6 阶段）

```
PDF 上传
  │
  ▼
[1] MinerU API 解析 → ZIP (full.md + layout.json + images/)
  │
  ▼
[2] VLM 图片描述注入 → MD 中 ![](img) 变为 ![hash|描述](img)
  │                     每批 5 张打包调用 Kimi API，失败逐个 fallback
  ▼
[3] 清洗 → 区域检测 (cover_toc / body / end_noise)
  │         过滤噪音（页眉页脚、答辩表、致谢等）
  ▼
[4] 切块 → 固定 1000 token/块，段落边界对齐
  │         超长段落按 2000 token 上限再切
  ▼
[5] Embedding → 硅基流动 Qwen3-Embedding-8B (1536维)
  │              批量 32 条/次，并发 5，429 自动重试
  ▼
[6] 存储 → Zvec (向量 + payload) + BM25 (关键词) + SQLite (元数据)
```

### 问答管道

```
用户提问 (prompt + selection + draft)
  │
  ▼
[1] Embedding → 查询向量化
  │
  ▼
[2] Zvec 密集检索 (topk=10) → 可选 paper_id 过滤
  │
  ▼
[3] 构建上下文 (token 预算 30k) + 来源标注
  │
  ▼
[4] DeepSeek LLM 流式生成 → SSE 事件流
  │                           metadata → chunk → done
  ▼
[5] Redis 保存对话历史 (TTL 1h)
```

## 模块依赖关系

```
api/routes/ ──依赖──→ pipelines/service ──依赖──→ clients/
     │                      │                      │
     │                      └──依赖──→ stores/     │
     │                                      │      │
     └──依赖──────────────────────────────→ │ ←────┘
                                            │
                                       deps.py (DI 容器)
```

**依赖方向**: routes → services → clients/stores，无循环依赖。

## 存储设计

### Zvec（向量库）

- 单 Collection `papers`，payload 字段做过滤
- Schema: `paper_id`, `file_hash`, `chunk_type`, `chunk_index`, `content`, `source_page`, `section_title`, `has_image`
- 过滤语法: `paper_id = 'xxx'`（单等号）
- CRUD: insert → query(filter) → delete_by_filter

### SQLite（元数据）

- `papers` 表: paper_id, title, authors, file_path, file_hash, file_size, chunk_count, import_time, status
- `import_logs` 表: task_id, paper_id, status, stage, message, created_at
- `config` 表: key-value 配置存储
- 索引: paper_id, file_hash, status

### BM25（关键词检索）

- jieba 分词 + rank_bm25
- 文档 ID 格式: `{paper_id}_{chunk_index}`
- 删除时按 paper_id 前缀过滤

### Redis（缓存，可选）

- 对话历史: `conv:{session_id}` LIST，TTL 1h
- 轮询缓存: `poll:{session_id}` STRING，TTL 1h
- 不可用时优雅降级，不影响核心功能

## 外部服务

| 服务 | 用途 | 协议 | 特殊要求 |
|------|------|------|----------|
| MinerU | PDF 解析 | REST API | 异步轮询，自适应间隔 |
| Kimi VLM | 图片描述 | Anthropic 兼容 | UA: `claude-code`, header: `anthropic-version` |
| DeepSeek | LLM 对话 | OpenAI 兼容 | SSE 流式 |
| SiliconFlow | Embedding | OpenAI 兼容 | 批量 32 条，429 重试 |

## 错误处理

错误码分段:
- `1xxx`: 客户端错误（参数校验）
- `2xxx`: 业务错误（论文不存在、导入失败）
- `3xxx`: 外部服务错误（API 调用失败）

所有错误继承 `AppError(HTTPException)`，统一通过 FastAPI exception handler 返回。
