# V1 开发指南

> 2026-04-19

## 环境准备

### 必要工具

- Python 3.13+
- uv (包管理)
- Redis (可选，不可用时降级)

### 启动

```bash
cd backend
uv sync
cp .env.example .env   # 填写 API keys
uv run python main.py
```

服务启动在 `http://localhost:8000`。

### 配置项 (.env)

| 变量 | 必填 | 说明 |
|------|------|------|
| `LLM_API_KEY` | ✓ | DeepSeek API Key |
| `LLM_BASE_URL` | | 默认 `https://api.deepseek.com` |
| `LLM_MODEL` | | 默认 `deepseek-chat` |
| `KIMI_API_KEY` | ✓ | Kimi VLM API Key |
| `EMBEDDING_API_KEY` | ✓ | 硅基流动 API Key |
| `MINERU_API_KEY` | ✓ | MinerU API Key |
| `EMBEDDING_MODEL` | | 默认 `Qwen/Qwen3-Embedding-8B`（不可改，改了索引失效） |
| `EMBEDDING_DIMENSION` | | 默认 `1536`（不可改） |

### 测试

```bash
uv run pytest tests/ -v
```

## 开发原则

1. **先跑通，再优化** — 不过早工程化，Demo 阶段暴力解决
2. **代码是真相源** — 文档只记"为什么"，不记"是什么"
3. **不可变数据** — 始终创建新对象，不修改已有对象
4. **小文件** — 典型 200-400 行，上限 800 行
5. **Redis 可选** — 所有 Redis 调用需要 try/except 降级

## 已验证的管道

以下流程已经端到端验证通过：

- MinerU PDF 解析 → full.md (78k chars) + 106 images
- VLM 图片描述 → 74/74 张成功注入（批量 5 张/请求 + 单图 fallback）
- 清洗 → 136 chunks, 噪音过滤正常
- 切块 → 96 chunks, 固定 1000 token
- Embedding → 96 vectors, 1536 维
- Zvec CRUD → insert/query(filter)/delete 全部通过
- BM25 CRUD → add/query/delete 全部通过
- SQLite CRUD → insert/query/update/delete 全部通过
- LLM 对话 → DeepSeek 流式输出正常

## 已知限制

- BM25 删除是全量重建索引（rank_bm25 不支持增量删除）
- Zvec filter 只支持 `=` 单条件，不支持组合过滤
- 切块器不感知表格结构，表格可能被切断
- 清洗器的英文论文支持较弱（区域检测正则偏中文）
- Redis 不可用时对话历史和轮询功能降级
