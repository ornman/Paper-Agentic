# V1 开发指南

> 2026-04-25

## 环境准备

### 必要工具

- Python 3.13+
- uv
- Node.js 18+
- pnpm

### 启动

```bash
# 后端
cd backend
uv sync
cp .env.example .env
uv run python main.py

# 前端
cd ../frontend
pnpm install
pnpm dev --host 127.0.0.1 --port 3904
```

- 后端：`http://127.0.0.1:8000`
- 前端：`http://127.0.0.1:3904/app.html`

### 配置项（backend/.env）

| 变量 | 必填 | 说明 |
|------|------|------|
| `KIMI_API_KEY` | ✓ | Kimi Coding API Key |
| `KIMI_BASE_URL` | | 默认 `https://api.kimi.com/coding/v1` |
| `SILICONFLOW_API_KEY` | ✓ | 硅基流动 API Key |
| `MINERU_API_KEY` | ✓ | MinerU API Key |
| `EMBEDDING_MODEL` | | 默认 `Qwen/Qwen3-Embedding-8B` |
| `EMBEDDING_DIMENSIONS` | | 默认 `1536` |

## 当前已稳定链路

### 1. PDF 导入

后端导入阶段固定为：

```text
parsing → vlm → cleaning → chunking → embedding → storing
```

当前行为：

- 上传文件统一写入 `backend/data/uploads`
- 数据统一落在 `backend/data`
- `backup.json` 记录 `task_id`、`current_stage`、`last_error`
- 导入失败时返回明确阶段错误
- 重复 PDF 按 `file_hash` 拦截
- 已有备份时可从未完成阶段恢复

### 2. 检索增强

当前行为：

- 只有用户选中文献时才传 `enable_rag=true`
- 检索使用 Dense + BM25 融合
- 返回来源包含标题、页码、节名、摘录
- 未选中文献时直接走 LLM

### 3. 历史对话

当前行为：

- 对话保存在 SQLite `conversations` 表
- 侧栏可查看历史列表
- 新对话完成后自动刷新列表
- 点击历史项可恢复消息内容
- 支持批量删除
