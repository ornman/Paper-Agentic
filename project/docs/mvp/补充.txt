改写策略是分情况的：
可能（1）：用户什么都没写，只有向量化的东西，这时候只能通过 prompt 来起手
[prompt 权重 100%，根据这个去 query 改写去匹配语义向量 ]

可能（2）：用户已经写了，但是用户没有圈选，而且没有 prompt：这种时候（用户点击发送按钮），就是根据用户已有的上下文，推断接下来可能的句子（在右边对话框内输出）
[ " 用户已写的内容"权重 100%，根据这个去 query 改写去语义匹配 ]

可能（3）：用户已经写了，而且用户圈选了，但是没有 prompt：这种时候（用户点击发送按钮），根据用户已有的上下文，加上用户圈选出来的上下文，推断接下来可能的句子。
[ "用户已写的内容 " 权重 30%，"用户圈选出来的内容 "权重 70%，组合他们去 query 改写去语义匹配 ]

可能（4）：用户已经写了，而且用户圈选了，而且有 prompt：这种时候（用户点击发送按钮），根据用户已有的上下文，加上用户圈选出来的上下文，加上用户的 prompt， 推断接下来可能的句子
["用户已写的内容 " 权重 20 % ，"用户圈选出来的内容 "权重 30%，" 用户 prompt"权重 50%，组合他们去 query 改写去语义匹配]

K的参数默认是60，跑测试的；如果头部太弱、长尾噪音多就调小K;如果头部过强，漏了有价值的中排名文档就调大k（这个未来我们会做一个自适应区间）

这个权重是prompt权重，是三种query分别去匹配，然后召回（打包为一个或多个chain：一次性操作，只给agent返回过滤无关内容后的结果）
