# MVP技术文档

## 一、背景

当前学术写作场景中，高校学生、科研工作者普遍存在以下痛点：

1. **文献管理与复用效率低**：用户积累的大量参考文献，写作时无法快速定位与当前写作内容匹配的论据、观点，反复查阅文献耗时耗力；
2. **引用溯源难度大**：写作中难以快速追溯观点、数据的原始文献来源，引用格式规范调整成本高；
3. **现有工具能力不匹配**：通用AI写作工具无法基于用户私有文献库生成内容，易出现编造文献、内容失真问题；查重工具仅能完成事后重复率检测，无法在写作过程中提供实时匹配与规避支持。

基于以上痛点，需打造一款以用户自定义知识库为核心的论文写作辅助工具，深度嵌入写作流程，解决学术写作的效率与严谨性痛点。

## 二、目标

1. 以 WPS 插件的形式集成
2. 基于知识库问答
3. 溯源（页码级 + bbox 坐标）
4. 对话框给出路径，进行全自动清洗向量化流程

## 三、非目标

1. 明确无需全流程的 Agentic
2. 无需知识图谱
3. 无需 Word 适配

## 四、范围

### 前端

1. 插件布局
2. 与后端的桥接口（HTTP + SSE）

### 后端

1. 数据清洗（MinerU API：PDF → Markdown + JSON）
2. 语义切片（父子块）
3. 索引与算法（ChromaDB + BM25）
4. Embedding（ChromaDB 自动向量化 + Query 改写）
5. Reranker 重排序
6. BM25 关键词筛选
7. LangChain/LangGraph（自动化 memory 上下文管理 + 工具注册）
8. FastAPI + Pydantic（规范化暴露接口）
9. SQLite 会话管理（对话历史 + 工具调用记录）

## 五、边界

1. MVP 阶段使用 MinerU API，后续研究源码本地化
2. 仅支持 PDF 格式文献导入
3. 单机部署，无多用户并发需求
4. 不支持实时协作编辑

## 八、数据与接口

### 8.1 API 接口设计

| 接口 | 方法 | 功能 | 请求 | 响应 |
|------|------|------|------|------|
| `/ingest` | POST | 导入 PDF，执行清洗 + 切片 + 索引 | `{ "pdf_path": "D:/papers/xxx.pdf" }` | `{ "task_id": "abc123", "status": "processing" }` |
| `/status/{task_id}` | GET | 查询索引构建状态 | - | `{ "status": "completed", "chunks": 156 }` |
| `/session/create` | POST | 创建新会话 | `{ "title": "论文写作" }` | `{ "session_id": "sess_001" }` |
| `/session/list` | GET | 获取会话列表 | - | `{ "sessions": [...] }` |
| `/session/{id}/history` | GET | 获取会话历史 | - | `{ "messages": [...] }` |
| `/session/{id}` | DELETE | 删除会话 | - | `{ "status": "deleted" }` |
| `/query` | POST | RAG 检索 + 生成（SSE 流式） | 见下方 | Server-Sent Events |
| `/sync` | POST | 常驻循环同步（5秒） | `{ "session_id": "...", "written_content": "..." }` | `{ "status": "synced" }` |

#### `/query` 请求结构

```json
{
  "session_id": "sess_001",
  "written_content": "深度学习在图像识别...",
  "selected_content": "卷积神经网络的池化层",
  "prompt": "解释池化层的作用",
  "mode": "qa"
}
```

#### `/query` SSE 响应

```
data: {"type": "sources", "data": [...]}
data: {"type": "content", "text": "..."}
data: {"type": "content", "text": "..."}
data: {"type": "done"}
```

### 8.3 会话数据库 Schema（SQLite）

```sql
-- 会话表
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,
    title TEXT,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    metadata JSON
);

-- 消息表
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT,
    role TEXT,                     -- "user" / "assistant" / "system"
    content TEXT,
    timestamp TIMESTAMP,
    metadata JSON,                 -- 工具调用、来源、模式等
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);

-- 工具调用表
CREATE TABLE tool_calls (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    message_id INTEGER,
    tool_name TEXT,                -- "query_rewrite" / "rag_search" / "rerank"
    tool_input JSON,
    tool_output JSON,
    timestamp TIMESTAMP,
    FOREIGN KEY (message_id) REFERENCES messages(id)
);

-- 索引
CREATE INDEX idx_messages_session ON messages(session_id);
CREATE INDEX idx_messages_timestamp ON messages(timestamp);
```

## 九、风险与异常处理

| 阶段 | 风险 | 处理方案 |
|------|------|---------|
| **数据清洗** | MinerU API 调用失败 / 解析质量差 | 超时重试（3次，指数退避）；API 失败提示重试；质量低分标记待审核；大文件分片 + 断点续传 |
| **向量索引** | Embedding API 限流 / ChromaDB 写入失败 | 批量请求 + 速率控制；写入失败本地缓存后台重试；索引损坏从 documents/ 重建 |
| **检索** | 无匹配结果 / 检索质量差 | 无匹配返回提示；匹配度低扩大召回；Query 改写失败用原始 Query |
| **LLM 生成** | API 限流 / 响应超时 / 幻觉 | 限流排队提示；超时 SSE 断开重试；Prompt 强制引用来源；多模型备份切换 |
| **会话管理** | SQLite 损坏 / 会话丢失 | 定期自动备份（每10次对话）；SQLite WAL 模式；损坏从备份还原 |
| **前端通信** | SSE 连接断开 / 网络不稳定 | 自动重连（最多3次）；断线期间消息缓存；心跳检测（30秒） |

### 边界情况处理

- PDF 路径不存在 → 提示用户检查路径
- PDF 加密/损坏 → 提示文件无法解析
- 文献库为空 → 引导用户先导入 PDF
- 会话历史过长 → 自动摘要旧对话
- 用户输入为空 → 忽略，不触发检索
